## 二、技能分层架构（Base + Override）

`.claude/` 下的技能体系采用三层架构，实现 HE 通用能力与项目定制的分离：

```
linscode/skills/{path}/SKILL.md       ← HE Base 层（git pull 更新，只读）
.claude/skills/{name}/OVERRIDES.md    ← Override 层（仅差异，项目维护）
.claude/commands/{name}.md            ← 路由层（决定加载路径）
.claude/rules/ + docs/                ← 规则层（不动）
```

### 44 个命令的路由模式

| 路由模式 | 数量 | 说明 |
|---------|------|------|
| **Pure HE** | 24 | command 直接指向 `linscode/skills/{path}/SKILL.md`，无项目定制 |
| **Override** | 14 | command 加载 Base(linscode) + Override(.claude) 合并执行 |
| **项目专属** | 5 | command 指向 `.claude/skills/{name}/SKILL.md`，HE 中不存在 |
| **巡检工具** | 1 | `/check-overrides` — 比对 Override 与 HE Base 的版本一致性 |

### 14 个 Override Skill

Override 目录仅包含差异文件（OVERRIDES.md + 本地定制的 assets/references），不保存 HE Base 的完整副本。

| Skill | 覆盖数 | 本地定制资产 |
|-------|-------|------------|
| test-case-design | 2 | references/smoke-test-standard.md |
| test-case-review | 2 | assets/case-review-template.md |
| test-point-extraction | 1 | assets/test-points-template.md |
| test-execution | 1 | — |
| api-test-execution | 1 | — |
| api-test-case-design | 1 | — |
| test-case-prd-consistency | 1 | references/consistency-checklist.md |
| test-data-preparation | 1 | references/variable-handoff-protocol.md |
| visual-location-fallback | 1 | — |
| multimodal-visual-assertion | 1 | — |
| tech-doc-review | 2 | assets/review-templates.md, references/review-dimensions.md |
| backend-tech-design-mvc | 2 | — |
| backend-solution-review | 3 | assets/review-report-template.md, references/review-checklist.md |
| backend-tech-design-presentation | 3 | — |

### 5 个项目专属 Skill

HE 中不存在，完整定义保留在 `.claude/skills/` 下：

- **acceptance-criteria** — 验收检查
- **code-implementation** — 接口联调实施计划
- **requirements-update** — 需求变更同步
- **system-function-analysis** — 系统功能梳理（含 4 个 references/）
- **yunxiao-sync** — 云效 Testhub 同步（含 Python 脚本）

### 日常维护

| 场景 | 操作 |
|------|------|
| HE 升级后 | 运行 `/check-overrides` 检查版本漂移 |
| 新增项目定制 | 创建 `.claude/skills/{name}/OVERRIDES.md` + 改 command 为 Override 路由 |
| 新增 HE Skill（无定制） | 创建 command 指向 linscode，无需在 .claude/skills/ 建目录 |
| 移除定制 | 删除 OVERRIDES.md + 改 command 为 Pure HE 路由 |

## 三、三个核心维度

### 维度一：systems/ — 系统知识库

**定位**：按系统组织的**持久化知识**，不随迭代消失，只会持续增长。

| 文件 | 作用 | 更新时机 |
|------|------|---------|
| `{系统}/knowledge/功能清单.md` | 模块列表、接口、权限、已知问题 | 迭代完成后 |
| `{系统}/knowledge/changelog.md` | 变更记录，含迭代链接 | 迭代启动时 |
| `{系统}/knowledge/data-testid/` | UI 元素定位标识映射 | 新增 data-testid 后 |
| `_shared/系统关系图.md` | 系统间调用关系和数据流 | 新增跨系统交互时 |
| `_shared/枚举值字典.md` | 全局统一枚举定义 | 新增枚举值时 |
| `_shared/代码仓库管理.md` | 仓库总索引 + 管理规范 | 新增仓库时 |

### 维度二：iterations/ — 迭代管理

**定位**：按迭代组织的**项目推进**目录，每次迭代一个文件夹。

**命名规则**：`{年份}-{季度}_{系统}-{版本}_{主题}`

| 子目录 | 存放内容 |
|-------|---------|
| `input/prd/` | PRD 文档 + 原型（README.md 标注基线版本） |
| `input/tech/` | 技术设计文档 |
| `review/` | PRD 审阅报告、技术文档审阅报告、测试点清单 |
| `testcase/` | Markdown 测试用例套件（suite_*.md）+ 测试数据 |
| `report/` | 测试报告 + 执行截图 |

### 维度三：.claude/ — AI 工程配置

**定位**：Claude Code 的规则、技能、命令体系。采用 Base+Override 分层架构。

| 目录 | 内容 | 加载方式 |
|------|------|---------|
| `rules/` | 基础规则（rules.md） | 每次对话自动注入 |
| `docs/` | 11 个规则/参考文档（~240KB） | Skill 执行时按需读取 |
| `commands/` | 44 个 /斜杠命令（路由层） | 用户输入 `/命令名` 触发 |
| `skills/` | 14 个 Override + 5 个项目专属 + 1 个 _shared | 由 command 路由调用 |
| `secrets/` | 敏感凭证 | 测试执行时按需读取 |

## 四、两个维度如何串联

```
iterations/ (找迭代改了什么)          systems/ (找系统被谁改过)
       │                                    │
       ▼                                    ▼
  README.md                           changelog.md
  "涉及系统"表格 ──引用链接──→        "变更记录"表格
  链接到 systems/                     链接到 iterations/
```

**从迭代找系统**：`iterations/{迭代}/README.md` → 涉及系统表格 → 链接到 `systems/{系统}/knowledge/`

**从系统找迭代**：`systems/{系统}/knowledge/changelog.md` → 变更记录 → 链接到 `iterations/{迭代}/`

## 五、模板使用说明

### 开始新迭代

1. 在 `iterations/` 下创建目录（命名：`{年份}-{季度}_{系统}-{版本}_{主题}`）
2. 复制 `templates/iteration_readme.md` → `README.md`，填写涉及系统和产出物
3. 创建子目录：`input/prd/` `input/tech/` `review/` `testcase/` `report/`
4. 在 `input/prd/` 下创建 `README.md` 标注 PRD 基线版本
5. 在每个涉及系统的 `changelog.md` 中追加变更记录

### 新增系统

1. 在 `systems/` 下创建 `{系统名}/knowledge/`
2. 复制 `templates/system_knowledge.md` → `功能清单.md`，复制 `templates/changelog.md`
3. 如有跨系统交互，更新 `_shared/系统关系图.md`

### 使用 AI 测试工作流

```
D1 PRD 审阅     → /test-prd-review → /prd-gate → /test-point-extraction
D2 技术文档审阅  → /tech-doc-review
D3 用例设计     → /test-case-design → /test-case-review
D4 测试执行     → /test-execution（冒烟+功能）
D5 发布门禁     → /bug-sync → /test-report → /release-gate
```

完整 Skill 列表和触发语句见 `.claude/docs/PROMPT_TEMPLATES.md`。
