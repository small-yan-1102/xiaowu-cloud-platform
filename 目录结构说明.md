## 二、两个核心维度

### 维度一：systems/ — 系统知识库

**定位**：按系统组织的**持久化知识**，不随迭代消失，只会持续增长。

**核心作用**：
- 沉淀每个系统的功能模块、接口、权限、已知问题
- 通过 `changelog.md` 记录"哪个迭代改了我什么"，形成**反向索引**
- 通过 `_shared/` 维护跨系统共享的枚举值、关系图、环境信息

**文件说明**：

| 文件 | 作用 | 更新时机 |
|------|------|---------|
| `{系统}/knowledge/功能清单.md` | 该系统的模块列表、接口、权限、已知问题 | 每次涉及该系统的迭代完成后 |
| `{系统}/knowledge/changelog.md` | 该系统的变更记录，含迭代链接 | 每次涉及该系统的迭代启动时 |
| `{系统}/knowledge/data-testid/` | UI 元素定位标识映射（README 索引 + 按模块拆分） | 迭代新增 data-testid 后同步 |
| `_shared/系统关系图.md` | 系统间调用关系和数据流 | 新增跨系统交互时 |
| `_shared/枚举值字典.md` | 全局统一枚举定义（状态/类型/原因等） | 新增枚举值时 |
| `_shared/环境信息.md` | 测试/预发环境地址和测试账号 | 环境变更时 |

---

### 维度二：iterations/ — 迭代管理

**定位**：按迭代组织的**项目推进**目录，每次迭代一个文件夹。

**核心作用**：
- 存放该迭代的所有输入（PRD）和输出（反审/用例/报告）
- 通过 `README.md` 串联涉及的系统和产出物
- 快速定位某次迭代的所有相关文件

**目录命名规则**：`{年份}-{季度}_{系统}-{版本号}_{迭代主题}`

示例：
```
2026-Q1_AMS-V2.0.0_视频下架
2026-Q1_AMS+CRM_作品管理与交接单改造
2026-Q1_AMS-V1.5.0_译制成品
2026-Q2_CRM-V3.0_合约管理
```

**子目录说明**：

| 子目录 | 存放内容 | 说明 |
|-------|---------|------|
| `input/prd/` | PRD 文档、调研报告、原型文件 | 需求输入物 |
| `review/` | PRD 反审清单、需求总结、审阅报告 | 反审 / 评审产出 |
| `testcase/` | 测试用例（Markdown 格式） | 按模块/功能组织 |
| `report/` | 测试报告、执行记录 | 测试执行产出 |

## 三、两个维度如何串联

```
                    iterations/                           systems/
                    (找迭代改了什么)                        (找系统被谁改过)
                         │                                     │
                         ▼                                     ▼
               ┌──────────────────┐                  ┌──────────────────┐
               │ README.md        │                  │ changelog.md     │
               │ 涉及系统表格中    │ ──引用链接──→     │ 变更记录表格中    │
               │ 链接到 systems/   │                  │ 链接到 iterations/│
               └──────────────────┘                  └──────────────────┘
```

**从迭代找系统**：打开 `iterations/{迭代}/README.md` → "涉及系统"表格 → 点击链接到 `systems/{系统}/knowledge/功能清单.md`

**从系统找迭代**：打开 `systems/{系统}/knowledge/changelog.md` → "变更记录"表格 → 点击链接到 `iterations/{迭代}/`

## 四、模板使用说明

### 开始新迭代

```
步骤 1  在 iterations/ 下创建目录
        命名：{年份}-{季度}_{系统}-{版本}_{主题}

步骤 2  复制 templates/iteration_readme.md → 新目录/README.md
        填写：涉及系统、数据流、产出物索引

步骤 3  创建子目录：input/prd/  review/  testcase/  report/

步骤 4  在每个涉及系统的 changelog.md 中追加变更记录
```

### 新增系统

```
步骤 1  在 systems/ 下创建 {系统名}/knowledge/ 目录

步骤 2  复制 templates/system_knowledge.md → 功能清单.md
        复制 templates/changelog.md → changelog.md

步骤 3  填写系统概述、功能模块初始信息

步骤 4  如有跨系统交互，更新 _shared/系统关系图.md
```

### 新增枚举值

```
步骤 1  打开 systems/_shared/枚举值字典.md

步骤 2  在对应类别下新增行
        填写：枚举值、中文显示、来源迭代、备注

步骤 3  通知相关系统负责人对齐
```
