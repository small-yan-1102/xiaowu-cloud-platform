# 接口执行报告模板库

> 本文件是 api-test-execution skill 的报告输出模板，由 SKILL.md 在 Phase 3 输出报告时加载。
> **替换此文件即可改变报告格式，无需修改 SKILL.md 中的指令逻辑。**

---

## 模板一：单条用例执行结果

每条用例执行完成后，按此模板记录结果。

```markdown
### [用例编号] [用例名称]

**执行结果**：通过 / 失败 / 阻塞 / 跳过
**执行时间**：YYYY-MM-DD HH:mm:ss
**执行耗时**：XXX ms
**执行环境**：{Base URL}

**请求详情**：
- **Method**：{GET/POST/PUT/DELETE/PATCH}
- **URL**：{完整 URL（含路径参数）}
- **Headers**（脱敏）：
  ```json
  {
    "Authorization": "Bearer ***（脱敏）",
    "Content-Type": "application/json"
  }
  ```
- **Body**：
  ```json
  {实际发送的请求体，含变量替换后的值}
  ```

**响应详情**：
- **HTTP 状态码**：{实际状态码}
- **响应体**（关键字段）：
  ```json
  {实际响应体，仅记录断言涉及的字段}
  ```

**断言结果**：

| 断言项 | 级别 | 期望值 | 实际值 | 状态 |
|--------|------|--------|--------|------|
| HTTP 状态码 | L1 | 200 | {实际} | 通过/失败 |
| `data.id` | L1 | 不为空 | {实际值} | 通过/失败 |
| `code` | L1 | 0 | {实际值} | 通过/失败 |
| `data.status` | L2 | "ACTIVE" | {实际值} | 通过/警告 |

**失败详情**（仅失败时填写）：
- 失败断言：{断言项}
- 期望：{预期值}
- 实际：{实际值}

**L2 警告**（仅有软验证不通过时填写）：
- [{断言项}] 期望：XXX，实际：YYY（不影响用例判定）

**变量提取**：
| 变量名 | 提取路径 | 提取值 |
|--------|---------|--------|
| {变量名} | `data.id` | {实际值} |

**清理步骤结果**（如有）：
- [清理] {清理步骤描述}：{通过/失败/已跳过}
  - ⚠️ 清理失败时标注：需人工清理（{数据描述}）

**备注**：{重试情况、阻塞原因、Token 刷新等说明}
```

## 执行结果状态定义

| 状态 | 含义 | 触发条件 |
|------|------|---------|
| **通过** | 所有 L1 断言通过 | 全部 L1 强断言通过（L2 警告不影响） |
| **失败** | 存在 L1 断言不通过 | 任一 L1 强断言未通过 |
| **阻塞** | 无法执行 | 前置条件不满足、依赖变量缺失、依赖用例失败 |
| **跳过** | 主动跳过 | P0 用例失败导致后续用例跳过 |
