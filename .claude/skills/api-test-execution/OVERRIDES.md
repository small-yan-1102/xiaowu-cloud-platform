# api-test-execution 项目定制

## 覆盖 1：清理步骤处理矩阵

**HE 原文位置**：Phase 2 → 步骤 7（执行清理步骤）
**HE 原文摘要**：清理失败时记录警告，不影响当前用例判定，标注需人工清理
**定制为**：替换为结构化处理矩阵：

| 清理结果 | 处理方式 | 影响当前用例判定 | 影响下游用例 |
|---------|---------|----------------|------------|
| 清理成功（2xx） | 正常记录 | 否 | 否 |
| 清理失败（4xx/5xx） | 记录警告，标注「需人工清理：{接口} {错误码}」 | **否** | **可能**：若下游依赖该数据不存在，标注 `⚠️ 数据残留风险` |
| 清理超时 | 重试 1 次；仍失败则同上 | 否 | 同上 |
| 无清理步骤 | 跳过 | 否 | 否 |

> 所有「需人工清理」项汇总到报告末尾「待清理数据」章节。

## 覆盖 2：执行后回写 API 套件「执行清单」段

**HE 原文位置**：Phase 2 → 单条用例执行后记录执行结果到报告
**HE 原文摘要**：执行结果仅写入执行报告文件，套件 md 文件本身保持只读
**定制为**：单条 API 用例执行完成（含清理步骤）后，除写执行报告外，**必须**用 Edit 工具更新**当前迭代**对应 API 套件文件（`iterations/**/testcase/api_suite_*.md`）中 `## 执行清单（状态记录入口）` 段内该用例所在行——勾选 checkbox 并在行尾追加 AI 执行标记 + 报告链接。

> ⚠️ 本覆盖项替代旧版"回写执行顺序表最右列"逻辑。执行顺序表**不再**含「执行记录」列，状态唯一入口是「执行清单」段。

### 写入格式

**匹配目标行**（默认状态）：
```
- [ ] **{用例编号}** {用例名称}（{优先级}）
```

**回写后**：
```
- [x] **{用例编号}** {用例名称}（{优先级}） · {状态 emoji + "AI" 标识} · {执行日期} · [报告](相对路径)
```

### 状态映射

| 执行结论 | 回写结果 |
|---------|--------|
| HTTP 响应码与断言全部符合 | `- [x] **{编号}** {名称}（{优先级}） · ✅ AI {日期} · [报告]({execution_report}#{锚点})` |
| HTTP 响应码/断言任一失败（有 BUG-id） | `- [ ] **{编号}** {名称}（{优先级}） · ❌ AI {日期} · [失败详情]({execution_report}#{锚点}) · BUG-{id}` |
| HTTP 响应码/断言任一失败（无 BUG-id） | `- [ ] **{编号}** {名称}（{优先级}） · ❌ AI {日期} · [失败详情]({execution_report}#{锚点})` |
| 认证失败/依赖用例未通过/接口超时 | `- [ ] **{编号}** {名称}（{优先级}） · 🚫 AI · {阻塞原因}` |
| 用例标注"本轮不执行" | 不回写（暂缓用例不出现在清单段） |

**字段说明**：
- `{编号}`、`{名称}`、`{优先级}` 来自清单段原行，**必须原样保留**
- `{日期}` 格式 `YYYY-MM-DD`，取执行当天
- `{execution_report}` 相对路径如 `report/execution_report_20260417.md`（与老迭代 `report/functional_test_report_*.md` 同目录）
- `{锚点}` 用例编号小写（如 `api-s03-001`）

### 回写时机与失败处理

1. **回写时机**：单条用例及清理步骤结束后**立即**回写，不等整个套件跑完
2. **匹配规则**：用 Grep 在清单段内按 `**{用例编号}**` 精确匹配
3. **保留原字段**：替换时保留 `**{编号}**`、`{名称}`、`{优先级}`，只改 checkbox 状态 + 追加行尾
4. **清理失败独立**（见覆盖 1）：主断言通过时仍写 `✅ AI`，清理警告只落报告"待清理数据"章节
5. **回写失败处理**：清单段缺失或编号匹配不到 → 记录警告到执行报告「回写异常」章节，**不中断**套件执行

### 真源约定

- `report/execution_report_*.md` 为**审计真源**（承载 HTTP 响应体/断言明细/清理状态）
- 套件「执行清单」段为**进度真源**
- 两者冲突时 `/test-report` 汇总以 `execution_report` 为准
- **禁止**在清单段行里写超过一行的详情
- 人工先改过的行（无 AI 标记），AI 执行时**保留人工备注**并追加 AI 标记于其后
