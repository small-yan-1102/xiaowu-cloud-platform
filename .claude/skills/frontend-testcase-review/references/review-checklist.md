# 前端测试用例审阅检查清单

本文件定义了前端审阅测试用例的多维度检查清单。

## 一、功能覆盖度检查

### 1.1 核心功能覆盖
- [ ] 所有用户故事/功能点都有对应测试用例
- [ ] 核心业务流程有完整测试用例
- [ ] 增删改查操作都有测试覆盖
- [ ] 数据查询和展示有测试覆盖

### 1.2 功能优先级匹配
- [ ] P0 功能有 P0 测试用例
- [ ] 高优先级功能优先覆盖
- [ ] 核心功能测试用例优先级匹配

### 1.3 验收标准覆盖
- [ ] 用户故事的验收标准都有验证用例
- [ ] 业务规则都有测试验证
- [ ] 边界条件都有测试覆盖

---

## 二、交互场景覆盖度检查

### 2.1 正常路径（Happy Path）
- [ ] 用户完成核心任务的主流程
- [ ] 标准操作序列有测试用例
- [ ] 预期结果验证完整

### 2.2 异常路径（Exception Path）
- [ ] 用户操作中断场景（返回、取消、刷新）
- [ ] 网络异常场景（超时、断网、500错误）
- [ ] 数据异常场景（空数据、错误数据）
- [ ] 权限异常场景（未登录、无权限、登录过期）

### 2.3 表单校验场景
- [ ] 必填字段验证
- [ ] 格式验证（邮箱、手机号、身份证）
- [ ] 长度验证（最小、最大长度）
- [ ] 唯一性验证（重复提交、重复数据）
- [ ] 特殊字符验证（表情、HTML标签、SQL注入）
- [ ] 跨字段验证（密码确认、日期范围）

### 2.4 按钮状态测试
- [ ] 禁用状态（条件不满足时）
- [ ] 加载状态（提交中）
- [ ] 成功状态（操作成功）
- [ ] 失败状态（操作失败）
- [ ] 重复点击防护

### 2.5 页面跳转逻辑
- [ ] 正常跳转流程
- [ ] 参数传递验证
- [ ] 权限拦截验证
- [ ] 返回逻辑验证

## 六、覆盖度计算

### 6.1 计算公式

```
功能覆盖度 = (已覆盖功能点数 / 总功能点数) × 100%
交互覆盖度 = (已覆盖交互场景数 / 总交互场景数) × 100%
UI覆盖度 = (已覆盖UI状态数 / 总UI状态数) × 100%
前端专属覆盖度 = (已覆盖前端测试点数 / 总前端测试点数) × 100%
```

### 6.2 覆盖度等级

| 覆盖度 | 等级 | 处理建议 |
|--------|------|---------|
| ≥ 90% | ✅ 优秀 | 可以进入测试执行 |
| 80-89% | ⚠️ 良好 | 建议补充后进入测试 |
| 70-79% | ⚠️ 一般 | 必须补充核心功能测试 |
| < 70% | ❌ 不足 | 不建议进入测试执行 |

## 七、问题分级标准

### Critical（必须补充）
- 核心功能未覆盖
- 高优先级场景缺失
- 主流程测试用例缺失
- 验收标准未验证

### Warning（建议补充）
- 交互场景不完整
- UI状态缺失
- 异常路径未覆盖
- 表单校验不完整

### Info（可选补充）
- 前端专属测试点缺失
- 性能测试缺失
- 兼容性测试缺失
- 优化建议

---

## 使用建议

### 常规审阅
- 全量检查所有维度

### 快速审阅
- 重点关注"功能覆盖度"和"交互场景覆盖度"

### 质量专项
- 重点关注"UI状态覆盖度"和"前端专属测试点"

### 准入检查
- 核心功能覆盖度 ≥ 90%
- 无 Critical 未覆盖场景
- Warning 未覆盖场景 ≤ 5 个
