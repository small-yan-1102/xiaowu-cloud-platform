# 前端实现模块分析

> **设计模式**：Tool Wrapper（按需加载项目组件/方法知识）+ Generator（模板化输出分析报告）

## 技能说明

本技能用于根据产品 PRD 或原型文档，分析并输出前端实现方案，包括：
1. **组件选型**：匹配项目现有组件库，给出组件选择建议
2. **公共方法**：匹配项目 hooks/utils，给出方法使用建议
3. **界面布局**：基于项目布局规范，给出页面结构方案

## 输入/输出契约

### 输入
| 输入项 | 来源 | 缺失时处理 |
|-------|------|----------|
| PRD/原型文档 | 用户提供 | Fatal：向用户索取 |
| 项目组件库 | `src/components/` 目录 | Fatal：无法分析 |
| 项目 Hooks/Utils | `src/hooks/`、`src/utils/` 目录 | Warn：跳过方法推荐 |
| 布局规范 | `references/layout-standard.md` | Warn：使用通用布局建议 |

### 输出
| 输出产物 | 格式 |
|---------|------|
| 前端实现分析报告 | Markdown |

## 核心工作流

```
Task Progress:
- [ ] Phase 1: 需求解析
- [ ] Phase 2: 项目资源扫描（Tool Wrapper 加载）
- [ ] Phase 3: 组件匹配
- [ ] Phase 4: 方法匹配
- [ ] Phase 5: 布局方案输出
- [ ] Phase 6: 生成分析报告（Generator）
```

### Phase 1: 需求解析

1. 读取 PRD/原型：获取功能描述、页面结构、交互流程
2. 提取页面清单：识别涉及的页面/组件
3. 提取功能模块：识别核心功能点和交互

### Phase 2: 项目资源扫描（Tool Wrapper 模式）

**按需加载项目知识**——只在需要时扫描相关目录：

1. **扫描组件库**：
   - `src/components/common/` — 通用组件
   - `src/components/business/` — 业务组件
   - 其他组件目录（按项目实际结构）

2. **扫描 Hooks/Utils**：
   - `src/hooks/` — 组合式函数
   - `src/utils/` — 工具函数

3. **识别项目技术栈**：通过 `package.json` 确定框架、UI 库、工具链

### Phase 3: 组件匹配

根据功能需求，匹配项目已有组件：

| 匹配优先级 | 说明 |
|-----------|------|
| 1. 项目公共组件 | `src/components/common/` 中的封装组件 |
| 2. 项目业务组件 | `src/components/business/` 中的业务组件 |
| 3. UI 库组件 | 项目已引入的 UI 库提供的组件 |
| 4. 需新建 | 以上都无匹配，标记为需新建 ⚠️ |

**对每个功能模块输出**：

| 功能模块 | 推荐组件 | 来源 | 使用方式 | 备注 |
|---------|---------|------|---------|------|
| {功能} | {组件名} | {路径} ✅ | {使用方式} | |
| {功能} | {组件名} | ⚠️ 需新建 | {设计方案} | |

### Phase 4: 方法匹配

根据功能需求，匹配项目已有的 hooks 和工具方法：

| 功能场景 | 推荐方法 | 来源 | 使用示例 |
|---------|---------|------|---------|
| {场景} | {方法名} | {路径} | {示例} |

### Phase 5: 布局方案输出

基于项目布局规范（如有），输出页面结构方案：
- 页面容器结构
- 区域划分（搜索区、操作区、内容区、分页区）
- 弹窗/抽屉的组织方式

### Phase 6: 生成分析报告（Generator 模式）

按以下模板生成报告：

```markdown
# [页面名称] 前端实现分析报告

## 一、页面结构

[页面结构图，使用 Mermaid 或文本树形结构]

## 二、组件选型

| 功能模块 | 推荐组件 | 来源 | 使用方式 | 备注 |
|---------|---------|------|---------|------|

## 三、公共方法

| 功能场景 | 推荐方法 | 来源 | 使用示例 | 备注 |
|---------|---------|------|---------|------|

## 四、交互流程

[核心交互流程图或步骤说明]

## 五、实现建议

[特殊处理点、注意事项、优化建议]
```

## 约束

- 组件推荐必须基于项目实际存在的组件，不推荐假设存在的组件
- 标记为"需新建"的组件必须提供设计方案
- 分析报告聚焦"用什么"和"怎么用"，不涉及具体代码实现
