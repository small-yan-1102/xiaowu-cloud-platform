# test-execution 项目定制

## 覆盖 1：冒烟模式 Flaky 规则

**HE 原文位置**：Phase 2-S → 步骤 4（判定单条结论）→ 重试上限说明
**HE 原文摘要**：每条用例最多重试 1 次，不做 Flaky 判定，冒烟结论只有通过/失败两态
**定制为**：在重试上限说明后追加 Flaky 与冒烟关系的明确定义：
- 冒烟模式中即使重试后通过，也**不**记录为 Flaky（冒烟关注阻断性问题而非稳定性）
- 仅当用例在标准执行模式中出现重试才适用 Flaky 升级规则
- 冒烟中的失败**始终计入连续失败次数**，不因 Flaky 而豁免

## 覆盖 2：执行后回写套件「执行清单」段

**HE 原文位置**：Phase 2 → 单条用例执行后记录执行结果到报告
**HE 原文摘要**：执行结果仅写入执行报告文件，套件 md 文件本身保持只读
**定制为**：单条用例执行完成（含 teardown）后，除写执行报告外，**必须**用 Edit 工具更新**当前迭代**对应套件 md 文件（`iterations/**/testcase/suite_*.md`）中 `## 执行清单（状态记录入口）` 段内该用例所在行——勾选 checkbox 并在行尾追加 AI 执行标记 + 报告链接。

> ⚠️ 本覆盖项替代旧版"回写执行顺序表最右列"逻辑。执行顺序表**不再**含「执行记录」列，状态唯一入口是「执行清单」段。

### 写入格式

**匹配目标行**（默认状态）：
```
- [ ] **{用例编号}** {用例名称}（{优先级}）
```

**回写后**：
```
- [x] **{用例编号}** {用例名称}（{优先级}） · {状态 emoji + "AI" 标识} · {执行日期} · [报告](相对路径)
```

### 状态映射

| 执行结论 | 回写结果 |
|---------|--------|
| L1 全部通过 | `- [x] **{编号}** {名称}（{优先级}） · ✅ AI {日期} · [报告]({execution_report}#{锚点})` |
| 任一 L1 失败（有 BUG-id） | `- [ ] **{编号}** {名称}（{优先级}） · ❌ AI {日期} · [失败详情]({execution_report}#{锚点}) · BUG-{id}` |
| 任一 L1 失败（无 BUG-id） | `- [ ] **{编号}** {名称}（{优先级}） · ❌ AI {日期} · [失败详情]({execution_report}#{锚点})` |
| 前置/登录/数据/环境异常 | `- [ ] **{编号}** {名称}（{优先级}） · 🚫 AI · {阻塞原因}` |
| 用例标注"暂缓/本轮不跑" | 不回写（暂缓用例不出现在清单段） |
| 冒烟模式重试后通过 | `- [x] **{编号}** {名称}（{优先级}） · ✅ AI {日期} · [报告]({path})`（不记 Flaky，见覆盖 1） |

**字段说明**：
- `{编号}`、`{名称}`、`{优先级}` 来自清单段原行，**必须原样保留**
- `{日期}` 格式 `YYYY-MM-DD`，取执行当天
- `{execution_report}` 相对路径如 `execution/execution_report_20260417.md`
- `{锚点}` 用例编号小写（如 `smoke-001`、`overdue-s01-003`）
- 阻塞无执行日期时可省略日期字段

### 回写时机与失败处理

1. **回写时机**：单条用例 teardown 后**立即**回写；不等整个套件跑完再批量更新
2. **匹配规则**：用 Grep 在清单段内按 `**{用例编号}**` 精确匹配（加粗用例编号），定位到整行后整行替换
3. **保留原字段**：替换时保留 `**{编号}**`、`{名称}`、`{优先级}`，只改 checkbox 状态 + 追加行尾信息
4. **回写失败处理**：清单段缺失或编号匹配不到 → 记录警告到执行报告「回写异常」章节，**不中断**当前套件执行
5. **冒烟模式**：同样回写，结论简化映射为 `✅ AI` / `❌ AI`

### 真源约定

- `execution/execution_report_*.md` 为**审计真源**（承载失败详情/截图/步骤明细）
- 套件「执行清单」段为**进度真源**（人工 + AI 共享的状态记录入口）
- 两者冲突时 `/test-report` 汇总以 `execution_report` 为准
- **禁止**在清单段行里写超过一行的详情；超出内容一律放 `execution_report`
- 人工先改过的行（无 AI 标记），AI 执行时**保留人工备注**并追加 AI 标记于其后（避免覆盖人工判断）
