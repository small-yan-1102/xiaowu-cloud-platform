# 执行报告模板库

> 本文件是 test-execution skill 的报告输出模板，由 SKILL.md 在 Phase 3 输出报告时加载。
> **替换此文件即可改变报告格式，无需修改 SKILL.md 中的指令逻辑。**

---

## 模板一：单条用例执行结果

每条用例执行完成后，按此模板记录结果。

```markdown
### [用例编号] [用例名称]

**执行结果**：通过 / 失败 / 阻塞 / 跳过
**执行时间**：YYYY-MM-DD HH:mm:ss
**执行耗时**：X 分 X 秒
**执行环境**：{环境URL}

**步骤执行明细**：

| 步骤 | 操作 | 预期结果 | 实际结果 | 验证级别 | 状态 | 截图 |
|------|------|---------|---------|---------|------|------|
| 1 | 操作描述 | 预期描述 | 实际看到的内容 | L1 | 通过/失败 | CP1_XXX_step01_pass_{时间戳}.png |
| 2 | 操作描述 | 预期描述 | 实际看到的内容 | L1 | 通过/失败 | - |
| 3 | 操作描述 | 预期描述 | 实际看到的内容 | L2 | 通过/警告 | - |

**失败详情**（仅失败时填写）：
- 失败步骤：步骤X
- 期望：{预期结果描述}
- 实际：{实际看到的内容}
- 截图：{失败前状态截图}, {失败时截图}（至少 2 张）

**L2 警告**（仅有软验证不通过时填写）：
- [步骤X] 预期：XXX，实际：YYY（不影响用例判定）

**备注**：{重试情况、环境异常、人工介入说明等}
```

## 模板四：证据索引文件

每日截图目录（`test_reports/screenshots/YYYY-MM-DD/`）下自动生成。

```markdown
# 证据索引 - YYYY-MM-DD

**关联报告**：test_report_YYYYMMDD_HHmmss.md
**截图总数**：N 张（CP1: X, CP2: Y, CP3: Z）

| 文件名 | 检查点 | 用例编号 | 步骤 | 状态 | 时间 |
|--------|-------|---------|------|------|------|
| CP1_JQB-TEAM-001_init_start_{时间戳}.png | CP1 | JQB-TEAM-001 | init | start | HH:mm:ss |
| CP1_JQB-TEAM-001_final_pass_{时间戳}.png | CP1 | JQB-TEAM-001 | final | pass | HH:mm:ss |
```

---

## 执行结果状态定义

| 状态 | 含义 | 触发条件 |
|------|------|---------|
| **通过** | 所有 L1 验证通过 | 全部 L1 强验证通过（L2 警告不影响） |
| **失败** | 存在 L1 验证不通过 | 任一 L1 强验证未通过 |
| **阻塞** | 无法执行 | 环境问题、前置条件不满足、依赖用例失败 |
| **跳过** | 主动跳过 | 依赖的前序用例失败导致跳过 |
