      - 第三轮：建立依赖关系                 | 遵循规则 2.2 模板
      - 第四轮：跨系统用例                   |
              |                            |
              |  +-------------------------+
              |  |
              v  v
      Phase 4：质量审查 ---- 对照规则 第8节 审查清单

## 规则内部结构（附录索引）

生成过程中，具体的编写规范通过附录查阅：

| 附录 | 内容 | 什么时候查 |
|------|------|----------|
| **A** | 用例字段强制规范（10个必填/选填字段） | 确认每条用例的字段完整性 |
| **B** | 人工介入标识规范（7类场景） | 遇到验证码、人脸识别等场景时 |
| **C** | 预期结果分级（L1/L2/L3） | 编写每条预期结果时标注级别 |
| **D** | 预期结果反模式（10个禁用词） | 审查预期结果是否模糊 |
| **E** | 前置条件与数据准备（5种类型 + 4种数据策略） | 编写前置条件时 |
| **F** | 步骤编写规范（动作词 + 元素描述 + 动态内容 + 条件分支） | 编写测试步骤时 |
| **G** | 跨系统测试与用例组织（系统注册表 + 依赖关系 + 套件组织） | 涉及多系统或组织套件时 |

## 职责分工

| 维度 | skill 负责 | 规则负责 |
|------|-----------|---------|
| 设计方法（等价类/边界值/场景法/CRUD） | skill 第一章 | - |
| 生成顺序（模块排序/场景排序/依赖建立） | skill Phase 3 | - |
| 用例模板格式 | 引用规则 | 规则 2.2（唯一定义） |
| 质量标准（原则/覆盖要求/验证规范） | - | 规则 1-8 |
| 编写细则（反模式/动作词/元素描述） | - | 规则附录 D-G |
| 审查清单 | skill Phase 4 引用 | 规则 8（唯一定义） |
| 输出组装（文档结构/统计摘要） | skill Phase 5 | - |

## 关键设计决策

1. **模板唯一定义在规则中** -- skill 不重复定义模板，通过引用规则 2.2 确保一致性
2. **独立性优先，依赖显式声明** -- 用例优先独立执行，无法独立时通过 `**依赖用例**` 字段显式声明（规则 1.2 + 附录 G.2）
3. **生成顺序在 skill 中控制** -- 规则管每条用例的质量，skill 管整批用例的生产流程
4. **AI 直接执行适配** -- 所有规则面向 AI 直接执行模式设计，预期结果必须具体可判断，前置条件必须可自主达成

## 两种模式说明

### Mode A: 标准模式（默认）

- **输入**：PRD / 流程图 / 原型截图
- **输出**：完整测试用例套件
- **适用**：初次用例设计、功能迭代
- **触发**：用户请求"生成测试用例"且无审阅报告引用

### Mode B: 技术文档补充模式（v2.1新增）

- **输入**：技术审阅报告 + 正式技术文档（可指定目标模块 scope）
- **输出**：补充测试用例（命名带 S 前缀，多模块时按模块分文件）
- **适用**：技术文档审阅通过后，补充AI标准不覆盖的风险场景
- **触发**：用户请求含"依据技术文档补充"或引用审阅报告

### 模式选择

| 用户请求特征 | 选择模式 |
|------------|---------|
| "生成测试用例" + 无审阅报告引用 | Mode A |
| "依据技术文档补充" / 引用审阅报告 | Mode B |
| 首次设计某功能的用例 | Mode A |
| 审阅通过后补充风险场景用例 | Mode B |
