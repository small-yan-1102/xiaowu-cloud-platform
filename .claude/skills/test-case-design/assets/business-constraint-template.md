## 七、参考样例（2-3 条历史优质用例）

> 挑选团队过去编写的高质量用例，让 AI 学习你们的用例风格、粒度和描述习惯。

**选取标准（满足越多越好）**：

| 标准 | 说明 |
|------|------|
| 曾被实际执行且通过 | 证明步骤可操作、预期结果可判断 |
| 预期结果具体到 UI 元素/文案 | 如"页面顶部出现绿色 toast 提示{包含:保存成功}"，而非"系统提示成功" |
| 测试数据真实有意义 | 字段值符合业务格式，非 "aaa" / "123" 等占位符 |
| 步骤粒度适中 | 每步对应一个操作，不过细（逐字段描述）也不过粗（"完成整个表单"） |
| 覆盖了异常或边界场景 | 异常用例比正向用例更能体现团队的用例风格和严谨程度 |

**避免选取**：
- 已过期（对应功能已变更）的用例
- 执行时曾因步骤描述不清而卡住的用例
- 预期结果含有"正常"、"成功"等模糊词的用例
