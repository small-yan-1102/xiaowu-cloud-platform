## 模板六：跨模块 E2E 用例

> 适用于同一产品内，模块 A 的操作产生数据、模块 B 消费/展示该数据的端到端验证场景。
> 与模板二（跨系统）的区别：同一登录态可访问两个模块，无需切换系统账号。

```markdown
### PRJ-E2E-[序号] [用例名称]

**优先级**：P1 / P2
**用例类型**：E2E测试
**执行环境**：[ENV: TEST] / [ENV: BOTH]
**数据流**：FLOW-{编号}（{源模块} → {目标模块}）
**关联需求**：PRD V{X.X} 第X.X节
**依赖用例**：无 / PRJ-XXX-NNN（需先执行且通过）

**前置条件**：
1. [登录态] 使用账号 {账号}/{密码} 登录（URL: {登录页URL}），确认看到{成功标志}
2. [页面定位] 导航到「{源模块}」页面（URL: {页面路径}）
3. [数据存在][数据准备: UI自建/API] {源模块所需前置数据描述}
   [验证: 「{页面}」中可见{前置数据特征}]

**测试数据**：
| 字段 | 值 | 说明 |
|------|-----|------|
| 字段1 | 值1 | 源模块写入用 |

**测试步骤**：

*— 阶段一：{源模块}写入操作 —*
1. 导航到「{源模块}」中的「{页面}」
2. {标准动作词}「{元素}」完成数据写入
3. 记录写入结果中的关键标识字段（如 ID、名称）

*— 阶段二：{目标模块}消费/展示验证 —*
4. 导航到「{目标模块}」中的「{页面}」（URL: {页面路径}）
5. 定位并验证来自{源模块}的数据

**预期结果**：

*阶段一（源模块写入）*：
1. [断言: 视觉] {源模块}「{页面}」显示操作成功标志"{成功文案}"，记录 {字段名}=`{记录为:变量名}`（L1）
2. [断言: API] GET /api/{source-endpoint} 返回 {字段} = {预期值}（L1）

*阶段二（目标模块消费）*：
3. [断言: 视觉] {目标模块}「{页面}」中显示来自源模块的数据，{字段名}=`{引用:变量名}`（L1）
4. [断言: 视觉] 数据状态/格式符合{目标模块}的展示规则：{具体描述}（L1）
5. [断言: API] GET /api/{target-endpoint} 返回 {字段} = `{引用:变量名}`（L2，可选）

**清理步骤**（用例执行完毕后执行）：
1. [清理] 在{源模块}中删除本用例创建的数据（如可删除）
2. [清理] 验证{目标模块}中对应记录已同步消失或状态已复原（L1）

**是否需要人工介入**：否
**备注**：关联数据流 FLOW-{编号}；{目标模块}展示结果依赖{源模块}异步同步时，使用 `等待` 步骤替代固定延时
```

## 动态值标记语法速查

| 标记 | 含义 |
|------|------|
| `{时间戳}` | yyyyMMddHHmmss，确保字段唯一性 |
| `{当前日期}` | YYYY-MM-DD |
| `{当前时间}` | HH:mm，±1 分钟容差 |
| `{非空}` | 值不为空即可，不关心具体内容 |
| `{格式:PATTERN}` | 值需符合指定正则格式 |
| `{包含:关键词}` | 值包含指定关键词即可 |
| `{与测试数据.字段名一致}` | 值与测试数据表中对应字段相同 |
| `{记录为:变量名}` | 记录该值，供后续步骤通过 `{引用:变量名}` 使用 |
| `{引用:变量名}` | 引用前序步骤中用 `{记录为:}` 记录的值 |
