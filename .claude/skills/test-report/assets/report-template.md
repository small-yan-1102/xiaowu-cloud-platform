## 六、后续建议

> 根据质量结论给出推荐操作：

**质量结论为 🟢 时**：
```
→ 可执行 release-gate 发布门禁检查
```

**质量结论为 🟡 时**：
```
→ 建议先确认开放 Critical 缺陷的影响范围
→ 与产品/开发确认是否可带缺陷发布
→ 如确认可发布，执行 release-gate；如需修复，走修复流程后重新生成测试报告
```

**质量结论为 🔴 时**：
```
→ 修复高风险项（P0 失败 / Fatal 缺陷）
→ 重新执行相关测试
→ 重新执行 bug-sync 更新缺陷状态
→ 重新生成测试报告（test-report）
```

## 质量结论判定速查

| 结论 | P0 | Fatal（开放）| Critical（开放）| P0+P1 通过率 |
|------|----|-------------|----------------|-------------|
| 🟢 质量良好 | 全部通过 | 0 | 0 | ≥ 90% |
| 🟡 基本达标 | 全部通过 | 0 | > 0 或 | 80%~89% |
| 🔴 未达标 | 有失败 | > 0 | — | < 80% |

> 🟢 条件须全部满足；🔴 任一条件触发即判定；🟡 介于两者之间。
