# 执行报告格式规范

> 本文件是 test-result-sync skill 的执行报告格式标准，由 SKILL.md 在 Phase 1 解析报告和 Phase 3 输出统计时加载。
> **替换此文件即可改变报告格式，无需修改 SKILL.md 中的指令逻辑。**

---

## 一、AI 执行报告输入格式

AI 执行测试后产出的结构化执行报告格式（Markdown）：

```markdown
## 执行报告

| 用例编号 | 用例名称 | 结果 | 耗时 | 备注 |
|---------|---------|------|------|------|
| PRJ-CERT-001 | 登录后完成企业认证 | 通过 | 1m30s | - |
| PRJ-CERT-002 | 无效信用代码提交 | 失败 | 0m45s | 期望错误提示未出现 |
| SMOKE-CERT-001 | 认证全流程冒烟 | 通过 | 1m50s | - |
| PRJ-CERT-003 | 编辑认证信息 | 跳过 | - | 前置用例 PRJ-CERT-001 失败 |

### 失败详情

#### PRJ-CERT-002

- **失败步骤**：步骤4 - 验证错误提示
- **期望结果**：页面显示"信用代码格式错误"
- **实际结果**：页面无任何提示，表单提交成功
- **截图**：[附件]
```

## 二、Markdown 用例文档回填格式

回填后，用例标题行格式为：

```
### PRJ-XXX-001 用例名称 【PASS 2026-04-02】
### PRJ-XXX-002 用例名称 【FAIL 2026-04-02】
### PRJ-XXX-003 用例名称 【SKIP 2026-04-02】
```

---

## 三、统计报告输出格式

```markdown
=== 测试执行报告 ===
执行时间：{YYYY-MM-DD HH:mm:ss}
数据来源：AI 直接执行
执行环境：{测试环境 URL}

总用例数：{N}
通过：{N} ({通过率}%)
失败：{N} ({失败率}%)
跳过：{N} ({跳过率}%)

失败用例：
- {PRJ-XXX-002}：{失败原因摘要}
- {PRJ-XXX-005}：{失败原因摘要}

跳过用例：
- {PRJ-XXX-003}：{跳过原因，如"前置用例 PRJ-XXX-001 失败"}
```

## 四、Excel 回填规范

Excel 文件结构约定：

| 列 | 字段 | 说明 |
|----|------|------|
| A | 用例编号 | 格式 `PRJ-XXX-NNN`，用于匹配 |
| B | 用例名称 | 仅读取，不修改 |
| 结果列（默认G列） | 执行结果 | 回填"通过"/"失败"/"跳过" |

> **用例ID列自动识别规则**：扫描第1行，找到包含"用例"且包含"ID"（不区分大小写）的列头即为用例ID列。

---

## 五、结果状态定义

| 状态值 | 含义 | 回填文本 | 标记符号 |
|--------|------|---------|---------|
| 通过 | 用例执行，所有断言通过 | 通过 | `PASS` |
| 失败 | 用例执行，存在断言失败 | 失败 | `FAIL` |
| 跳过 | 用例未执行（前置失败/不满足条件） | 跳过 | `SKIP` |
| 阻塞 | 用例因数据准备失败无法执行 | 阻塞 | `BLOCKED` |
