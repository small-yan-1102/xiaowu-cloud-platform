## 规范分级与加载策略

| 层级 | 适用场景 | 加载优先级 |
|-----|---------|-----------|
| **L1 核心** | 全程必载 | P0（强制） |
| **L2 代码** | 编写业务代码时 | P1 |
| **L3 数据** | 涉及数据库/缓存操作时 | P1 |
| **L4 架构** | 领域建模/适配层/Spec 设计时 | P1 |

各阶段加载组合：

```
Phase 2-3 (Spec 设计)    → L1 + L4
Phase 4.1 (领域层)       → L1 + L2
Phase 4.2 (Client 层)    → L1 + L2 + L3
Phase 4.3 (用例层)       → L1 + L2
Phase 4.4 (适配层)       → L1 + L2 + L3 + L4
```

## 通用编码规范检查（L1 核心）

- [ ] **类命名符合规范**（聚合根、Cmd、Query、Controller 等）
- [ ] **包路径正确**（`com.xw.[模块].[层级].[功能]`）
- [ ] **仅使用 WarnException 和 ErrorException**（禁止 RuntimeException/NullPointerException）
- [ ] **布尔变量无 is 前缀**
- [ ] **日志级别使用恰当**
- [ ] **方法命名符合规范**（动词开头，驼峰命名）
- [ ] **常量命名符合规范**（全大写，下划线分隔）
- [ ] **无冗余代码**（无未使用的导入、变量、方法）
- [ ] **无 TODO 占位**（禁止只加 `// todo` 而不实现实际逻辑）
- [ ] **方法返回值使用 POJO**（禁止使用 `Map<String, Object>`）
- [ ] **字符串判空使用 StrUtil.isBlank()/isNotBlank()**
- [ ] **对象判等使用 Objects.equals()**
- [ ] **集合判空使用 CollectionUtils.isEmpty()/isNotEmpty()**
- [ ] **字符串拼接使用 String.format()**
- [ ] **禁止使用 JdbcTemplate**（统一使用 JPA 或 MyBatis）
- [ ] **JSON 解析使用 POJO**（禁止使用 JSONObject）

### 代码深度控制（阿里规约）

- [ ] **方法嵌套层级不超过 3 层**
- [ ] **单个方法圈复杂度不超过 10**
- [ ] **方法体长度不超过 80 行**
- [ ] **类继承层级不超过 3 层**
- [ ] **禁止直接使用 `list.get(0)` 或 `array[0]`**（必须先校验长度）

## 业务逻辑完整性检查（L2 代码）— 关键

**以下情况直接审计不通过**：

```java
// ❌ 方法只有签名，无实现体
public void payOrder(String orderId) {
    // todo: 实现支付逻辑
}

// ❌ 方法体只有 TODO 注释
public Order createOrder(CreateOrderCmd cmd) {
    // TODO: 待实现
    return null;
}

// ❌ 关键业务逻辑留空
public void validateOrder(Order order) {
    // 待补充验证逻辑
}
```

- [ ] **所有方法有完整实现**（无空方法体、无纯 TODO 注释）
- [ ] **单一职责原则**（一个业务逻辑对应一个 private 方法）
- [ ] **ID 生成逻辑封装为独立方法**
- [ ] **异常处理规范**（参数校验、数据不存在、外部服务超时均已处理）
- [ ] **枚举判断逻辑封装在枚举类中**
- [ ] **锁的使用规范**（标准分布式锁使用范式）

### 日志规范

- [ ] **方法开始有入口日志**
- [ ] **方法结束有出口日志（成功/失败）**
- [ ] **敏感信息脱敏**（手机号、身份证等）
- [ ] **日志内容包含关键业务参数**

---

## 数据层检查（L3 数据）

- [ ] **数据库字段类型规范**（JSON 内容使用 JSON 类型）
- [ ] **索引命名规范**（`uk_`、`idx_`、`ft_` 前缀）
- [ ] **避免 SELECT ***
- [ ] **分页查询必须使用数据库分页**（禁止内存分页）
- [ ] **大批量数据分批处理**
- [ ] **无循环内数据库查询（N+1 问题）**
- [ ] **无循环内远程服务调用**
- [ ] **缓存使用规范**（优先从缓存获取、更新时同步失效）

---

## 架构层检查（L4 架构）

- [ ] **API 路径规范**（`/appApi/` 或 `/cloudApi/`，类命名后缀正确）
- [ ] **`/appApi/` 路径必须添加 `@NeedLogin` 注解**
- [ ] **越权问题防范**（数据归属校验，用户只能访问自己的数据）
- [ ] **公共组件优先使用**（MapperUtil、LockerService、RetryUtil）
- [ ] **SQL 注入防护**（使用参数化查询）

---

## 领域层专项检查

- [ ] **聚合根使用静态工厂方法创建**（禁止直接 `new`）
- [ ] **业务行为封装在聚合根内**（不在 Service 中直接操作字段）
- [ ] **领域事件通过 `publisher().attachEvent()` 发布**
- [ ] **实体间关系注解正确**（`@OneToMany`、`@ManyToOne` 等）
- [ ] **领域方法命名符合业务语言**（不使用技术词汇如 `update`/`set`）
- [ ] **值对象不可变**（所有字段 final）

---

## 适配层专项检查

- [ ] **Controller 路径前缀正确**（`/cloudApi/` 或 `/appApi/`）
- [ ] **DTO/VO 转换完整**（无字段遗漏）
- [ ] **Repository 实现方法体完整**（禁止只有 TODO）
- [ ] **Client 实现方法体完整**（禁止只有 TODO）
- [ ] **事件监听器注册正确**
- [ ] **Controller 方法命名符合 RESTful 规范**

---

## 单元测试强制检查（⚠️ 必须）

**每个主类必须对应一个测试类，禁止只生成主代码而不生成测试。**

- [ ] **测试类已生成**（每个主类必须有对应测试类 `XxxTest`）
- [ ] **测试方法命名清晰**（`should_xxx_when_xxx` 格式）
- [ ] **使用 AAA 模式**（Arrange-Act-Assert）
- [ ] **Mock 依赖正确注入**
- [ ] **覆盖正常流程和异常流程**
- [ ] **断言完整且有意义**（不只是 `assertNotNull`）

```

✅ L1 核心规范：通过（命名/包路径/异常/工具使用）

✅ L3 数据规范：通过/不适用（无数据库操作）
✅ L4 架构规范：通过/不适用（非适配层）
✅ 业务逻辑完整性：所有方法有完整实现，无 TODO 占位
✅ 单元测试：XxxTest 已生成，覆盖正常/异常流程

⚠️ 改进建议：
1. [建议1]
2. [建议2]
```

**审计不通过时**，必须明确列出问题项，禁止在问题未解决时进入下一个类的生成：

```
【AI 审计结果】❌ 不通过

发现以下问题：
1. [文件名].[方法名]：[具体问题描述]
   要求：[修复要求]

请修复以上问题后重新审计。
```
