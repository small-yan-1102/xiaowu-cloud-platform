# 缺陷报告模板库

> 本文件是 bug-sync skill 的输出模板，由 SKILL.md 在 Phase 3 输出缺陷清单时加载。
> **替换此文件即可改变缺陷记录格式，无需修改 SKILL.md 中的指令逻辑。**

---

## 模板一：单条缺陷记录（Markdown）

```markdown
### {缺陷编号} {缺陷标题}

| 字段 | 内容 |
|------|------|
| **缺陷编号** | BUG-{MODULE}-{SEQ} |
| **关联测试用例** | {PRJ-XXX-NNN 或 API-XXX-NNN} |
| **所属模块** | {模块中文名} |
| **严重等级** | 致命 / 严重 / 一般 / 轻微 |
| **缺陷优先级** | 高 / 中 / 低 |
| **缺陷类型** | 功能缺陷 / 环境阻塞 / 性能问题 |
| **复现环境** | {Base URL}（执行时间：{YYYY-MM-DD HH:mm}）|
| **发现人** | AI 自动执行 |

**复现步骤**：

1. {前置条件：账号/角色/数据状态}
2. {操作步骤一}
3. {操作步骤二}
4. ...

**期望结果**：

{从用例 L1 预期结果提取，描述应该发生什么}

**实际结果**：

{执行时的实际观测值，含状态码/响应字段/页面状态等}

**附件**：
- 截图：{截图文件路径 或 无}
- 请求/响应摘要（接口用例）：
  ```
  {METHOD} {URL}
  响应：HTTP {状态码} — {关键响应字段}
  ```

**备注**：{如有导致其他用例跳过，注明：「本缺陷导致 N 条依赖用例跳过：{用例编号列表}」}
```

## CSV 导入格式

文件命名：`bug-import-YYYYMMDD_HHmmss.csv`
存储位置：`bug_reports/`

### 字段说明

| CSV 列名 | 对应缺陷字段 | 说明 |
|---------|-----------|------|
| `title` | 缺陷标题 | `[{模块名}] {用例名称}` |
| `severity` | 严重等级 | Fatal / Critical / Major / Minor |
| `priority` | 缺陷优先级 | High / Medium / Low |
| `module` | 所属模块 | 模块英文标识 |
| `steps` | 复现步骤 | 换行用 `\n` 分隔 |
| `expected` | 期望结果 | |
| `actual` | 实际结果 | |
| `environment` | 复现环境 | Base URL |
| `linked_case` | 关联测试用例 | 用例编号 |
| `attachments` | 附件路径 | 截图文件路径，多个用 `;` 分隔 |
| `type` | 缺陷类型 | functional / env-blocked / performance |

### CSV 样例

```csv
title,severity,priority,module,steps,expected,actual,environment,linked_case,attachments,type
"[订单] 创建订单-必填项缺失时应返回400",Critical,High,order,"1. 使用合法 Token\n2. 发送 POST /api/orders，Body 中省略 amount 字段","HTTP 400，message 包含 '参数不能为空'","HTTP 200，订单被异常创建",https://api.example.com,API-ORDER-005,,functional
```

## 严重等级与优先级定义

| 严重等级 | 说明 | 对应用例优先级 |
|---------|------|-------------|
| **致命（Fatal）** | 核心功能完全不可用，阻塞主业务流程 | P0 |
| **严重（Critical）** | 主流程功能缺陷，有明显错误或安全漏洞 | P1 |
| **一般（Major）** | 功能部分不正常，有合理工作绕路 | P2 |
| **轻微（Minor）** | 小问题，不影响主要功能使用 | P3 |
| **环境阻塞** | 因环境/数据原因无法复现，非产品缺陷 | — |
