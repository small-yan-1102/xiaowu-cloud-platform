# 跨 Skill 变量传递语法规范

> 本文件定义 test-case-design、api-test-case-design、test-execution、api-test-execution 等 Skill 共享的变量语法。

## 变量声明与引用

| 语法 | 含义 | 示例 |
|------|------|------|
| `{记录为:变量名}` | 在当前步骤记录一个值供后续引用 | 订单号为 `{记录为:订单号}` |
| `{引用:变量名}` | 引用前序步骤记录的值 | 显示订单号 `{引用:订单号}` |
| `{时间戳}` | 执行时动态生成的时间戳后缀 | 测试公司_{时间戳} |
| `{当前日期}` | 执行当天日期 YYYY-MM-DD | 创建时间显示 {当前日期} |
| `{非空}` | 值不为空即可 | 订单编号 {非空} |
| `{格式:PATTERN}` | 值需符合正则格式 | 工单号 {格式:WO-\d{6}} |
| `{包含:关键词}` | 值包含指定关键词 | 提示 {包含:成功} |

## 适用 Skill

- test-case-design（用例设计时声明变量）
- api-test-case-design（接口用例设计时声明变量）
- test-execution（浏览器执行时提取和引用变量）
- api-test-execution（接口执行时提取和引用变量）
- test-data-preparation（数据准备时引用变量映射表）

## 注意事项

- 变量名全局唯一，跨步骤、跨用例可见（同一执行会话内）
- `{时间戳}` 在每次执行开始时生成一次，整个会话内保持不变
- 嵌套引用不支持（如 `{引用:{记录为:x}}`）
