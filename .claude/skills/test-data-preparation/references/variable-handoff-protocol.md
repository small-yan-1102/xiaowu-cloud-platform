> **版本**: v1.0
> **创建日期**: 2026-04-13
> **适用范围**: test-data-preparation（生产侧）与 AI 测试执行（消费侧）之间的变量传递标准
> **引用方**: test-data-preparation/SKILL.md §变量传递机制；ai_execution_strategy.md §6.1 执行流程

## 一、变量映射表格式（生产侧标准）

test-data-preparation 在 Mode A Phase 5 输出的数据准备报告中，**必须**包含以下格式的变量映射表（§5）：

```markdown
## §5 变量映射表

> 供测试用例前置条件中 `{引用:变量名}` 使用。

| 用例编号 | 变量名 | 实际值 | 数据类型 | 备注 |
|---------|-------|-------|---------|------|
| TC-001  | company_id   | comp_20260413153021 | String | 企业记录唯一 ID |
| TC-001  | company_name | 测试公司_20260413153021 | String | 企业名称（含时间戳） |
| TC-002  | order_id     | ord_20260413153045  | String | 由 TC-001 的 company_id 关联创建 |
```

**字段说明**：

| 字段 | 必填 | 说明 |
|------|------|------|
| 用例编号 | 是 | 该变量服务于哪条测试用例 |
| 变量名 | 是 | 与测试用例前置条件中 `{引用:变量名}` 的名称精确匹配（大小写敏感） |
| 实际值 | 是 | 创建时分配的实际 ID 或数据值（不可为空） |
| 数据类型 | 是 | String / Number / Boolean / Array |
| 备注 | 否 | 说明数据来源或依赖关系 |

---

## 二、变量引用格式（消费侧标准）

测试用例前置条件中引用变量时，使用以下格式：

```
{引用:变量名}
```

**示例**：

```markdown
**前置条件**：
3. [数据存在][数据准备: API] 企业记录已创建，ID 为 {引用:company_id}
   [验证: GET /company/{引用:company_id} 返回 status=active]
```

## 三、AI 执行时的替换规则

AI 在执行测试用例前，**必须**执行以下步骤：

1. **定位数据准备报告**：在当前迭代的 `testcase/` 目录下查找最新的 `data-prep-report-*.md`
2. **读取 §5 变量映射表**：提取本轮执行涉及的所有用例对应的变量键值对
3. **替换前置条件**：将用例前置条件和测试步骤中所有 `{引用:变量名}` 替换为实际值
4. **替换失败处理**：
   - 找不到变量名 → **阻塞当前用例**，标注「变量 {变量名} 未在数据准备报告中找到」
   - 数据准备报告不存在 → **询问用户**是否先执行数据准备，或手动提供变量值

## 四、变量命名规范

| 规则 | 示例 | 说明 |
|------|------|------|
| 使用 snake_case | `company_id` | 不用驼峰或中文 |
| 名称描述实体+属性 | `company_name`、`order_id` | 避免 `id1`、`data_a` 等无意义名称 |
| 跨用例共享变量只定义一次 | `company_id` 可被 TC-001 和 TC-002 共用 | 不同用例若引用同一数据，变量名相同 |
| 同实体多实例加序号 | `company_id_1`、`company_id_2` | 一次测试创建多条同类数据时 |

## 五、生产侧与消费侧的完整交接流程

```
test-data-preparation (Mode A)
  │
  ├── 创建测试数据（API 调用）
  │     ↓
  ├── 记录实际 ID 到变量映射表
  │     ↓
  └── 输出 data-prep-report-{date}-{time}.md
        │ §5 变量映射表
        ↓
AI 执行测试用例
  │
  ├── 读取 data-prep-report 中的 §5 变量映射表
  │     ↓
  ├── 将 {引用:变量名} 替换为实际值
  │     ↓
  └── 执行用例步骤（前置条件已就绪）
```

## 六、常见问题

| 问题 | 处理方式 |
|------|---------|
| 数据准备报告有多份 | 取最新创建时间的一份（文件名中时间戳最大） |
| 同一变量名出现在多条用例中 | 都使用相同实际值（共享数据） |
| 变量值为空字符串 | 视为无效，阻塞该用例并告知用户 |
| 数据准备报告中有变量但用例中没有引用 | 忽略，不报错 |
