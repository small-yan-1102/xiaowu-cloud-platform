# 用例与PRD一致性检查清单

> 本文件是 test-case-prd-consistency skill 的 Reviewer 知识库，由 SKILL.md 在 Phase 2 执行审查时加载。
> **替换此文件即可改变审查标准和修复方案，无需修改 SKILL.md 中的指令逻辑。**

---

## 一、不一致根因分类

### 类型 A：AI 理解偏差

| 子类型 | 表现 | 识别方法 |
|-------|------|---------|
| PRD描述模糊导致过度设计 | 用例中出现 PRD 未提及的细节或逻辑 | 逐句对比 PRD 原文，找 PRD 中无对应描述的步骤/预期 |
| 术语误解 | 功能名词、按钮文案与 PRD 不一致 | 提取用例中的所有 UI 文本，与 PRD 一一比对 |
| 流程顺序错误 | 操作步骤顺序与 PRD 流程图相反或错位 | 核对步骤编号和依赖关系 |

### 类型 B：测试设计方法过度扩展

| 方法 | 合理扩展（可保留） | 过度设计（需处理） |
|------|-----------------|----------------|
| 边界值分析 | PRD 中明确写明的边界条件 | AI 自行假设的边界值（PRD 无对应） |
| 异常场景 | PRD 中提及的异常流程 | PRD 未提及的假设性异常 |
| 等价类划分 | PRD 明确的分类枚举 | AI 自行创造的分类 |
| 安全测试 | PRD 明确要求验证的安全规则 | AI 自行补充的 XSS/注入等安全检查 |

### 类型 C：历史用例干扰

- 参考了旧版本 PRD 的用例逻辑
- 继承了已过时或错误的用例模式
- 受到其他模块用例的模式污染（功能点误用）

### 类型 D：PRD 与系统不同步

- PRD 已更新，用例基于旧版本生成
- 系统已实现变更，PRD 未同步，用例无法判断孰对孰错
- 用例未严格按 PRD 章节编写，来源追溯困难

## 二、逐项审查清单

对每条问题用例，逐项执行以下检查：

### PRD 一致性检查

| 编号 | 检查项 | 判断 | 说明 |
|------|-------|------|------|
| CH-1 | PRD 是否明确描述了该步骤或预期结果？ | 是/否 | 否 → 可能是类型 A 或 B |
| CH-2 | 用例引用的 PRD 版本是否是当前最新版？ | 是/否 | 否 → 类型 C 或 D |
| CH-3 | 用例中的术语（功能名、按钮文案、提示文案）是否与 PRD 完全一致？ | 是/否 | 否 → 类型 A |
| CH-4 | 是否参考了历史用例？ | 是/否 | 是 → 检查历史用例是否过时（类型 C）|
| CH-5 | 步骤顺序是否与 PRD 流程图/描述一致？ | 是/否 | 否 → 类型 A |
| CH-6 | 测试设计扩展（边界/异常）是否有 PRD 依据？ | 是/否 | 否 → 类型 B，按修复方案处理 |

> **断言可执行性（CH-7～CH-9）由 `test-case-review` 统一负责，本 skill 不重复执行。**
> 如需检查断言模糊、层级缺失、数值断言未升级等问题，请先执行 `test-case-review`。

## 三、修复方案决策表

| 场景 | 修复方案 | 处理方式 |
|------|---------|---------|
| PRD 正确，用例错误 | **方案 A：修正用例** | 按 PRD 原文修正步骤和预期结果；更新「关联需求」字段明确 PRD 出处 |
| 用例正确，PRD 过时 | **方案 B：更新 PRD** | 标记 PRD 需要同步更新；用例按实际系统行为执行，备注差异原因 |
| AI 合理补充（有测试价值） | **方案 C：保留并标注** | 在用例「备注」字段中说明扩展理由；标注为「基于 PRD 的合理扩展：XXX」 |
| 重复或无效 | **方案 D：删除或跳过** | 与其他用例重复 → 标记跳过；无法验证或无效 → 删除 |
| 预期结果模糊/不可执行 | **→ 转交 test-case-review** | 断言质量问题（CH-7/8/9）由 `test-case-review` 统一处理，本 skill 不输出修复方案 |

## 四、快速检查清单

> 逐项核对，全部通过则用例质量合格。

```
── PRD 一致性 ──
☐ 用例是否标注了 PRD 版本和章节（「关联需求」字段）？
☐ 功能名称/模块名称是否与 PRD 一致？
☐ 操作步骤顺序是否与 PRD 流程图/描述一致？
☐ 预期结果是否能在 PRD 中找到明确依据？
☐ 测试数据是否符合 PRD 中的示例或规则？
☐ 是否存在 PRD 未提及的额外步骤？（若有，是否标注为合理扩展？）
☐ 边界值和异常场景是否有 PRD 依据？（若无，是否标注为合理扩展？）
☐ 所有 UI 文本（按钮文案、提示文案）是否与 PRD/原型精确匹配？

── 断言可执行性（CH-7～CH-9）──
> ⚠️ 此部分由 test-case-review 负责，本 skill 跳过。如发现断言质量问题，请触发 test-case-review。
```

## 五、预防规则

### 用例生成时

| 规则 | 说明 |
|------|------|
| 强制引用 PRD 章节 | 每条用例「关联需求」字段必须标注「PRD V{X.X} 第X.X节」 |
| 区分核心与扩展 | 用例「备注」字段中明确标注哪些是 PRD 要求、哪些是测试设计扩展 |
| 版本锁定 | 用例中注明基于的 PRD 版本号，PRD 更新后同步审查用例 |

### 用例评审时

| 规则 | 说明 |
|------|------|
| 抽样对照 | 随机抽取 20% 用例与 PRD 原文对照 |
| 关键词匹配 | 检查功能名词、按钮文案、提示信息是否与 PRD 一致 |
| 流程验证 | 验证操作步骤顺序是否与 PRD 流程图一致 |

## 六、典型案例

### 案例 1：AI 过度设计（类型 B）

**PRD 原文**：
> 用户点击"提交"按钮后，系统保存数据。

**问题用例步骤**：
> 步骤3：输入特殊字符"@#$%"验证XSS攻击防护

**根因**：PRD 未提及 XSS 防护，属于 AI 自行假设的安全测试（类型 B）
**修复**：方案 D（删除）或移至独立的安全测试用例，添加备注说明扩展理由

---

### 案例 2：术语不一致（类型 A）

**PRD 原文**：
> 点击「立即认证」按钮进入认证流程

**问题用例步骤**：
> 步骤1：点击「开始认证」按钮

**根因**：按钮名称"立即认证" ≠ "开始认证"（类型 A 术语误解）
**修复**：方案 A（修正用例），与 PRD 保持一致

---

### 案例 3：合理的测试扩展（类型 B，可保留）

**PRD 原文**：
> 支持上传图片格式：JPG、PNG

**用例扩展步骤**：
> 上传 GIF 格式文件（预期：提示"不支持的格式"）

**根因**：虽然 PRD 未明确提及 GIF，但属于合理的反向测试（类型 B，有测试价值）
**修复**：方案 C（保留），添加备注「基于 PRD 的合理扩展：验证非支持格式的错误处理」
