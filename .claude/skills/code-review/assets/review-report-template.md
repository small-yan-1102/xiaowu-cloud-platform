---

## 使用指南

### 1. 报告结构说明

- **审查概览**: 快速了解整体质量
- **文件审查详情**: 每个文件的具体问题
- **问题汇总**: 按严重度分级的问题列表
- **审查结论**: 是否通过的明确判断
- **后续行动**: 需要修复的问题清单

### 2. 状态标记

| 标记 | 含义 |
|------|------|
| ✅ 通过 | 无问题或仅有 Info 建议 |
| ⚠️ 有建议 | 有 Warning 问题,建议修复 |
| ❌ 需修改 | 有 Critical 问题,必须修复 |

### 3. 变更类型标记

| 标记 | 含义 |
|------|------|
| 🆕 新增 | 新创建的文件 |
| ✏️ 修改 | 修改现有文件 |
| 🗑️ 删除 | 删除的文件 |

### 4. 填写建议

- **问题描述**: 客观描述,不带有主观情绪
- **位置**: 精确到行号
- **建议**: 给出具体修复方案,而非仅指出问题
- **原因**: 说明为什么需要修复,帮助理解
