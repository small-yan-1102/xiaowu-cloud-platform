## 后续步骤建议

### 门禁通过时

```
1. 执行线上验证：调用 test-execution Mode C（线上回归）
2. 验证完成后，使用 test-result-sync 同步线上验证结果
```

### 门禁不通过时

```
1. 修复阻塞项（见上表）
2. 重新执行相关测试用例
3. 重新执行 bug-sync 更新缺陷清单
4. 重新触发发布门禁
```

---

## 检查项说明

| 检查项 | 通过条件 | 说明 |
|--------|---------|------|
| **P0 用例全部通过** | 所有 P0 用例结果 = 通过 | P0 代表核心业务链路，失败即阻塞发布 |
| **无开放 Fatal 缺陷** | 开放 Fatal 数 = 0 | 致命缺陷不允许带着上线 |
| **无开放 Critical 缺陷** | 开放 Critical 数 = 0 | 严重缺陷不允许带着上线 |
| **P0+P1 通过率 ≥ 阈值** | 通过率 ≥ 设定阈值（默认 90%）| 衡量整体质量水平，阈值可按项目调整 |
