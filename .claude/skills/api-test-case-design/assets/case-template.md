**前置条件**：
1. [认证] 调用 POST /auth/login（账号：{账号} / 密码：{密码}），获取 Token，记录为 {记录为:token}
2. [数据存在] {前置数据描述}，ID 记录为 {记录为:变量名}
   [验证: GET /api/{endpoint}/{引用:变量名} 返回 HTTP 200]

**请求**：
- **Method**：GET / POST / PUT / DELETE / PATCH
- **URL**：`{Base URL}/api/{endpoint}`
- **Headers**：
  ```json
  {
    "Authorization": "Bearer {引用:token}",
    "Content-Type": "application/json"
  }
  ```
- **Body**：
  ```json
  {
    "field1": "{测试数据值}",
    "field2": {值}
  }
  ```

**测试数据**：
| 字段 | 值 | 说明 |
|------|-----|------|
| field1 | 值1 | 说明1 |

**预期结果**：
1. HTTP 状态码：200（L1）
2. [断言: 响应体] `data.id` 不为空（L1）
3. [断言: 响应体] `data.name` = "{测试数据值}"（L1）
4. [断言: 响应体] `code` = 0（L1）
5. [断言: 响应体] `data.status` 为枚举值之一：{枚举列表}（L2）

**清理步骤**（用例执行完毕后执行，如有写操作）：
1. [清理] 调用 DELETE /api/{endpoint}/{引用:变量名}
2. [清理] 验证 GET /api/{endpoint}/{引用:变量名} 返回 HTTP 404（L1）

## 动态值标记语法速查

| 标记 | 含义 |
|------|------|
| `{时间戳}` | yyyyMMddHHmmss，确保字段唯一性 |
| `{非空}` | 值不为空即可 |
| `{格式:PATTERN}` | 值需符合指定正则格式 |
| `{包含:关键词}` | 值包含指定关键词即可 |
| `{记录为:变量名}` | 记录该值，供后续用例通过 `{引用:变量名}` 使用 |
| `{引用:变量名}` | 引用前序用例中用 `{记录为:}` 记录的值 |

## 断言级别定义

| 级别 | 含义 | 适用场景 |
|------|------|---------|
| **L1 强验证** | 必须通过，失败则用例失败 | HTTP 状态码、核心响应字段、副作用验证 |
| **L2 软验证** | 不通过记录警告，不影响用例判定 | 响应字段格式、枚举值范围 |
| **L3 观察性** | 仅记录，不判定 | 响应时间、可选字段存在性 |
