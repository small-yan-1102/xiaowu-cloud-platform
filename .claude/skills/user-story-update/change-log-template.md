# 用户故事变更日志模板

本文件定义了根据审阅反馈修改用户故事后,生成变更日志的标准格式。

## 1. 生成规则

### 1.1 必填字段

- **修改时间**: 格式 `YYYY-MM-DD HH:mm`
- **修改原因**: 明确标注审阅来源(测试/产品/后端/综合)
- **修改概览**: 统计修改/新增/删除的 Quest 数量
- **详细变更**: 每条修改必须记录 Quest、修改类型、修改内容、审阅来源、状态
- **审阅意见处理情况**: 区分已处理和待确认的意见
- **修改影响范围**: 说明涉及文件和下游影响
- **人工复核建议**: 列出需要人工确认的事项

### 1.2 修改类型分类

| 类型 | 说明 | 示例 |
|------|------|------|
| 需求变更 | PRD理解偏差、功能遗漏 | 更新产品原文、需求解读 |
| 接口更新 | 接口路径/参数/方法变更 | 更新接口对接表 |
| 组件调整 | 组件选型变更 | 更新组件选型表 |
| 验收标准 | 补充或修改验收条目 | 增加边界场景验证 |
| 需求补充 | 新增 Quest | 新增功能模块 |
| 需求删除 | 删除 Quest | 确认不需要的功能 |

### 1.3 状态标记

- ✅ **已更新**: 修改已完成
- ⚠️ **待确认**: 需要用户进一步确认

## 3. 填写说明

### 3.1 详细变更表

| 字段 | 说明 | 示例 |
|------|------|------|
| Quest | 修改的 Quest 编号或名称 | `Quest 1: 用户列表管理` |
| 修改类型 | 从"修改类型分类"中选择 | `接口更新` |
| 修改内容 | 简要描述修改了什么 | `接口路径 /api/users → /api/users/v2` |
| 审阅来源 | 审阅反馈文档中的意见编号 | `后端审阅反馈 #3` |
| 状态 | ✅ 已更新 或 ⚠️ 待确认 | `✅ 已更新` |

### 3.2 审阅意见处理情况

**已处理意见**: 已经根据意见完成修改的条目
- 审阅来源: 标注是哪份审阅文档的第几条意见
- 原始意见: 引用审阅反馈的原文
- 处理方式: 说明如何修改的

**待确认意见**: 需要用户进一步决策的条目
- 待确认原因: 说明为什么需要确认(如:影响范围大、需确认版本范围等)

### 3.3 人工复核建议

列出需要人工检查的事项,例如:
- 新增功能是否纳入当前版本
- 接口变更是否已与后端确认
- 组件选型是否符合项目规范
- 是否有遗漏的审阅意见未处理
