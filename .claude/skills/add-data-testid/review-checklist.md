# data-testid 添加检查清单

本文件定义了添加 data-testid 时的检查清单。

## 一、命名规范检查

- [ ] 格式符合 `{模块编号}_{场景编号}_{序号}`
- [ ] 模块编号为两位数字
- [ ] 场景编号为 `layout` 或 `S{N}-{M:02d}` 格式
- [ ] 序号为两位数字,从 01 递增
- [ ] 整个页面中 data-testid 唯一

## 二、元素覆盖检查

### 布局元素
- [ ] 页面根容器
- [ ] 主要内容区
- [ ] 侧边栏(如有)
- [ ] 顶部导航(如有)

### 交互元素
- [ ] 按钮(提交、取消、删除等)
- [ ] 输入框(文本、数字、日期等)
- [ ] 下拉选择框
- [ ] 复选框/单选框
- [ ] 弹窗触发点

### 状态元素
- [ ] 条件渲染容器
- [ ] 空状态容器
- [ ] 加载中状态
- [ ] 错误状态
- [ ] 成功提示

### 数据展示
- [ ] 列表/表格容器
- [ ] 分页组件
- [ ] 排序/筛选控件

## 三、添加规则检查

### 静态元素
- [ ] 使用 `data-testid="xxx"` 直接添加

### 动态/条件渲染
- [ ] 使用 `:data-testid="condition ? 'xxx' : 'yyy'"` 动态绑定
- [ ] 所有分支都有对应 testid

### v-for 列表
- [ ] 仅在容器上加,不在循环项加
- [ ] 容器有明确的 testid

### 弹窗/抽屉
- [ ] 加在弹窗组件本身
- [ ] 弹窗内关键元素也有 testid

## 四、代码安全性检查

- [ ] 未修改业务逻辑
- [ ] 未修改 class 或样式
- [ ] 未修改组件功能
- [ ] 仅添加 data-testid 属性
- [ ] 未覆盖已有 data-testid(除非用户要求)

## 五、映射关系检查

- [ ] 映射清单已生成
- [ ] 每个 testid 都能追溯到测试用例场景
- [ ] 用户已确认映射清单
- [ ] 映射关系无遗漏

## 使用建议

### 常规添加
- 全量检查所有维度

### 快速添加
- 重点关注"元素覆盖检查"和"添加规则检查"

### 质量检查
- 重点关注"命名规范检查"和"代码安全性检查"
