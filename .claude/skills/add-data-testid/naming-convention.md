# data-testid 命名规范

## 格式

`{模块编号}_{场景编号}_{序号}`

## 组成部分

| 部分 | 格式 | 说明 | 示例 |
|------|------|------|------|
| 模块编号 | 两位数字 | 与路由页面对应 | `03`, `04` |
| 场景编号 | `layout` 或 `S{N}-{M:02d}` | 布局或场景 | `layout`, `S1-01` |
| 序号 | 两位数字 | 同一场景内递增 | `01`, `02` |

## 示例

| 场景 | data-testid |
|------|------------|
| 页面根容器 | `03_layout_01` |
| 主要内容区 | `03_layout_02` |
| 场景1-按钮 | `03_S1-01_01` |
| 场景1-弹窗 | `03_S1-01_02` |
| 场景2-表单 | `03_S2-01_01` |

## 特殊规则

### 1. v-for 列表
- ❌ 不在循环项加固定 testid
- ✅ 在列表容器上加

```vue
<!-- ❌ 错误 -->
<div v-for="item in list" :data-testid="`item_${item.id}`">

<!-- ✅ 正确 -->
<ul data-testid="03_S1-01_01">
  <li v-for="item in list">{{ item.name }}</li>
</ul>
```

### 2. 条件渲染
使用动态绑定:

```vue
<div :data-testid="isVisible ? '03_S1-01_01' : '03_S1-01_02'">
```

### 3. 弹窗组件
加在弹窗组件本身:

```vue
<el-dialog data-testid="03_S1-01_02" title="编辑">
  <!-- 内容 -->
</el-dialog>
```

### 4. 静态元素
直接添加:

```vue
<button data-testid="03_S1-01_01">提交</button>
```

## 注意事项

1. **模块编号**: 由用户定义或根据项目路由配置确定,本技能不硬编码映射表
2. **序号递增**: 同一场景内从 01 开始递增
3. **唯一性**: 整个页面中 data-testid 必须唯一
4. **语义化**: 虽然使用编号,但通过映射清单可以追溯含义
