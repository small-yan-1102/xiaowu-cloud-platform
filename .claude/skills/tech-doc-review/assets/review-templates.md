# 技术文档审阅报告模板

> 本文件是 tech-doc-review skill 的报告模板，由 SKILL.md 在 Phase 5 输出阶段加载。
> **替换此文件即可改变报告输出格式，无需修改 SKILL.md 指令逻辑。**

---

## 一、单文档审阅报告模板

```markdown
# 技术文档审阅报告

**文档名称**: {文档名称}
**文档类型**: {后端技术设计 / 前端用户故事 / 接口定义 / 数据库设计 / data-testid映射 / 综合技术方案}
**审阅日期**: {YYYY-MM-DD}
**关联 PRD**: {PRD 版本号}
**关联迭代**: {迭代名称}

---

## 一、审阅范围

| 审阅维度 | 状态 | 说明 |
|---------|------|------|
| C - 需求一致性与风险场景 | ✅ 已审阅 / ⬜ 不适用 | |
| U - UI 元素文案精确性 | ✅ / ⬜ | |
| F - 表单字段约束完整性 | ✅ / ⬜ | |
| O - 状态流转可观测性 | ✅ / ⬜ | |
| D - 测试数据准备支撑 | ✅ / ⬜ | |
| A - 异步流程等待策略 | ✅ / ⬜ | |
| T - data-testid 覆盖度 | ✅ / ⬜（可选） | |
| R - 排序/筛选规则精确性 | ✅ / ⬜ | |
| K - 跨系统测试可执行性 | ✅ / ⬜ | |

---

## 二、审阅发现

### 二.1 阻塞项（B）— 必须在测试设计前解决

| # | 维度 | 检查项 | 问题描述 | 建议修正方式 |
|---|------|-------|---------|------------|
| 1 | {C/U/F/...} | {检查项编号} | {具体问题} | {建议} |

### 二.2 补充项（S）— 建议补充，可并行处理

| # | 维度 | 检查项 | 缺失内容 | 对 AI 测试的影响 |
|---|------|-------|---------|----------------|
| 1 | {维度} | {编号} | {缺失什么} | {AI 无法做什么} |

### 二.3 补充 AI 测试场景建议（R-补充）

> 本节产出的场景建议将作为 `test-case-design` Skill **Mode B** 的输入。

| # | 风险类型 | 风险描述 | AI 可测性 | 建议验证策略 |
|---|---------|---------|----------|------------|
| 1 | {C-7/C-10/...} | {风险描述} | 可测 / 需辅助 / 不可测 | {验证方式} |

### 二.4 疑问项（Q）— 需要开发确认

| # | 维度 | 问题 | 影响范围 |
|---|------|------|---------|
| 1 | {维度} | {疑问} | {影响} |

### 二.5 改进建议（A）— 可选

| # | 维度 | 建议内容 | 预期收益 |
|---|------|---------|---------|
| 1 | {维度} | {建议} | {收益} |

---

## 三、AI 不可测风险登记

| # | 风险类型 | 需要的专项测试 | 建议负责人 |
|---|---------|-------------|----------|
| 1 | {风险} | {测试类型} | {角色} |

---

## 四、整体评估

| 评估维度 | 结论 |
|---------|------|
| 审阅完成度 | {X}% （已审维度 / 应审维度） |
| B 级问题数 | {N} 个 |
| S 级问题数 | {N} 个 |
| **整体判定** | **通过 / 有条件通过 / 不通过** |

**判定依据**:
- 通过：无 B 级问题，S 级 ≤ 3 个
- 有条件通过：无 B 级问题，S 级 > 3 个
- 不通过：存在 B 级问题

---

**审阅人**: AI 自动审阅
**审阅时间**: {YYYY-MM-DD HH:mm}
```

## 二、审阅汇总报告模板

```markdown

**迭代名称**: {迭代名称}
**审阅日期**: {YYYY-MM-DD}

---

| 1 | {名称} | {类型} | 通过/有条件/不通过 | {N} | {N} | {N} | {N} |
```
