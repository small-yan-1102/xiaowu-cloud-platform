# UI Mock Gen 最佳实践

## 概述

本文档总结了前端界面 Mock 数据化搭建过程中的最佳实践，用于指导类似任务的执行。

---

## 1. 组件选择优先级

### 核心原则

**必须优先使用项目已封装公共组件**，而非底层 UI 库的原生组件。

### 组件选择流程

```
需要某种功能的组件
  ├── 1. 搜索 src/components/common/ → 找到 → 使用 ✅
  ├── 2. 搜索 src/components/business/ → 找到 → 使用 ✅
  ├── 3. 搜索项目内类似页面的实现 → 找到参考 → 参照使用 ✅
  └── 4. 以上都没有 → 使用 UI 库原生组件 ⚠️ 需确认
```

### 常见错误

- **问题**：直接使用底层 UI 库的原生组件，而项目已有封装版本
- **解决**：实现前先搜索 `src/components/` 目录，确认是否有现成组件
- **问题**：手搓确认弹窗，而项目有统一的 confirm 方法
- **解决**：简单确认框优先使用项目提供的 confirm 方法

## 2. 搜索表单规范

### 适用场景
- 列表页搜索区域
- 详情页子列表搜索
- 任何需要多字段搜索的场景

### 核心原则

如果项目已封装搜索表单组件，**必须使用**，不要自行拼接输入框组合。

---

## 3. 表单校验规范

### 核心原则

使用项目标准的校验方案（如项目封装的校验 hook），不要手动判断表单有效性。

### 提交按钮规范

**正确**：提交按钮始终可点击，通过校验提示错误

```javascript
// 正确：校验不通过时提示错误，按钮不禁用
async function handleSubmit() {
  const valid = await formRef.value.validate()
  if (!valid) return
  // 继续提交逻辑
}
```

**错误**：禁用提交按钮

```javascript
// 错误：不要用 computed 判断表单有效性来禁用按钮
const formValid = computed(() => formData.name && formData.type)
// <button :disabled="!formValid">提交</button>
```

## 4. 数据结构规范

### Mock 数据必须覆盖的场景

1. **列表数据量**
   - 至少 20+ 条数据用于分页测试
   - 包含第 1 页、中间页、最后页的数据

2. **状态值覆盖**
   - 所有枚举状态（待审核、已通过、已驳回等）
   - 各种业务状态组合

3. **边界情况**
   - 空数据状态
   - 超长文本（验证截断和 tooltip）
   - 错误/失败状态
   - 加载状态

### 数据模型定义示例

```javascript
// enums.js - 统一定义枚举
export const STATUS_MAP = {
  PENDING: { value: 'PENDING', text: '待处理', color: '#999' },
  PROCESSING: { value: 'PROCESSING', text: '处理中', color: '#1890ff' },
  COMPLETED: { value: 'COMPLETED', text: '已完成', color: '#52c41a' },
  FAILED: { value: 'FAILED', text: '处理失败', color: '#ff4d4f' },
}

// list.js - 生成足够的数据量
export const mockList = Array.from({ length: 35 }, (_, i) => ({
  id: `ITEM${String(i + 1).padStart(3, '0')}`,
  status: Object.keys(STATUS_MAP)[i % 4],
  // ... 其他字段
}))
```

## 5. 组件拆分原则

### 何时需要拆分子组件

1. **独立分页需求** — 父子列表各有分页
2. **复杂交互隔离** — 子组件有独立的搜索、筛选、排序

---

## 6. 样式实现规范

### 严格对照原型图

1. 颜色值必须使用原型图定义的精确值
2. 不确定的颜色、间距、字体大小必须询问
3. 不得使用"看起来差不多"的近似值

---

## 7. 不确定点处理流程

### 遇到以下情况必须暂停询问

1. **接口字段不明确** — 含义、类型、是否必填
2. **交互逻辑不明确** — 按钮行为、状态流转、权限控制
3. **样式规范不明确** — 颜色值、布局规则

### 询问模板

```
关于 [功能/页面] 有以下不确定点需要澄清：

1. [具体问题 1]
   - 当前理解：...
   - 不确定原因：...

2. [具体问题 2]
   - 当前理解：...
   - 不确定原因：...

请确认以上理解是否正确，或提供正确的实现方式。
```
