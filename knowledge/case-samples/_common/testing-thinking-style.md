## R1 选择类字段

**规则**：遇到下拉框 / 单选 / 多选 / 选择框类字段时，测试以下子项：

1. **选项列表完整性** — 可选值是否与 PRD/数据源一致
2. **默认值** — 是否有默认选中项，默认值是否正确
3. **必填校验** — 未选择时提交，提示"这是必填项"
4. **条件显隐** — 该字段是否仅在特定条件下出现（如"频道标签=子频道时才显示关联频道"）
5. **弹窗选择交互** — 点击选择框是否弹出选择弹窗，弹窗内的筛选/勾选/确认/取消行为
6. **联动置灰** — 选择某选项后，其他互斥选项是否正确置灰

**证据**：
```
频道标签 → 必填项 / 单选 / 选项(新开/子频道/合并/重签续签) / 默认选中(新开)
频道/作品 → 点击选择框出现弹窗 / 已勾选频道类→作品类置灰不可勾选
发布语种 → 多选 / 取语种配置表 / 未选择时默认'全部语种'
禁用语种 → 多选 / 与发布语种互斥（其一选择，另一置灰）
```

## R2 条件分支

**规则**：遇到"条件 A = X 时，显示/隐藏/变更字段 B"的逻辑时，测试以下子项：

1. **每个条件值独立展开** — 条件的每个可能值（如频道标签的 4 个值）都要有对应的测试分支
2. **最深展开 3 层** — 条件 → 子字段 → 子字段的校验规则，逐层展开
3. **条件变更后联动** — 从条件值 A 切换到条件值 B 时，子字段是否正确更新

**证据**：
```
根据'已选线索的类型'判断填写字段：
  线索类型 = 主频道 → 展示频道标签/选择关联频道/盗版视频信息
  线索类型 = 子频道 → 展示频道标签/选择关联频道/盗版视频信息
  线索类型 = 视听作品 → 展示单开合集/首发时间/发布语种/禁用语种
  ...

选择关联频道 → 出现条件：
  频道标签 = 子频道 → 海外频道ID必填
  频道标签 = 合并 → 海外频道ID与国内频道名必填其一
  频道标签 = 重签/续签 → 海外频道ID必填
```

## R3 联合校验

**规则**：遇到跨字段联合校验（多个字段值组合决定通过/失败）时：

1. **枚举有效/无效组合** — 列出所有可能的字段值组合
2. **配具体数据举例** — 复杂规则必须附带真实业务数据场景
3. **区分页面校验和接口校验** — 前端同页面校验 vs 后端接口校验可能有不同规则

**证据**：
```
联合频道标签校验 → 频道标签=合并时：
  填写例子：
    线索组1：王小二/频道标签：合并/国内频道名：罗小黑（未交接）
    线索组2：小萝卜/频道标签：合并/国内频道名：王小二（未交接）
    → 王小二/罗小黑/小萝卜 均未交接，频道标签≠合并 → 提示：请选择主频道关联

接口校验：每组线索之间的线索类型是否一致
  不一致 → 提示：当前交接单存在频道、作品类信息，请分批交接
```

## R4 提交流程

**规则**：遇到提交 / 保存 / 确认类操作时，按以下顺序测试：

1. **必填校验** — 逐个必填字段清空后提交，验证提示文案
2. **业务校验（同页面）** — 字段组合是否符合规则（如联合校验）
3. **接口校验（后端）** — 提交后后端是否有额外校验（如跨线索组类型一致性）
4. **成功路径** — 校验全通过后，页面跳转/列表新增/状态变更
5. **失败路径** — 每种校验失败的具体提示文案
6. **保存草稿 vs 提交** — 两种操作的区别（是否推送、状态不同）
7. **重复提交防护** — 快速连续点击是否有防重机制

**证据**：
```
【提交】→ 校验必填项
  未填写 → 提示：这是必填项
  已填写 → 校验组合规则
    不符合 → 报错：请选择主频道关联
    符合 → 允许提交 → 列表新增交接单(已交接) + 数据推送至AMS

【保存草稿】→ 校验同提交
  → 列表新增交接单(待交接) + 数据不推送至AMS

【取消】→ 关闭抽屉页
```

## R5 异常枚举

**规则**：遇到输入/选择类场景时，按业务状态枚举异常变体：

1. **每个输入场景枚举 3~5 种状态变体** — 按业务实体的生命周期状态分类
2. **枚举模式**：不存在 / 已失效(解约/过期/删除) / 正常状态 / 特殊状态
3. **每个变体明确预期提示文案**

**证据**：
```
海外频道ID → 场景：
  不存在的YT频道ID → 提示：当前海外频道不存在
  运营状态=搁置的频道 → 提示：当前海外频道不存在
  运营状态=解约的频道 → 提示：当前海外频道不存在
  运营状态=在运营的频道 → 回显：海外频道名称+ID

国内频道名 → 场景：
  不存在的国内频道 → 提示：当前国内频道不存在/未生效
  已解约的国内频道 → 提示：当前国内频道不存在/未生效
  已过期的国内频道 → 提示：当前国内频道不存在/未生效
  已删除合约中的频道 → 提示：当前国内频道不存在/未生效
  已生效的国内频道 → 回显：国内频道名称
```

## R6 跨系统

**规则**：遇到操作结果需要推送/同步到其他系统时：

1. **验证推送是否发生** — 操作成功后，接收方系统是否收到数据
2. **验证推送条件** — 哪些操作触发推送、哪些不触发（如保存草稿不推送，提交推送）
3. **验证接收方数据一致性** — 接收方展示的数据是否与发送方一致
4. **验证接收方独立展示** — 接收方有自己的展示逻辑（如 AMS 签约频道新增频道标签列）

**证据**：
```
【提交】→ 允许提交 → 数据'推送'至 AMS
【保存草稿】→ 数据'不推送'至 AMS

AMS 签约频道：
  列表新增'频道标签'icon（位置：入库链接后）
  检索项新增'频道标签'（下拉多选）
  导出新增：频道标签、关联频道名称、关联海外频道名、关联海外频道ID

建群任务：
  点击'交接单编号' → 调用AMS接口查询 → 打开交接单详情页
```
