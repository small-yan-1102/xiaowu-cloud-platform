## 1. 业务背景：为什么需要逾期结算

### 正常结算流程

```
YouTube 视频发布 → 产生月度收益 → 生成冲销报表 → 分销商登记视频归属 → 生成结算单 → 财务付款
```

正常情况下，分销商在**视频发布次月 28 日前**完成视频登记，收益按正常流程结算。

### 问题：分销商逾期登记

部分分销商在次月 28 日之后才登记视频，导致该视频的收益错过了当月结算窗口。这些"逾期"的收益需要单独处理——这就是**逾期结算处理**模块的核心目的。

### V4.5 新增：区分"逾期"与"跨期正常"

V4.5 之前，只要错过当月结算窗口的记录统一标记为"逾期"。V4.5 细化了分类：

| 情况 | 判定 | 处理方式 |
|------|------|---------|
| 登记日 **> 次月 28 日** | **逾期** | 拆分时可能扣除违约金 |
| 登记日 **≤ 次月 28 日** | **跨期正常** | 正常跨月认领，拆分时不扣违约金 |

> 简单理解：28 日是分界线。28 日当天及之前登记 = 正常；29 日及之后 = 逾期。

## 2. 整体业务链路

```
┌─────────────────────────────────────────────────────────────────┐
│                        剧老板系统（分销商端）                      │
│                                                                  │
│  分销商登记视频归属  ──────────────────────────────┐              │
│                                                    │              │
└────────────────────────────────────────────────────│──────────────┘
                                                     │ 异步通知
                                                     ▼
┌─────────────────────────────────────────────────────────────────┐
│                        结算系统（内部）                            │
│                                                                  │
│  ① SET-01 登记状态分发                                            │
│     检索该视频的【未登记】收益记录                                   │
│     ├── 登记日 ≤ 次月28日 → 标记为【跨期正常未拆分】                 │
│     └── 登记日 > 次月28日 → 标记为【逾期登记未拆分】                 │
│                          │                                       │
│                          ▼                                       │
│  ② SET-03 导入误差处理（运营操作）                                  │
│     运营导入无归属视频ID → 占比校验 → 零值清理                  │
│     → 误差抹平 → 生成【未登记】收益记录                              │
│                          │                                       │
│                          ▼                                       │
│  ③ SET-02 财务结算处理（财务操作）                                  │
│     财务在对应 Tab 勾选记录 → 批量拆分                               │
│     → 记录流转至【已拆分】→ 进入后续结算流程                          │
│                                                                  │
│  ④ SET-04 漏抓责任界定（辅助展示）                                  │
│     系统自动判定技术漏抓 → 展示橙色标签 → 辅助财务线下决策             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**关键理解**：SET-01 ~ SET-04 不是严格的顺序流程。SET-03（导入）产生数据 → SET-01（登记触发状态分发）→ SET-02（财务拆分）是主链路；SET-04（漏抓标签）是贯穿展示，不影响数据流转。

## 3. 角色与权限

| 角色 | 在哪操作 | 能做什么 | 不能做什么 |
|------|---------|---------|-----------|
| **分销商** | 剧老板系统 | 登记视频归属 | 不接触结算系统 |
| **运营人员** | 结算系统 → 逾期结算处理 → 【未登记】Tab | 导入无归属视频ID | 不能拆分、不能看其他 Tab |
| **财务人员** | 结算系统 → 逾期结算处理 → 全部 4 个 Tab | 查看列表、批量拆分、导出 | 不能导入 |

```
运营看到的页面：
┌──────────┐
│ 未登记    │  ← 仅此一个 Tab，含【导入】按钮
└──────────┘

财务看到的页面：
┌──────────┬──────────────┬──────────────┬────────┐
│ 未登记    │ 逾期登记未拆分 │ 跨期正常未拆分 │ 已拆分  │
└──────────┴──────────────┴──────────────┴────────┘
                 ↑                ↑            ↑
             可拆分           可拆分        只读查看/导出
```

## 4. 核心概念速查

### 4.1 收益记录状态

一条视频收益记录在逾期结算模块中的生命周期：

```
                        ┌──────────────────┐
                        │   未登记 (0)       │  ← 导入时的初始状态
                        └────────┬─────────┘
                                 │
                    分销商在剧老板登记视频
                                 │
                 ┌───────────────┼───────────────┐
                 │                               │
        登记日 > 次月28日                 登记日 ≤ 次月28日
                 │                               │
                 ▼                               ▼
    ┌────────────────────┐          ┌────────────────────┐
    │ 逾期登记未拆分 (1)   │          │ 跨期正常未拆分 (3)   │  ← V4.5 新增
    └─────────┬──────────┘          └─────────┬──────────┘
              │                               │
         财务批量拆分                      财务批量拆分
              │                               │
              ▼                               ▼
    ┌─────────────────────────────────────────────────┐
    │                  已拆分 (2)                       │
    │  原状态 = "逾期登记"（默认色）                       │
    │  原状态 = "跨期正常"（绿色）                         │
    └─────────────────────────────────────────────────┘
```

### 4.2 关键时间节点

| 时间节点 | 用途 | 判定规则 |
|---------|------|---------|
| **发布次月 28 日** | 逾期分界线 | 登记日 ≤ 28日 = 跨期正常；> 28日 = 逾期 |
| **发布次月 15 日 23:59:59** | 漏抓分界线 | 视频同步时间 > 此时刻 = 技术漏抓 |

> 示例：视频 2026 年 1 月发布
> - 次月 28 日 = 2026-02-28
> - 次月 15 日 = 2026-02-15 23:59:59
> - 分销商 2026-02-28 登记 → 跨期正常
> - 分销商 2026-03-01 登记 → 逾期
> - 视频 2026-02-16 才同步到系统 → 技术漏抓

### 4.3 技术漏抓标签

**是什么**：橙色只读标签，展示在视频 ID 旁边，表示"这个视频是因为技术原因同步晚了，不是分销商的责任"。

**有什么用**：辅助财务在线下判断是否扣除违约金——技术漏抓的视频通常不扣。

**在哪展示**：发布视频登记管理列表、逾期结算管理各 Tab 列表、导出文件中。

**交互**：标签不可点击，hover 显示 tooltip："同步时间 > 发布次月15号，系统判定为技术漏抓"。

## 5. 四个功能模块详解

### 5.1 SET-01 登记状态分发（自动，用户无感）

**触发时机**：分销商在剧老板系统提交视频登记后，结算系统异步执行。

**做了什么**：

```
1. 接收剧老板的登记通知
2. 在逾期结算管理中检索该视频的所有【未登记】记录
3. 对每条记录独立判定：
   - 登记日 ≤ 该记录对应视频发布次月28日 → 状态改为【跨期正常未拆分】
   - 登记日 > 该记录对应视频发布次月28日 → 状态改为【逾期登记未拆分】
4. 如果没找到匹配的【未登记】记录 → 什么都不做，登记正常完成
```

**用户感知**：分销商只看到"登记成功"，不知道后台发生了什么。财务刷新页面后能在对应 Tab 看到状态变化。

---

### 5.2 SET-02 财务结算处理（财务手动操作）

**入口**：结算系统 → 逾期结算处理

**页面结构**：4 个 Tab，从左到右

| Tab | 展示内容 | 可操作 |
|-----|---------|--------|
| 未登记 | 尚未被分销商登记的收益记录 | 运营可导入；财务只读 |
| 逾期登记未拆分 | 登记日超过次月28日的记录 | 财务可批量拆分、导出 |
| **跨期正常未拆分**（新增） | 登记日未超过次月28日的记录 | 财务可批量拆分、导出 |
| 已拆分 | 已完成拆分的记录 | 财务只读、导出 |

**财务核心操作——批量拆分**：

```
1. 进入【逾期登记未拆分】或【跨期正常未拆分】Tab
2. 通过到账月份、频道名称、频道ID、视频ID 筛选定位记录
3. 勾选要拆分的记录（至少1条）
4. 点击【批量拆分】按钮
5. 弹出确认弹窗："确认拆分选中的 X 条记录？"
   ├── 点击【确认】→ 执行拆分 → 提示"操作成功" → 记录从当前 Tab 消失
   └── 点击【取消】→ 不执行，回到列表
6. 被拆分的记录进入【已拆分】Tab
   - 原状态列展示来源：
     · 来自逾期 Tab → 显示"逾期登记"（默认文字色）
     · 来自跨期 Tab → 显示"跨期正常"（绿色文字）
```

**跨期正常 Tab 特殊标注**：右上角有注释文案——"注：此类视频登记时间未超过发布次月28日，属于正常跨期认领，拆分时不扣除违约金。"

### 5.3 SET-03 导入误差处理（运营手动操作）

**入口**：结算系统 → 逾期结算处理 → 【未登记】Tab → 【导入】按钮

**做了什么**：运营上传仅含**视频ID**一列的 Excel 文件，系统根据视频ID自动关联冲销报表中的收益占比数据，再按频道维度逐一执行误差处理。

**导入模板**：只有1个字段——`*视频ID`（必填）。系统通过视频ID自动匹配频道、收益占比等信息。

**处理流程（每个频道独立执行）**：

```
导入文件（仅含视频ID列表）
│
▼ 系统根据视频ID自动关联冲销报表，获取收益占比等数据
▼ 按【频道ID + 收款系统 + 到账月份】分组
│
├── 频道 A
│   │
│   ▼ R1 占比完整性校验
│   │  导入视频的收益占比求和 ≟ 冲销报表该维度无归属视频占比总和
│   │  ├── 不等 → ❌ 阻断该频道，失败原因："本次导入视频不全，占比校验不通过"
│   │  └── 相等 → 继续 ▼
│   │
│   ▼ R2 零值清理
│   │  对每个视频计算：ROUND(占比 × 频道CMS收益, 2)
│   │  ├── = 0.00 → 舍弃该视频（金额太小，无意义）
│   │  └── > 0.00 → 保留
│   │
│   ▼ R3 误差抹平
│   │  保留视频的四舍五入金额求和 vs 冲销报表无归属金额
│   │  ├── 有差额且 ≤ $1 → 差额累加到占比最大的视频上
│   │  └── 无差额 → 直接通过
│   │
│   ▼ R4 安全阈值检查
│   │  ├── 差额 > $1 → ❌ 阻断该频道，失败原因："拆分差额超过1美金，请检查"
│   │  └── 差额 ≤ $1 → ✅ 该频道导入成功
│   │
│   ▼ R5 极端情况（V4.5 新增）
│      全部视频取整后均为 0.00
│      → ❌ 阻断，失败原因："异常：单笔金额过小导致精度丢失，请检查"
│
├── 频道 B → 同样执行 R1~R5 ...
│
└── 频道 C → 同样执行 R1~R5 ...
```

**重要补充（V4.5）**：美国收益和新加坡收益各自独立执行 R2~R5，使用各自的调差值（调差2/调差3），三种收益任一超限即阻断该频道。

**导入结果**：

| 场景 | 提示 |
|------|------|
| 全部频道成功 | "共导入 X 条记录" |
| 部分频道失败 | "部分频道导入失败，请下载失败文件查看原因" |
| 全部频道失败 | 同上，失败文件包含全部频道 |

### 5.4 SET-04 漏抓责任界定（系统自动展示）

**用户不操作，只看**。

**展示规则**：

| 条件 | 展示 |
|------|------|
| 视频同步时间 **>** 发布次月 15 日 23:59:59 | 橙色【技术漏抓】标签 |
| 视频同步时间 **≤** 发布次月 15 日 23:59:59 | 不展示 |
| 视频同步时间为空 | 不展示（按非漏抓处理） |
| 视频发布时间为空 | 不展示（按非漏抓处理） |

**展示位置**：视频 ID 同行旁侧（不是换行）。

**导出文件**：包含"技术漏抓"字段（是/否），财务用于线下核算违约金。

### A.1 SET-01 登记状态分发 — 判定规则细节

| 判定项 | 规则 | 边界值 | 备注 |
|-------|------|--------|------|
| 逾期阈值 | 登记日 > 视频发布次月第 28 日 | 28日 23:59:59 = 未逾期；29日 00:00:00 = 逾期 | "登记日"定义待确认（A-1：剧老板提交时间 vs 结算系统处理时间） |

| 无匹配记录 | 结算系统未找到该视频的【未登记】记录 | — | 正常完成登记，不抛异常，不影响分销商端 |
| 异步失败 | DB 写入失败 | — | 分销商端仍收到"登记成功"，后台记录异常日志 |
| 并发登记 | 两人同时登记同一视频 | — | 后发出的请求应收到"该视频已被登记"提示（S-1 待确认） |
| 重复登记 | 已完成状态变更的视频再次被登记 | — | 系统行为待确认（S-8） |

**状态码写入**：

| 场景 | 写入 status 值 | 数据库表 |
|------|---------------|---------|
| 未逾期 | 3（跨期正常未拆分） | 收益记录表 |
| 逾期 | 1（逾期登记未拆分） | 收益记录表 |

| 4 | 点击【确认】→ 全部成功 | toast "操作成功"；当前 Tab 刷新，被拆分记录消失 |
| 5 | 点击【确认】→ 部分失败 | toast "X条成功，Y条失败，请重试失败记录"；成功记录流转至已拆分 |
| 6 | 点击【取消】 | 弹窗关闭，不执行，停留当前 Tab（勾选状态是否保留：B-2 待确认） |

**自动扩展隔离规则**：跨期 Tab 拆分只扩展跨期记录；逾期 Tab 只扩展逾期记录；两者不互相影响。

### A.3 SET-03 导入误差处理 — 校验规则与提示文案

#### A.3.1 文件约束

| 约束 | 值 |
|------|-----|
| 文件大小上限 | 5MB |
| 文件格式 | 待确认（TBD-5） |

| 超限提示 | "文件过大，请分批导入（最大5MB）" |
| 格式错误提示 | "文件格式错误，请使用标准模板" |

#### A.3.2 校验规则完整清单（按执行顺序）

| 顺序 | 规则 | 适用收益类型 | 条件 | 结果 | 提示文案 |
|------|------|------------|------|------|---------|
| 1 | R1 占比完整性 | 总收益 | 导入视频占比求和(不取整) ≠ 冲销报表无归属视频占比总和 | 阻断该频道 | "本次导入视频不全，占比校验不通过" |
| 2 | R2 零值清理 | 总收益 | ROUND(占比×CMS收益+调差1, 2) = 0.00 | 舍弃该视频 | — |
| 3 | R3 误差抹平 | 总收益 | 四舍五入后求和 vs 无归属金额有差额且 ≤$1 | 差额累加到占比最大视频 | — |
| 4 | R4 安全阈值 | 总收益 | 差额绝对值 > $1 | 阻断该频道 | "异常：拆分差额超过1美金，请检查" |
| 5 | R5 极端情况 | 总收益 | 全部视频取整后均为 0.00 | 阻断该频道 | "异常：单笔金额过小导致精度丢失，请检查" |
| 6 | R6-美国 | 美国收益 | 同 R2~R5，使用调差2 | 同上 | 同上 |
| 7 | R6-新加坡 | 新加坡收益 | 同 R2~R5，使用调差3 | 同上 | 同上 |

**关键边界值**（测试重点）：

| 边界 | 值 | 预期 |
|------|-----|------|
| 差额 = $0.00 | 精确匹配 | 直接通过，无需抹平 |
| 差额 = $1.00 | ≤$1 边界 | R3 抹平，导入成功 |
| 差额 = $1.01 | >$1 边界 | R4 阻断 |
| 单视频 ROUND 后 = $0.00 | 零值边界 | R2 舍弃 |
| 全部视频 ROUND 后 = $0.00 | R5 极端 | R4 阻断（R5 触发） |
| 占比求和 = 占比总和 | R1 通过 | 继续执行 R2 |
| 占比求和 ≠ 占比总和 | R1 失败 | 阻断 |

**调差对应关系**（V4.5 明确）：

| 收益类型 | 公式 | 调差 |
|---------|------|------|
| 总收益 | revenue + 调差1 | 调差1 |
| 美国收益 | 美国收益 + 调差2 | 调差2 |
| 新加坡收益 | 新加坡收益 + 调差3 | 调差3 |

## 7. 典型操作场景（用户故事）

### 场景 1：正常跨期结算（最常见）

> 分销商小王在 2 月 25 日登记了 1 月发布的视频。

1. 小王在剧老板点击"登记"
2. 结算系统收到通知，检索该视频的【未登记】记录
3. 判定：登记日 2/25 ≤ 次月28日(2/28) → 跨期正常
4. 该记录状态从【未登记】变为【跨期正常未拆分】
5. 财务刘姐打开逾期结算处理 → 【跨期正常未拆分】Tab
6. 看到注释："拆分时不扣除违约金"
7. 勾选该记录 → 点击【批量拆分】→ 确认
8. 记录进入【已拆分】Tab，原状态列显示绿色"跨期正常"

### 场景 2：逾期结算

> 分销商小李在 3 月 5 日才登记 1 月发布的视频。

1. 小李在剧老板登记
2. 结算系统判定：登记日 3/5 > 次月28日(2/28) → 逾期
3. 记录状态变为【逾期登记未拆分】
4. 财务刘姐在【逾期登记未拆分】Tab 拆分
5. 记录进入【已拆分】Tab，原状态列显示"逾期登记"（默认色）
6. 刘姐导出文件，结合"技术漏抓"字段判断是否扣违约金

### 场景 3：运营导入视频ID

> 运营小张需要导入 3 月的无归属视频。

1. 小张打开逾期结算处理 → 【未登记】Tab → 点击【导入】
2. 上传 Excel 文件
3. 系统逐频道执行 R1~R5 校验
4. 某频道差额 $0.50 ≤ $1 → 自动抹平，导入成功
5. 某频道占比求和不一致 → 阻断，标记"本次导入视频不全"
6. 提示"部分频道导入失败"，小张下载失败文件排查
7. 成功导入的视频出现在【未登记】Tab 列表中，等待分销商登记

### 场景 4：技术漏抓视频的完整流转

> 一个视频 1 月发布，但 2 月 20 日才同步到系统。

1. 系统判定：同步时间 2/20 > 次月15日(2/15) 23:59:59 → 技术漏抓
2. 该视频在各列表中展示橙色【技术漏抓】标签
3. 财务导出文件，"技术漏抓"字段 = "是"
4. 财务线下判断：技术原因导致的延迟，不扣违约金
