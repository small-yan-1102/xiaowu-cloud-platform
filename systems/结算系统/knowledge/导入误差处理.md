### 5.3 SET-03 导入误差处理（运营手动操作）

**入口**：结算系统 → 逾期结算处理 → 【未登记】Tab → 【导入】按钮

**做了什么**：运营上传仅含**视频ID**一列的 Excel 文件，系统根据视频ID自动关联冲销报表中的收益占比数据，再按频道维度逐一执行误差处理。

**导入模板**：只有1个字段——`*视频ID`（必填）。系统通过视频ID自动匹配频道、收益占比等信息。

**占比数据来源说明**：

导入时系统直接查询**冲销报表（`yt_reversal_report`）中已有的占比字段**，不是实时从上游重新计算。占比在冲销报表生成阶段就已写入，完整的上游链路为：

```
yt_month_channel_revenue_source（视频级收益明细）
    ↓ 按 videoId 查快照表 youtube_video_pipeline → 得到 pipelineId
    ↓ 按 pipelineId + cms 聚合收益（分子）
    ↓ 按 channelId + cms 汇总全部视频收益（分母）
    ↓ 占比 = 分子 / 分母（精度10位 HALF_UP）
yt_month_channel_revenue（频道月度汇总，含占比字段 source_channel_revenue_ratio）
    ↓ 冲销报表生成时读取此表，按占比拆分子集收益
yt_reversal_report（冲销报表，每行已包含占比）
    ↓ 无归属视频行：sign_channel_id = -1, pipeline_id = "unattributed"
    ↓ SET-03 导入时，系统按视频ID匹配这些无归属行，直接读取其占比字段
```

> **关键理解**：导入的视频ID对应的是冲销报表中 `sign_channel_id = -1`（无归属）的记录。这些记录在冲销报表生成阶段已携带 `source_channel_revenue_ratio`（通道占比）、`us_revenue_ratio`（美国收益占比）、`sg_revenue_ratio`（新加坡收益占比）等字段，导入时直接使用，无需反向查询上游表。

**处理流程（每个频道独立执行）**：

```
导入文件（仅含视频ID列表）
│
▼ 系统按视频ID查询冲销报表中的无归属记录（sign_channel_id=-1）
▼ 直接读取每行已有的收益占比（冲销报表生成时已计算写入）
▼ 按【频道ID + 收款系统 + 到账月份】分组
│
├── 频道 A
│   │
│   ▼ R1 占比完整性校验
│   │  导入视频的收益占比求和 ≟ 冲销报表该维度无归属视频占比总和
│   │  ├── 不等 → ❌ 阻断该频道，失败原因："本次导入视频不全，占比校验不通过"
│   │  └── 相等 → 继续 ▼
│   │
│   ▼ R2 零值清理
│   │  对每个视频计算：ROUND(占比 × 频道CMS收益, 2)
│   │  ├── = 0.00 → 舍弃该视频（金额太小，无意义）
│   │  └── > 0.00 → 保留
│   │
│   ▼ R3 误差抹平
│   │  保留视频的四舍五入金额求和 vs 冲销报表无归属金额
│   │  ├── 有差额且 ≤ $1 → 差额累加到占比最大的视频上
│   │  └── 无差额 → 直接通过
│   │
│   ▼ R4 安全阈值检查
│   │  ├── 差额 > $1 → ❌ 阻断该频道，失败原因："拆分差额超过1美金，请检查"
│   │  └── 差额 ≤ $1 → ✅ 该频道导入成功
│   │
│   ▼ R5 极端情况（V4.5 新增）
│      全部视频取整后均为 0.00
│      → ❌ 阻断，失败原因："异常：单笔金额过小导致精度丢失，请检查"
│
├── 频道 B → 同样执行 R1~R5 ...
│
└── 频道 C → 同样执行 R1~R5 ...
```

**重要补充（V4.5）**：美国收益和新加坡收益各自独立执行 R2~R5，使用各自的调差值（调差2/调差3），三种收益任一超限即阻断该频道。
