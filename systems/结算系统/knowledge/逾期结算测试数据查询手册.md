# 逾期结算测试数据查询手册

> **所属系统**：结算系统（silverdawn-finance）
> **用途**：查找同时满足「可拆分子集 + 有无归属收益 + 可逾期拆分」的冲销表测试数据
> **创建日期**：2026-04-16
> **数据库**：`silverdawn_finance`（Host: 172.16.24.61:3306）
> **关联文档**：[逾期结算处理.md](逾期结算处理.md) §5 批量拆分、[YT核算表逻辑.md](YT核算表逻辑.md) §7 冲销表查询链路

---

## 目标

找到一条冲销表数据，同时满足三个条件：

| # | 条件 | 对应表/字段 |
|---|------|-----------|
| 1 | **可拆分子集** | `yt_reversal_report`: channel_type=2（合集）或 0（单开已转型），channel_split_status=1 |
| 2 | **有无归属收益** | `yt_reversal_report`: unattributed_revenue > 0 |
| 3 | **可进行逾期拆分** | `video_composition_overdue`: 有 status=1（逾期登记未拆分）的对应记录 |

---

## 查询步骤

### Step 1：基础条件查询 `[仅供理解思路，查询结果不可直接使用]`

**思路**：先用最简单的条件圈出候选频道——合集或转型单开、有无归属收益、已到账、且当前没有逾期未拆分记录（说明还可以补充逾期数据）。

```sql
SELECT r.month AS 结算月份, r.receipted_at AS 到账日期, 
       r.channel_id AS 频道ID, r.channel_name AS 频道名称, 
       r.cms AS 收款系统,
       CASE r.channel_type 
           WHEN 0 THEN '单开(已转型)' 
           WHEN 2 THEN '合集' 
       END AS 频道类型,
       ROUND(r.unattributed_revenue, 2) AS 无归属总收益
FROM yt_reversal_report r
WHERE (r.channel_type = 2 
       OR (r.channel_type = 0 AND r.transform_date IS NOT NULL 
           AND r.transform_date >= CONCAT(r.month, '-01')))
  AND IFNULL(r.unattributed_revenue, 0) > 0
  AND r.received_status != 0
  AND NOT EXISTS (
      SELECT 1 FROM video_composition_overdue o
      WHERE o.channel_id = r.channel_id 
        AND o.month = r.month AND o.cms = r.cms
        AND o.status = 1 AND o.deleted = 0
  )
ORDER BY r.unattributed_revenue DESC
LIMIT 20;
```

**问题**：查出的频道包含"纯无归属"频道（如 Funny Girls Series Arabic，100% 收益为无归属，`channel_split_status` 为空，没有任何子集行）。技术上满足 SQL 条件但实际不可用。

**教训**：`unattributed_revenue > 0` 不等于"有子集且有剩余无归属"。需要加条件排除从未拆分过的频道。

---

### Step 2：加严条件 — 必须已拆分过（有子集行） `[仅供理解思路，查询结果不可直接使用]`

**思路**：追加 `channel_split_status = 1`，确保频道已有子集行存在，不是纯无归属的空壳。

```sql
SELECT r.id, r.month, r.receipted_at, r.channel_id,
       LEFT(r.channel_name, 22) as cn, r.cms,
       r.channel_split_status,
       ROUND(r.cms_revenue, 2) as total_rev,
       ROUND(r.unattributed_revenue, 2) as unattr_rev,
       (SELECT COUNT(*) FROM yt_reversal_report sub 
        WHERE sub.parent_id = r.id AND sub.channel_type = 1) as subset_count
FROM yt_reversal_report r
WHERE r.channel_type = 2
  AND r.channel_split_status = 1          -- 新增：必须已拆分
  AND IFNULL(r.unattributed_revenue, 0) > 0
  AND r.received_status != 0
  AND NOT EXISTS (
      SELECT 1 FROM video_composition_overdue o
      WHERE o.channel_id = r.channel_id 
        AND o.month = r.month AND o.cms = r.cms
        AND o.status = 1 AND o.deleted = 0
  )
ORDER BY r.unattributed_revenue DESC
LIMIT 15;
```

**问题**：查出 15 条，但验证发现大部分子集的 `contract_id = NULL`、`settlement_created_status = 2`（无需结算），子集收益极小（$0.01~$50），~98% 都是无归属。有子集 ≠ 子集能结算。

---

### Step 3：再加严 — 子集必须可结算 `⭐ 推荐日常使用`

**思路**：子集中至少有 1 个同时满足 `contract_id IS NOT NULL`（有合约）AND `settlement_created_status IN (0, 1)`（待生成或已生成结算单），排除全部"无需结算"的频道。

> **此 SQL 查出的数据可直接使用**——频道已拆分、有可结算子集、有无归属收益、已到账。后续只需按 Step 4~6 查视频 ID 和补数据即可。

```sql
SELECT r.id, r.month, r.receipts_at, r.channel_id,
       LEFT(r.channel_name, 22) AS cn, r.cms,
       ROUND(r.cms_revenue, 2) AS total_rev,
       ROUND(r.unattributed_revenue, 2) AS unattr_rev,
       (SELECT COUNT(*) FROM yt_reversal_report sub 
        WHERE sub.parent_id = r.id AND sub.channel_type = 1) AS total_subs,
       -- 可结算子集数（有合约 + 待生成/已生成）
       (SELECT COUNT(*) FROM yt_reversal_report sub 
        WHERE sub.parent_id = r.id AND sub.channel_type = 1 
        AND sub.contract_id IS NOT NULL
        AND sub.settlement_created_status IN (0, 1)) AS settleable_subs,
       -- 已出结算单的子集数
       (SELECT COUNT(*) FROM yt_reversal_report sub 
        WHERE sub.parent_id = r.id AND sub.channel_type = 1 
        AND sub.settlement_created_status = 1) AS settled_subs
FROM yt_reversal_report r
WHERE r.channel_type = 2
  AND r.channel_split_status = 1
  AND r.received_status != 0
  AND IFNULL(r.unattributed_revenue, 0) > 0
  -- 新增：至少有1个可结算子集
  AND EXISTS (
      SELECT 1 FROM yt_reversal_report sub
      WHERE sub.parent_id = r.id AND sub.channel_type = 1
      AND sub.contract_id IS NOT NULL
      AND sub.settlement_created_status IN (0, 1)
  )
  AND NOT EXISTS (
      SELECT 1 FROM video_composition_overdue o
      WHERE o.channel_id = r.channel_id 
        AND o.month = r.month AND o.cms = r.cms
        AND o.status = 1 AND o.deleted = 0
  )
ORDER BY r.unattributed_revenue DESC
LIMIT 15;
```

**结果**：15 条。`UC6KEKxHLxRqOmyyTweUWqPg` 在列（2 个可结算子集、1 个已出结算单、无归属 $9,120）。

### Step 4：查可导入的视频 ID

**思路**：确定目标频道后，查 `yt_month_channel_revenue_source` 中该频道的视频，排除已在 overdue 表中的，LEFT JOIN `video_composition` 看分销商信息。

**关键月份关系**：
```
冲销表 month = 2025-12（结算月份/Payout Period）
冲销表 receipted_at = 2026-01-22（到账日期）
逾期导入到账月份 = 2026-01
revenue_source 查询月份 = 到账月份 - 1 = 2025-12
```

> ⚠️ 首次查询用了 month=2025-11 导致 0 条结果，原因是搞错了月份关系。逾期导入代码中 `revenueMonth = 到账月份 - 1`，不是 `结算月份 - 1`。

```sql
SELECT 
    s.target_video_id AS video_id,
    s.cms,
    ROUND(s.revenue, 2) AS revenue,
    LEFT(s.pipeline_id, 14) AS pipeline,
    CASE vc.related_type 
        WHEN 0 THEN 'NORMAL' 
        WHEN 1 THEN 'OVERDUE' 
        ELSE NULL 
    END AS reg_type,
    vc.team_id,
    LEFT(vc.team_name, 20) AS team_name
FROM yt_month_channel_revenue_source s
LEFT JOIN video_composition vc 
    ON s.target_video_id = vc.video_id AND vc.deleted = 0
WHERE s.target_channel_id = 'UC6KEKxHLxRqOmyyTweUWqPg'
  AND s.month = '2025-12'                     -- 到账月份(2026-01) - 1
  AND NOT EXISTS (
      SELECT 1 FROM video_composition_overdue o
      WHERE o.video_id = s.target_video_id
        AND o.receipted_month = '2026-01'     -- 到账月份
        AND o.deleted = 0
  )
ORDER BY s.revenue DESC
LIMIT 30;
```

**结果**：787 个可导入视频，总收益 $9,121.58。但全部 `pipeline_id = "unattributed"`，`video_composition` 无记录，无分销商。

### Step 5：查 AMS pipeline → 子集映射

**思路**：逾期拆分需要 pipeline_id + sign_channel_id + sign_channel_name 三个字段一致。从 AMS 查该频道有哪些发布通道可用。

```sql
-- 数据库：silverdawn_ams
SELECT pc.pipeline_id, pc.sign_channel_id, pc.sign_channel_name,
       pc.service_page_name, pc.status AS pub_status
FROM ams_publish_channel pc
WHERE pc.register_channel_id = 'UC6KEKxHLxRqOmyyTweUWqPg'
ORDER BY pc.status, pc.sign_channel_id;
```

**结果**：2 条映射。

| pipeline_id | sign_channel_id | 子集名称 | 状态 |
|------------|----------------|---------|------|
| `5f068c97...` | 19864 | 豆瓣短剧 | 0 |
| `97806db8...` | 76580 | 小萝nice | 1 |

---

### Step 6：验证 pipeline 在冲销表中的使用情况

**思路**：`checkPipelineIdExist` 校验规则要求：冲销表中同 pipeline_id + month + cms 下若存在 `settlement_created_status = 0` 的记录，则拒绝拆分。需要检查每个 pipeline 在冲销表中的现有状态。

```sql
-- 数据库：silverdawn_finance
SELECT pipeline_id, month, sign_channel_id, 
       settlement_created_status AS settle_st,
       contract_id,
       ROUND(cms_revenue, 2) AS rev
FROM yt_reversal_report
WHERE pipeline_id IN (
    '5f068c97500111efb7d300163e2f91e6',
    '97806db8236046aa8c7fa17ef52044bc'
)
AND parent_channel_id = 'UC6KEKxHLxRqOmyyTweUWqPg'
ORDER BY pipeline_id, month DESC;
```

**结果与判定**：

| pipeline_id | 冲销表是否存在 | settle_st | 能否用于逾期拆分 | 原因 |
|------------|-------------|-----------|----------------|------|
| `5f068c97...` | 存在（3 条） | 0（未生成） | **不能** | `checkPipelineIdExist` 发现 settle_st=0 → 抛异常 |
| `97806db8...` | 不存在 | — | **能** | 无冲突记录 |

## 最终结论

| 项目 | 值 |
|------|-----|
| 目标频道 | `UC6KEKxHLxRqOmyyTweUWqPg` |
| 冲销表 | month=2025-12, receipted_at=2026-01-22, CMS=XW |
| 可导入视频 | 787 个（全部 unattributed，无分销商） |
| 可用 pipeline | `97806db8236046aa8c7fa17ef52044bc` → 子集"小萝nice"(sign_channel_id=76580) |
| 限制 | 只有 1 个可用 pipeline，拆分后冲销表只新增 1 条子集行 |

**导入后需 DB 补数据的 SQL**：

```sql
-- 1. 在逾期结算处理页面导入视频（到账月份选 2026-01）
-- 2. 导入后执行以下 SQL 补充字段并改状态
UPDATE video_composition_overdue 
SET status = 1,
    pipeline_id = '97806db8236046aa8c7fa17ef52044bc',
    sign_channel_id = 76580,
    sign_channel_name = '小萝nice',
    team_id = '1988839584685428736',
    team_name = 'HELLO BEAR',
    registration_time = NOW()
WHERE receipted_month = '2026-01'
  AND channel_id = 'UC6KEKxHLxRqOmyyTweUWqPg'
  AND status = 0 AND deleted = 0;
```

## 查询条件逐步收敛过程

```
Step 1  合集 + 无归属 > 0 + 已到账
        → 20 条（含纯无归属空壳）
        ✗ Funny Girls Series Arabic: 100% 无归属，无子集

Step 2  + channel_split_status = 1（必须已拆分）
        → 15 条（排除了空壳）
        ✗ 大部分子集 contract_id=NULL, settle_st=2

Step 3  + EXISTS(子集 contract_id NOT NULL AND settle_st IN (0,1))
        → 15 条（子集真正可结算）
        ✓ UC6KEKxHLxRqOmyyTweUWqPg 入选

Step 4  查该频道视频 ID（month=到账月份-1）
        → 787 个视频，全部无分销商
        ⚠️ 月份踩坑：首次用 2025-11 查到 0 条

Step 5  查 AMS pipeline 映射
        → 2 条：豆瓣短剧 + 小萝nice

Step 6  验证 pipeline 冲销表冲突
        → 豆瓣短剧(settle_st=0) 不可用
        → 小萝nice(无记录) 可用 ✓
```
