> 触发方式：作品列表操作列「设置」按钮

| 成功提示 | `设置成功` |

### 5.1 表单字段

| 字段名 | 必填 | 组件类型 | 字段标识 | 校验规则 | 说明 |
|--------|:----:|---------|---------|---------|------|
| 垂类 | ✅ | VlLeadSelect（级联选择） | `categoryId` + `parentCategoryId` | `required` 校验 | 接口 `getCategory({fullData:0})` 获取树形数据 |
| 别名 | ❌ | VlInputTag（标签输入） | `aliasList` | 最多 5 个，每个最长 50 字符，不能与作品名重复 | 提示文案：`输入后按回车添加别名，最多可添加5个` |
| 入库链接 | ❌ | v-text-field 输入框 | `storageUrl` | URL 格式校验 `strictUrl`，最长 200 字符 | - |

### 5.2 别名校验规则（来源：Vue）

- 与作品名称重复时提示：`别名与作品名称重复，请修改`
- 最多 5 个别名
- 每个别名最长 50 字符

### 5.3 底部按钮

| 按钮 | 样式 | 行为 |
|------|------|------|
| 取消 | outlined | 关闭弹窗 |
| 确认 | primary | 表单校验 → 调用 `setArtwork` 接口 |

### 5.4 后端处理逻辑（来源：Service）

`updateCategory(ChannelCompositionVO vo)` 方法执行：
1. 更新 `ams_composition` 表的 `categoryId`、`aliasList`、`storageUrl`
2. 级联更新 `ams_publish_channel` 表的 `aliasList`
3. 异步推送 MQ 消息到 CRM（topic: `domestic-channel-category`），同步垂类变更
4. `@RepeatSubmit` 防重复提交（仅 Controller 层的 `setCategory` 方法有此注解）

### 7.4 关联频道自动推荐逻辑（来源：Vue）

- 子频道(`channelLabel=1`) 或 合并频道(`channelLabel=2` 且有海外频道) 场景下
- 自动调用 `getChannelDetail` 获取关联频道的运营团队
- 匹配到运营团队后自动勾选对应选项，并显示提示：`推荐分配到与关联频道相同的运营团队`

### 7.5 后端分配核心逻辑（来源：Service）

**分配冲突检测**：
- 维度：运营团队 + 套餐 + 授权平台 + 语种
- 排除已退回(status=3)的记录
- 冲突时抛出异常：`运营团队[X], 套餐[Y], 平台[Z]已分配过`

**分配状态默认值**：
- 外包团队 → 默认已领用(status=2)
- 其他团队 → 默认待领用(status=1)

**批量分配**：
- 仅分配 YouTube/Facebook 平台且合约状态=生效中(0) 的记录

## 14. 后端核心业务规则

### 14.1 列表查询（CompositionFacade.getCompositionPage）

**数据组装流程**：
1. 权限过滤 → `UserCenterUtil.getUserIdByAppUserDept()` 获取管理范围用户ID
2. 分页查询 → `getBaseCompositionPage(type, vo)` 基于 MyBatis Plus
3. 分类处理 → 批量查询 Category 表，拼接 `parentName/childName` 格式
4. 签约频道 → `getGroupedSignChannels()` 批量加载，Map<作品ID, 频道列表>
5. 发布通道 → `currentChannels()` 批量加载
6. 分配状态 → `getAllocateStatusCache()` 缓存查询
7. 用户信息 → 批量调用用户中台获取商务/客服名称
8. 授权到期标签 → 计算每个频道的到期天数和整体过期状态

**复杂 EXISTS 子查询过滤**：
- `allocateStatus`：通过关联 `sign_channel_allocate_record` 表判断分配状态
- `companyName`：关联 `ams_authorize_platform` 表过滤签约主体
- `packageId` / `contractStatus` / `sharePattern`：关联 `ams_sign_channel` 表
- `authorizeExpireType`：关联授权平台表计算到期日期范围，仅针对 YouTube(0)/Facebook(1)

### 14.2 设置操作（AmsCompositionServiceImpl.updateCategory）

1. 更新作品表 3 个字段：`categoryId` / `aliasList` / `storageUrl`
2. 级联更新发布通道表的 `aliasList`
3. 异步 MQ 推送垂类变更到 CRM（topic: `domestic-channel-category`）

### 14.3 导入操作（AmsCompositionServiceImpl.importData）

1. 从 OSS 读取 Excel 文件
2. 使用 EasyExcel 解析数据
3. 校验：作品名+CP名去重、存在性、唯一性
4. 成功时更新入库链接；失败时生成错误 Excel 并返回 OSS 链接

### 14.4 分配操作（DomesticChannelServiceImpl.allocate）

1. 参数校验（IDs/团队非空）
2. 查询签约频道和国内频道数据
3. 获取授权平台映射
4. 冲突检测（团队+套餐+平台+语种四维度）
5. 构建分配记录（外包默认已领用，其他待领用）
6. 写入时间线记录
7. 更新运营团队字段（去重追加）
