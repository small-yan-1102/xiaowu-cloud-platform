## 一、整体功能概述

剧老板端包含两个页面，均为**只读**，不涉及创建、审核等操作流程。

| 页面 | 展示维度 | 数据范围 | 入口 |
|------|---------|---------|------|
| 视频下架列表页 | 作品 + 频道 | **仅已完成**状态的数据 | 导航菜单 |
| 进度查看页（Drawer） | 单作品 + 单频道 | 该条目下所有视频明细 | 列表页「查看详情」 |

**数据权限：** 分配该作品的运营人员及拥有该运营人员管理范围的人员可见。

**数据来源：** AMS 任务单完成后通过 RocketMQ 推送，剧老板端不直连 AMS 数据库。

## 二、视频下架列表页

### 2.1 展示规则

- 按**作品 + 频道**维度展示，一个作品在一个频道下的下架记录 = 一条数据
- 同一作品在多个频道下架时，列表展示多条（每个频道各一条）

**示例：**

| 作品名称 | 频道名称/ID | 下架视频数 |
|---------|-----------|-----------|
| 恶魔之吻 | Nhiệp Chính Drama / UCxxx | 38 |
| 恶魔之吻 | TM - Drama Studio / UCyyy | 15 |
| 恶魔之吻 | VietDuc Network / UCzzz | 22 |

### 2.2 搜索条件

| 字段名称 | 输入类型 | 查询方式 |
|---------|---------|---------|
| 作品名称 | 文本输入 | 模糊查询 |
| 频道名称 | 文本输入 | 模糊查询 |
| 频道ID | 文本输入 | 精准查询 |
| 视频ID | 文本输入 | 精准查询 |
| 下架原因 | 下拉选择 | 全部、临时版权纠纷、调整上线时间、单作品解约、上游版权解约、分销商解约、其他原因 |

- 支持条件交叉查询
- 支持点击「重置」清空所有条件

### 2.3 列表字段

| 字段名称 | 说明 |
|---------|------|
| 作品名称 | 作品名称 |
| 频道名称/ID | 频道名称和ID |
| 下架视频数 | 该作品在该频道下的**全部视频数量**（含成功和失败）；鼠标悬停可查看视频ID明细（Tooltip） |
| 下架原因 | 任务单的下架原因；数据来源：推送时携带的任务单信息 |
| 运营人员 | 该作品的运营负责人；数据来源：剧老板系统中该作品的运营负责人 |
| 完成时间 | 下架完成时间 |
| 操作 | 查看详情（点击打开进度查看 Drawer） |

## 三、进度查看页（Drawer）

- **展示维度**：单作品 + 单频道（从列表页点入，上下文已确定）
- **Drawer 宽度**：900px
- **标题格式**：作品名称 + 频道名称/ID
- **不展示**整体完成进度模块（无进度条、无统计卡片）

### 3.1 视频明细列表字段

| 字段 | 说明 |
|-----|------|
| 序号 | 自增序号 |
| 视频ID | 视频唯一标识 |
| 视频标题 | 视频标题 |
| 频道名称/ID | 频道名称和ID |
| 下架状态 | 成功 / 失败 |
| 完成时间 | 下架完成时间 |
| 失败原因 | 失败时显示具体原因 |

> ⚠️ **待确认项**：技术文档（`剧老板视频管理_V2.0.0.md`）中下架状态枚举为"已完成/处理中/待处理/处理失败"，与 PRD 的"成功/失败"不一致。
> 由于剧老板列表页仅展示已完成状态的数据，进度查看页理论上也只会出现最终状态（成功/失败），需确认技术文档是否有误。

## 四、数据推送机制（AMS → 剧老板）

### 4.1 推送触发时机

- 任务单状态变更为**已完成**时触发（含"全部失败"场景）
- **仅推送一次**（最终结果同步，不做中间状态推送）

### 4.2 分销商匹配规则（2026-04-07 更新）

> 旧版：依据通道关系记录或海外频道绑定的分销商信息直接匹配。  
> **新版：改为按视频发布日期与分配期间匹配，引入数据隔离机制。**

**匹配逻辑：**

```sql
SELECT * FROM team_operate_record
WHERE channel_id = '视频所属海外频道ID'
AND delivery_date <= '视频发布日期'
AND (termination_date IS NULL OR termination_date >= '视频发布日期')
ORDER BY created_at DESC
LIMIT 1
```

**分配期间定义：**

| bind_status | termination_date 取值 |
|-------------|----------------------|
| 0（正常绑定） | 当前日期 |
| 2（已解约） | 解约日期 |

**匹配规则说明：**

| 规则 | 说明 |
|------|------|
| 数据隔离 | 视频发布日期须在分销商分配期间内（`delivery_date` ≤ 发布日期 ≤ `termination_date`）才推送 |
| 无匹配时 | 无匹配分销商的视频**不推送** |
| 多匹配时 | 取 `created_at` 最新的记录推送，并记录诊断日志 |

### 4.3 推送维度与目标

| 项目 | 说明 |
|------|------|
| 推送维度 | 作品 + 海外频道 |
| 推送目标 | 对应剧老板账号 |
| 视频状态映射 | AMS"已完成" → 剧老板"成功"；AMS"处理失败" → 剧老板"失败" |
| 完成时间 | 按视频级别推送（每个视频有独立完成时间） |

### 4.4 重复推送覆盖规则（新增）

| 维度 | 唯一键 | 覆盖规则 |
|------|--------|---------|
| 列表记录 | 作品 + 海外频道 | 同一作品+频道多次推送，**后覆盖前** |
| 视频明细 | 视频ID | 相同视频ID以**最新状态**为准 |

### 4.5 RocketMQ 消息体字段

| 字段名 | 类型 | 说明 | 变化 |
|-------|------|------|------|
| taskNo | string | 任务单编号 | 无变化 |
| workName | string | 作品名称 | 无变化 |
| channelId | string | 海外频道ID | 无变化 |
| channelName | string | 海外频道名 | 无变化 |
| videoId | string | 视频ID | 无变化 |
| videoTitle | string | 视频标题 | 无变化 |
| **publishDate** | datetime | **视频发布日期，用于分销商数据隔离判定** | **新增** |
| takedownStatus | string | 下架状态：成功 / 失败 | 无变化 |
| completeTime | datetime | 视频下架完成时间 | 无变化 |
| failReason | string | 失败原因（失败时必填） | 无变化 |

### 4.6 同步失败处理

> **技术确认项**：RocketMQ 消息发送失败时的重试和补偿机制由技术设计，PRD 未定义。

## 五、与 AMS 端的差异对比

| 对比项 | AMS 端 | 剧老板端 |
|--------|--------|---------|
| 展示维度 | 任务单维度（一个任务单含多个作品） | 作品 + 频道维度 |
| 数据范围 | 所有状态（创建中→已完成） | **仅已完成** |
| 进度模块 | 有整体完成进度（进度条+统计） | **无** |
| 详情页形式 | 抽屉页，全量视频明细 | Drawer，单作品+单频道明细 |
| 导出功能 | 有（已完成时可导出） | **无** |
| 下架状态枚举 | 待处理/处理中/成功/失败/重试中/终止 | 成功 / 失败 |
