## 一、总览：两层状态体系

系统采用**两层状态体系**，两层相互独立、各司其职：

| 层级 | 描述对象 | 状态数量 | 查看位置 |
|------|---------|:-------:|---------|
| **第一层** | 任务单整体所处阶段 | 7 个 | 列表页「处理状态」列 / 详情页状态标签 |
| **第二层** | 单个视频 ID 的执行进度 | 6 个 | 详情页「作品处理进度」→ 视频明细表格「处理状态」列 |

## 二、第一层：任务单生命周期状态

### 2.1 状态总览

| 状态 | 含义 | 颜色 | 是否终态 |
|------|------|------|:-------:|
| 创建中 | 提交后异步获取视频 ID 中 | 🟠 橙色 | — |
| 创建失败 | 所有作品均无可执行视频 ID | 🔴 红色 | ✓（可编辑重建） |
| 待审核 | 视频 ID 获取成功，等待人工审核 | 🟠 橙色 | — |
| 审核拒绝 | 审核不通过 | 🔴 红色 | ✓（可编辑重建） |
| 待处理 | 审核通过，在执行队列中排队 | 🟠 橙色 | — |
| 处理中 | 正在执行视频下架操作 | 🟠 橙色 | — |
| 已完成 | 所有视频均有最终结果（含全部失败场景） | 🟢 绿色 | ✓ |

> **颜色色值**：橙色 `#FFA80F`、红色 `#F24545`、绿色 `#00B09B`

## 三、第二层：视频处理状态

> 显示在任务单详情页「作品处理进度」→ 视频明细表格的「处理状态」列。

### 3.1 状态总览

| 状态 | 含义 | 用户可见展示 | 是否终态 |
|------|------|------------|:-------:|
| 待处理 | 已进入队列，尚未开始执行 | 待处理 | — |
| 处理中 | 正在执行下架操作 | 处理中 | — |
| 重试中 | 执行失败，正在自动重试（≤ 3次） | **处理中**（用户不感知） | — |
| 成功 | 视频下架完成 | 成功 | ✓ |
| 失败 | 不可重试 或 超过重试上限 | 失败（附失败原因） | ✓ |
| 终止 | 任务单创建失败/审核拒绝，视频未被执行 | 终止 | ✓ |

---

### 3.2 状态流转对照表

| 当前状态 | 流转到 | 触发条件 |
|---------|--------|---------|
| —（初始） | **待处理** | 任务单审核通过，视频进入执行队列 |
| **待处理** | **处理中** | 系统按排队顺序开始执行该视频的下架操作 |
| **处理中** | **成功** | 下架操作执行成功，YouTube 返回成功响应 |
| **处理中** | **重试中** | 执行失败，且失败原因属于可重试类型（接口超时 / 限流 / 服务不可用等） |
| **处理中** | **失败** | 执行失败，且失败原因属于不可重试类型（视频不存在 / 无权限 / 已删除等） |
| **重试中** | **重试中** | 等待 5 分钟后再次执行，仍失败且未超过 3 次上限（对用户展示仍为「处理中」） |
| **重试中** | **成功** | 重试执行成功 |
| **重试中** | **失败** | 累计重试达到 3 次上限后仍失败，或本次失败为不可重试类型 |
| —（特殊路径） | **终止** | 所属任务单进入「创建失败」或「审核拒绝」状态，视频未经执行直接终止 |

> **注**：重试中对用户的可见展示为「处理中」，用户感知不到内部重试次数。

---

### 3.4 终止状态说明

「终止」是**校验终态**，与「失败」有本质区别：

| 对比项 | 终止 | 失败 |
|--------|------|------|
| 触发时机 | 任务单创建失败 / 审核拒绝时 | 执行下架操作期间出错 |
| 视频是否被执行过 | **否**，从未进入执行流程 | **是**，执行后未成功 |
| 计入完成百分比分母 | **否**（不计入可执行视频总数） | **是**（计入处理失败数） |

## 四、两层状态的关联关系

```
任务单状态
    ├── 创建中 / 待审核              → 视频处理状态：尚未生成
    ├── 创建失败 / 审核拒绝           → 视频处理状态：终止
    └── 待处理 / 处理中 / 已完成      → 视频处理状态：待处理 / 处理中 / 重试中 / 成功 / 失败
```

## 五、整体完成进度计算规则

> **【2026-04-03 更新】** 修正术语定义，与 PRD 保持一致：
> - "提交视频数" → "提交视频总数" / "可执行视频总数"
> - 明确区分：提交视频总数（含终止）vs 可执行视频总数（不含终止）

---

### 5.1 详情页「整体完成进度」区块的数值来源

```
提交视频总数   =  用户提交的全量视频数（含终止状态）
      ↓
可执行视频总数  =  提交视频总数 − 终止数
      ↓
               =  待处理数 + 处理中数（含重试中） + 成功数 + 失败数

完成百分比     =  (成功数 + 失败数) / 可执行视频总数 × 100%
```

悬停 ℹ️ 图标可查看说明：

> 「提交视频总数=用户提交的全量视频数；终止=创建失败/审核拒绝的视频；可执行视频总数=提交视频总数-终止；已处理=成功+失败；完成百分比=已处理/可执行视频总数×100%」

---

### 5.2 术语对照表

> 【新增】明确四个核心术语的定义和关系

| 术语 | 定义 | 计算方式 |
|------|------|---------|
| **提交视频总数** | 用户提交的全量视频数（含终止状态） | 创建任务单时导入/选择的全部视频 |
| **可执行视频总数** | 实际进入执行阶段的视频数 | 提交视频总数 − 终止数 |
| **终止** | 校验阶段失败的视频数 | 创建失败/审核拒绝的视频 |
| **已处理** | 已完成执行的视频数 | 成功数 + 失败数 |

---

### 5.3 进度 ℹ️ 图标提示文案

悬停「整体完成进度」旁的 ℹ️ 图标，显示说明：

> 「提交视频总数 = 用户提交的全量视频数；终止 = 创建失败 / 审核拒绝的视频；可执行视频总数 = 提交视频总数 − 终止；已处理 = 成功 + 失败；完成百分比 = 已处理 / 可执行视频总数 × 100%」

## 六、处理方式与配额对执行路径的影响

| 处理方式 | 消耗配额 | 审核通过后的执行路径 |
|---------|:-------:|-------------------|
| 视频删除 | ✅ 消耗 | 审核通过 → **待处理**（排队等待配额）→ 处理中 |
| 视频私享 | ❌ 不消耗 | 审核通过 → **直接进入处理中**（跳过排队） |

> 「待处理」队列的本质是**配额管理队列**。视频私享调用权限变更接口、不消耗 YouTube API 配额，因此无需排队，审核通过后直接执行。

**额度不足对状态的影响：**

- 视频删除任务在执行前查询配额，**额度不足时当前批次暂停执行，任务停留在「处理中」**，待每日 16:00 额度更新后自动恢复继续。
- 视频私享任务不查询配额，**不受额度限制影响**。
