# 视频下架 — 完整状态流转图

> 本图整合「任务单状态（第一层）」与「视频处理状态（第二层）」的完整执行流程。  
> 矩形框 = 状态节点；① ② ③ ④ = 执行校验决策点；❌ = 终态·失败

---

```
      按作品 / 解约单创建                        按视频ID创建
              │                                       │
              │ 提交                                   │ 提交（同步校验，不经异步获取）
              ▼                                       │
      ┌──────────────┐                                │
      │    创建中     │                                │
      └──────┬───────┘                                │
             │                                        │
        ┌────┴──────────────┐                         │

        │             ┌────────────┐                  │

        │             └─────┬──────┘                  │

                                           ┌──────────┐

                                   │                          │

                                   │                         │ 编辑后重新提交
                                   │                         └────────→ 创建中 ↩
                                   │
                        ┌──────────┴──────────┐
                     视频删除               视频私享
                        │                      │

                   ┌─────────┐                 │

                                  ┌─────────┐

                                       │

                                  │
                       ┌──────────┴──────────┐

        ▼
```

---

> ℹ️ 校验未通过的视频（数据过滤）标记为「失败」，计入处理失败数，不计入已处理数。  
> ℹ️ 「重试中」对用户展示为「处理中」，用户感知不到内部重试次数。  
> ℹ️ 「待处理」队列本质是配额管理队列，仅视频删除需要排队；视频私享不消耗配额，审核通过后直接进入处理中。
