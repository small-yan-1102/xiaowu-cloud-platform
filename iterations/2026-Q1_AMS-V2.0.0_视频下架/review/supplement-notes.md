## 二、剧老板端权限体系补充

> 来源：`剧老板-权限逻辑.md`

### 2.1 三级权限模型

| 层级 | 机制 | 实现方式 |
|------|------|---------|
| 菜单权限 | 后端 `userAuthorityList` 返回 page 列表 | `handlePermissionMenu` 过滤侧边导航 |
| 路由权限 | 同上 page 列表 | `handlePermissionData` 动态注册路由，无权限→404 |
| 按钮权限 | 后端返回 btn 列表 | `v-auth` 指令（DOM移除）或 `can()` 函数（逻辑判断） |

**测试关注点**：视频下架功能新增菜单和按钮权限后，需验证：
- 无菜单权限用户访问视频下架/视频管理 URL → 预期 404
- 无按钮权限用户 → 对应按钮 DOM 不存在（非 disabled，而是完全移除）

### 2.2 行级数据隔离（authType）

| authType | 含义 | 实际数据过滤行为 |
|:--------:|------|-----------------|
| 1 | 本部门及下级 | **代码实际行为：仅可见自己的数据**（与 authType=3 逻辑相同，未实现部门树展开） |
| 2 | 全部员工 | 不添加任何 operatorId 过滤，可见全部数据 |
| 3 | 按员工选择 | 仅可见自己 + scopeList 中指定员工的数据 |

**与视频下架的关系**：
- AMS 视频下架本期不做数据权限隔离（PRD §7 SC-1 已确认）
- **剧老板端视频管理功能**：受 authType 影响，分销商用户仅能看到归属自己管理范围内的数据。测试时需使用不同 authType 的账号验证数据可见性

### 2.3 权限控制两种方式对比

| 方式 | 文件 | 用法 | 行为差异 |
|------|------|------|---------|
| `v-auth` 指令 | `directives/modules/anth.js` | `v-auth="['code']"` | DOM 不匹配直接 `el.remove()`，**不可恢复** |
| `can()` 函数 | `utils/useAccess.js` | `can(['code'])` → boolean | JS 逻辑判断，控制 Tab 显隐等 |

**测试关注点**：新增的视频下架菜单/按钮权限码需确认使用哪种方式控制。

### 2.4 已知鉴权缺失接口（安全风险备注）

以下接口缺少 `@NeedLogin` 注解，存在越权风险。虽不在本期测试范围内，但作为已知风险记录：

| Controller | 接口 | 风险 |
|-----------|------|------|
| TargetChannelController | `allocateOperator` | 无登录鉴权 |
| CompositionController | `batchAllocateOperator` | 无登录鉴权 |
| TeamController | `queryTeamMember` | 完全无鉴权 |
| TeamController | `teamBaseInfo` | 完全无鉴权 |
| TeamController | `addOrEditTeam` | 仅有限流，无登录鉴权 |

## 三、内容解约单现有功能基线补充

> 来源：`内容解约功能清单.md`（代码逆向梳理）

### 3.1 现有两种创建模式（V1.x 已实现）

| 维度 | 按分销商创建（`type=team`） | 按作品创建（`type=work`） |
|------|--------------------------|------------------------|
| 分销商名称字段 | 显示（必填） | 隐藏 |
| 添加作品弹窗 | 该分销商下待解约作品（下拉多选） | 全局作品搜索表格（左右双栏） |
| 解约分销商名称列 | 不显示 | 显示（点击打开 TeamDialog 多选） |
| 提交接口 | `/compositionTerminate/create` | `/compositionTerminate/batchCreate` |
| 生成结果 | 1 张解约单 | 按分销商自动分组，多张解约单 |

### 3.2 V2.0 新增内容对比（增量测试范围）

| 字段/功能 | V1.x 现有 | V2.0 新增 | 测试关注 |
|----------|:---------:|:---------:|---------|
| 视频是否下架 | 无 | 新增下拉（是/否） | 联动逻辑：解约类型→默认值+锁定状态 |
| 视频下架信息区 | 无 | 新增 4 字段（下架原因/说明/方式/截止日期） | 联动映射+锁定规则 |
| 解约提交二次确认 | 无 | 视频下架=是时弹出确认弹窗 | 弹窗文案+取消/确认行为 |
| 自动创建任务单 | 无 | 提交时自动创建视频下架任务单 | N张解约单→1个任务单 |
| 解约单详情-关联展示 | 无 | 展示关联的任务单编号 | 点击可展开详情 |
| CP名称字段 | 无 | **下期迭代，本期不包含** | 确保不误测 |

### 3.3 现有校验规则（V1.x 基线，需回归）

| 校验项 | 规则 | 错误码 |
|-------|------|--------|
| 文件内重复 | 同名作品导入文件中 ≥ 2 次 | `COMPOSITION_NAME_REPEAT` |
| 作品不存在 | 作品名在已分配列表中匹配不到 | `COMPOSITION_NAME_NOT_ALLOCATED` |
| 同名多条 | 同分销商下同名作品匹配到多条记录 | `COMPOSITION_NAME_DB_MULTI` |
| 作品上限 | 最多 500 个（超出自动截取前 500 条） | 前端提示 |

### 3.4 TeamDialog 选择解约分销商（按作品模式）

| 属性 | 值 |
|------|-----|
| 触发时机 | 按作品创建模式下，点击解约分销商名称单元格 |

## 五、业务背景补充（调研报告摘要）

> 来源：`【授权到期后通知分销商并下架视频】调研报告.md`

### 5.1 三大业务场景

| 场景 | 触发节点 | V2.0 系统化方案 |
|------|---------|----------------|
| 上游版权解约 | CRM 合约到期/解约 | AMS 解约单 → 自动创建视频下架任务（本期第一阶段） |
| 分销商解约 | AMS 发起解约单 | AMS 解约单 → 自动创建视频下架任务（本期第一阶段） |
| 临时版权纠纷/时间调整 | 人工触发 | AMS 手动创建视频下架任务单（本期第一阶段） |

### 5.2 当前痛点（系统改造目标）

1. **依赖人工协同**：多关键动作依赖钉钉 OA/通知，信息传递链路长，存在漏传、延迟风险
2. **系统能力割裂**：解约流程分散在 OA/CRM/AMS 等多系统，无自动触发接口
3. **流程无标准化**：临时下架需求来自非结构化通知，内外部操作不一致

### 5.3 实施路径

| 阶段 | 范围 | 本期覆盖 |
|------|------|:--------:|
| 第一阶段 | 手动创建入口 + 分销商解约单联动 | **本期范围** |
| 第二阶段 | OA 解约流程线上化，打通版权方解约 | 未来迭代 |
| 第三阶段 | 分销商合约管理线上化（签约/解约/补充/重签/续签） | 未来迭代 |
