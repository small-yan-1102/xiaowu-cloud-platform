## 二、排队优先级规则（待处理 / 处理中 Tab）

## 三、四维度排序规则一览表

| 优先级 | 排序维度 | 规则 | 说明 |
|:------:|---------|------|------|
| **1** | 是否当天截止 | 当天截止 **优先** | 截止日期 = 今天的任务排在所有非当天任务之前 |
| **2** | 截止执行日期 | **升序**（近 → 远） | 第1维度相同时，截止日期越近越靠前 |
| **3** | 下架原因优先级 | **1 > 2 > 3 > 4 > 5 > 6** | 第2维度相同时，按下架原因优先级排序 |
| **4** | 审批通过时间 | **升序**（先 → 后） | 第3维度相同时，先审批的排在前面 |

## 四、排序示例

> 今天 = 2026-03-10

```
输入：5 个任务单

  任务B  截止 2026-03-10  优先级 1（临时版权纠纷）
  任务A  截止 2026-03-10  优先级 3（单作品解约）
  任务C  截止 2026-03-11  优先级 1（临时版权纠纷）
  任务D  截止 2026-03-11  优先级 2（调整上线时间）
  任务E  截止 2026-03-12  优先级 1（临时版权纠纷）

排序过程：

  Step 1 — 当天截止（2026-03-10）优先
           ┌── 第一组：任务B、任务A（当天截止）
           └── 第二组：任务C、任务D、任务E（非当天截止）

  Step 2 — 第二组内按截止日期升序
           任务C、任务D（2026-03-11）< 任务E（2026-03-12）

  Step 3 — 同日期内按优先级排序
           第一组：任务B（优先级1）< 任务A（优先级3）
           第二组同日期：任务C（优先级1）< 任务D（优先级2）

最终排队顺序：

  第 1 位  任务B  ← 当天截止 + 优先级最高
  第 2 位  任务A  ← 当天截止 + 优先级次之
  第 3 位  任务C  ← 明天截止 + 优先级 1
  第 4 位  任务D  ← 明天截止 + 优先级 2
  第 5 位  任务E  ← 后天截止
```

## 五、排队规则在额度分配中的应用

```
          每日额度上限 5000 条（16:00 更新）
                        │
                        ▼
             按排队顺序逐任务分配额度
                        │
               ┌────────┴────────┐
           额度充足             额度不足
               │                  │
               ▼                  ▼
          全部执行         执行至额度耗尽
                           剩余视频暂停
                                  │
                                  ▼
                          次日 16:00 额度更新
                                  │
                                  ▼
                        按排队规则重新排序
                        （跨天任务与新任务一起参与）
                                  │
                                  ▼
                           继续分配额度执行
```

> ℹ️ 排队维度为**任务单**（汇总视频数量），分配粒度为**视频ID**（支持单任务单部分执行）。  
> ℹ️ 截止日期已过的任务**不停止执行**，继续排队并触发钉钉预警告警（17:00 仍有当天截止任务未完成时）。
