# 视频下架 — 任务单提交校验规则

> 来源：PRD V2.0-updated4（视频下架PRD-2026040202.md）§3.2 创建任务单  
> **校验分三层**：导入校验（点击导入时）→ 提交同步校验（点击提交时，阻止提交）→ 提交异步校验（提交后后台执行，详情页展示）

---

## 一、两种创建方式总览

```
                        创建视频下架任务单
                                │
                ┌───────────────┴───────────────┐
                │                               │
                ▼                               ▼
       ┌─────────────────┐             ┌─────────────────┐
       │   按作品创建      │             │  按视频ID创建    │
       │（含解约单创建）   │             │                 │
       └────────┬────────┘             └────────┬────────┘
                │                               │
         ┌──────┴──────┐                 ┌──────┴──────┐
         │  批量导入    │                 │  批量导入    │
         │  14 项校验   │                 │  16 项校验   │
         └──────┬──────┘                 └──────┬──────┘
                │ 部分/全部通过                   │ 部分/全部通过
                ▼                               ▼
         ┌──────────────┐               ┌──────────────┐
         │  提交同步校验  │               │  提交同步校验  │
         │  4 项         │               │  3 项         │
         └──────┬───────┘               └──────┬───────┘
                │ 通过                           │ 通过
                ▼                               ▼
            「创建中」                         「待审核」
           （异步执行中）                （同步完成，不经创建中）
                │                               │
                ▼                               ▼
         ┌──────────────┐               ┌──────────────┐
         │  提交异步校验  │               │  提交异步校验  │
         │  7 项         │               │  5 项         │
         └──────┬───────┘               └──────┬───────┘
                │                               │
        ┌───────┴────────┐              详情页展示校验结果
    全部失败         成功/部分失败

    「创建失败」           「待审核」
                    （失败视频在详情页标注原因）
```

## 二、按作品创建 — 批量导入校验（14 项，全部同步）

> **触发时机**：点击「导入」→「确认」时**同步执行**，不通过则不入库，用户在导入弹窗内立即看到结果。  
> **文件格式**：xls / xlsx，模板字段：作品名称、CP名称、海外频道ID。

### 2.1 校验顺序与规则

**① 文件级校验**（整体拦截，不进入逐行校验）

| 校验项 | 规则 | 错误提示 |
|--------|------|---------|
| 导入模板格式 | 文件格式须符合系统标准模板；文件为空或无可校验数据时同样报错 | 导入模板错误 |
| 导入条数 | 单次导入 ≤ **500 条** | 导入失败，单次导入上限500条 |
| 文件大小 | 文件大小 ≤ **5 MB** | 导入失败，文件大小不超过5M |

**② 自动处理**（静默执行，无提示）

| 处理项 | 规则 |
|--------|------|
| 去除空行 | 去除所有空行数据，不计入条数 |
| 去除空格 | 所有字段首尾空格自动去除 |

**③ 数据行校验**（逐行执行，不通过则标记该行错误）

| # | 校验项 | 规则 | 错误提示 |
|:---:|--------|------|---------|
| 1 | 作品名称必填 | 【作品名称】不得为空 | 【作品名称】不能为空 |
| 2 | 批内重复 | 同一文件内「作品名称+CP名称+海外频道ID」组合不得重复；重复的所有行全部拦截（**含第一条**） | 【作品名称】存在重复行 |
| 3 | 作品名称存在性 | 【作品名称】须在系统中存在 | 【作品名称】不存在 |
| 4 | CP名称与作品匹配 | 【CP名称】**有值时**，【作品名称】须属于该 CP | 【CP名称】不存在 |
| 5 | 作品+CP唯一性 | 【CP名称】有值时，作品+CP 组合在系统中只能有一条生效中数据 | 【作品名称+CP】存在多条数据 |
| 6 | CP为空时作品唯一性 | 【CP名称】为空时，该作品名称在系统中只能有一条有效记录 | 【作品名称】存在多条数据 |
| 7 | 海外频道ID格式 | 【海外频道ID】**有值时**，格式须以 UC 开头、共 24 位；填写多个值时须用**顿号、中英文逗号**分隔 | 【海外频道ID】格式错误 |
| 8 | 海外频道ID系统存在性 | 格式校验通过后，ID须在系统中存在；不存在时提示具体不存在的 ID | 【海外频道ID】不存在：xxx |
| 9 | 发布通道校验 | **仅前 8 项全部无错时触发**；细则见下方说明 | 见下方细则 |

**发布通道校验细则**（第 9 项）：

| 海外频道ID | 校验逻辑 | 错误提示 |
|-----------|---------|---------|
| **为空** | 校验作品在系统中是否有绑定发布通道（type=COMPOSITION，status=IN_THE_CONTRACT） | 作品无绑定海外频道 |
| **有值** | 对格式+存在性均通过的频道逐个校验是否有发布通道且已配置 pipelineId；**至少一个频道通过则整行通过，全部失败才报错** | 【海外频道ID】未找到发布通道：xxx；或【海外频道ID】发布通道未配置pipelineId：xxx |

> ℹ️ 第 4~6 项：CP名称**允许为空**（无 CP 时跳过第 4、5 项，仅执行第 6 项）。  
> ℹ️ 第 7~8 项：海外频道ID**允许为空**（第 9 项发布通道校验会针对空频道情形单独处理）。  
> ℹ️ **导入阶段无异步校验项**：所有校验均在点击「确认」时同步完成；视频状态等深度校验在提交后异步执行（见 §3.4）。

### 2.2 导入结果处理

| 结果 | 系统提示 | 后续操作 |
|------|---------|---------|
| **全部通过** | ✅ 导入成功，共导入 N 条数据 | 弹窗关闭，数据展示在列表中 |
| **部分失败** | 导入成功 N 条，失败 M 条，请下载失败文件查看 | 提供【下载失败文件】（Excel，含失败原因列） |
| **全部失败** | 导入失败，请检查数据后重新导入 | 提供【下载失败文件】 |

> ℹ️ **多次导入**：同一任务单内第二次导入会**覆盖**第一次，清空前次数据，仅保留最新导入结果。

## 三、按作品创建 — 提交校验

### 3.1 同步校验（阻止提交）

```
                        用户点击「提交」
                                │
                                ▼
         ┌──────────────────────────────────────────────┐
         │  校验 1  基础信息完整性                        │
         │  必填：下架原因 / 处理方式 / 截止执行日期        │
         └──────────────────────┬───────────────────────┘
         不通过 ← ─ ─ ─ ─ ─ ─ ─ ┤ 通过
         红框标注缺失字段          │
         阻止提交                 │
                                 ▼
         ┌──────────────────────────────────────────────┐
         │  校验 2  作品数量 > 0                         │
         │  · 作品列表至少有 1 个作品                     │
         └──────────────────────┬───────────────────────┘
         不通过 ← ─ ─ ─ ─ ─ ─ ─ ┤ 通过
         提示"请至少添加一个作品"   │
         阻止提交                 │
                                 ▼
         ┌──────────────────────────────────────────────┐
         │  校验 3  每个作品的海外频道ID > 0              │
         │  · 每个作品须至少选择 1 个海外频道 ID           │
         └──────────────────────┬───────────────────────┘
         不通过 ← ─ ─ ─ ─ ─ ─ ─ ┤ 通过
         红框标注缺失频道的作品     │
         提示"请至少选择一个海外频道"│
         阻止提交                 │
                                 ▼
         ┌──────────────────────────────────────────────┐
         │  校验 4  任务单内重复性（自身重复校验）          │
         │  · 「作品名称 + 海外频道ID」组合在当前任务单内   │
         │    不得重复                                   │
         └──────────────────────┬───────────────────────┘
         不通过 ← ─ ─ ─ ─ ─ ─ ─ ┤ 通过
         提示"存在重复数据"        │
         阻止提交                 │
                                 ▼
                      同步校验全部通过，任务单提交成功
                      任务单进入「创建中」状态，异步校验开始
```

### 3.2 重复校验完整说明（4 种场景）

| 场景 | 校验时机 | 校验范围 | 校验内容 | 不通过结果 |
|------|---------|---------|---------|-----------|
| **初次提交 — 自身重复** | 提交时（**同步**） | 当前任务单自身 | 「作品名称+海外频道ID」在同一任务单内是否重复 | 阻止提交，提示"存在重复数据" |
| **初次提交 — 跨任务单重复** | 提交后（**异步**） | 所有执行中的其他任务单 | 「作品名称+海外频道ID」是否已存在于其他执行中任务单 | 详情页展示重复数据，任务单**继续创建** |
| **编辑后重新提交 — 自身重复** | 提交时（**同步**） | 编辑后的当前任务单自身 | 同上 | 阻止提交，提示"存在重复数据" |
| **编辑后重新提交 — 跨任务单重复** | 提交后（**异步**） | 其他执行中任务单（**排除当前任务单**） | 同上 | 详情页展示重复数据，任务单继续创建 |

> ℹ️ **"执行中"任务单定义**：状态为创建中、待审核、待处理、处理中（不含创建失败、审核拒绝、已完成）。

### 3.3 重复校验的 UI 交互

| 场景 | 交互说明 |
|------|---------|
| 导入后再打开选择弹窗 | 已导入的作品在弹窗中显示为**已勾选且不可取消** |
| 弹窗中选择新作品 | 新选作品与已导入作品**共存**，追加至列表 |
| 多次导入 | 第二次导入**覆盖**第一次，清空上次数据 |
| 提交时发现跨任务单重复 | 弹窗提示，用户可选择**跳过重复数据继续提交**或**取消提交** |

### 3.4 异步校验（后台执行，详情页展示结果）

> 任务单进入「创建中」后，后台异步执行以下校验，结果写入 detail。

| 异步项 | 校验内容 | 失败原因 | 说明 |
|--------|---------|----------|------|
| 异步 1 | 作品+CP 系统存在性 | 作品不存在 | |
| 异步 2 | 作品绑定海外频道 | 作品无绑定海外频道 | 未指定频道时，自动查作品绑定频道 |
| 异步 3 | 发布通道存在性 | 未找到发布通道 | 含合同生效状态校验 |
| 异步 4 | 发布通道已配置 pipelineId | 发布通道未配置 pipelineId | |
| 异步 5 | 视频已有执行中任务 | 视频已有执行中任务 | 批量检查 |
| 异步 6 | 视频已通过删除任务下架 | 视频已删除，无法再处理 | 批量检查 |
| 异步 7 | 视频已通过转私任务处理 | 视频已转私，无法再处理 | 仅处理方式=转私时校验；批量检查 |

**异步结果汇总：**
- `videoStatus = FAILED`：业务校验失败，**不重试**
- `videoStatus = EXPAND_FAILED`：网络/系统异常，等待补偿任务重试
- 全部 detail 均为 FAILED → 任务状态变为「**创建失败**」（可编辑后重建）
- 部分或全部成功 → 任务状态变为「**待审核**」（失败作品在详情页标注原因）

### 3.5 相同内容任务单冲突场景

> 场景：第一个任务单仍在执行中，用户提交与其**完全相同内容**的第二个任务单。

```
第二个任务单提交（内容与第一个完全相同）
       │
       ▼ 同步校验（仅校验自身，不查跨任务单）
    通过 → 进入「创建中」
       │
       ▼ 异步校验 — 跨任务单重复校验
       │
       │ 发现所有「作品+频道」组合均与第一个任务单冲突
       │
  ┌────┴──────────────────────────┐
所有作品全部冲突                部分作品冲突
  （无可执行视频ID）
       │                              │
       ▼                              ▼
  「创建失败」                    「待审核」
  详情页展示每个                 冲突作品标注失败原因
  作品的失败原因                 其余作品正常审核流转
```

| 维度 | 说明 |
|------|------|
| 第一个任务单状态 | 创建中 / 待审核 / 待处理 / 处理中 → **均视为执行中，触发冲突拦截** |
| 第一个任务单状态 | 创建失败 / 审核拒绝 / 已完成 → **不视为执行中，不触发冲突** |
| 拦截时机 | **异步**（第二个任务单提交后），用户已能看到任务单进入创建中 |
| 用户感知 | 提交成功 → 进入创建中 → 详情页显示冲突失败 → 可能变为创建失败 |

## 四、按视频ID创建 — 批量导入校验（15 项，全部同步）

> **触发时机**：点击「导入」→「确认」时**同步执行**，不通过则不入库，用户在导入弹窗内立即看到结果。  
> **文件格式**：xls / xlsx，模板字段：视频ID、作品名称、CP名称、海外频道ID。  
> **前置条件**：须先填写归属信息（任务来源 / 分销商名称 / 运营团队），否则点击【批量导入】时提示"请先填写归属信息"。

### 4.1 校验顺序与规则

| 序号 | 校验项 | 校验规则 | 错误提示 | 处理方式 |
|:---:|--------|---------|---------|---------|
| 1 | 导入模板格式 | 文件格式须符合系统标准模板；文件为空或无可校验数据时同样报错 | 导入模板错误 | 阻止导入 |
| 2 | 导入条数 | 单次导入 ≤ **1000 条** | 导入失败，单次导入上限1000条 | 阻止导入 |
| 3 | 文件大小 | 文件大小 ≤ **5 MB** | 导入失败，文件大小不超过5M | 阻止导入 |
| 4 | 去除空行 | 去除所有空行数据，不计入条数 | 无提示 | **自动处理** |
| 5 | 去除空格 | 所有字段首尾空格自动去除 | 无提示 | **自动处理** |
| 6 | 视频ID必填 | 【视频ID】不得为空 | 【视频ID】不能为空 | 标记该行错误 |
| 7 | 作品名称必填 | 【作品名称】不得为空 | 【作品名称】不能为空 | 标记该行错误 |
| 8 | 海外频道ID必填 | 【海外频道ID】不得为空 | 【海外频道ID】不能为空 | 标记该行错误 |
| 9 | 视频ID格式 | 【视频ID】须为固定 **11 位字符**的 YouTube 视频 ID 格式 | 【视频ID】格式错误 | 标记该行错误 |
| 10 | 视频ID重复性 | 同一导入文件内【视频ID】不得重复；重复的所有行全部拦截（**含第一条**） | 【视频ID】存在重复行 | 标记重复行 |
| 11 | 作品名称存在性 | 【作品名称】须在系统中存在 | 【作品名称】不存在 | 标记该行错误 |
| 12 | CP名称存在性 | 【CP名称】**有值时**，【作品名称】须在该 CP 下存在 | 【CP名称】不存在 | 标记该行错误 |
| 13 | 海外频道ID系统存在性 | 【海外频道ID】须在系统中存在 | 【海外频道ID】不存在 | 标记该行错误 |
| 14 | 权限校验 | 频道存在时按 taskSource 校验操作权限（见下方细则）；无权限时提示同频道不存在 | 【海外频道ID】不存在 | 标记该行错误 |
| 15 | 视频ID在频道内存在性 | 【视频ID】须存在于对应【海外频道ID】内 | 【视频ID】不存在 | 标记该行错误 |
| 16 | 视频ID执行中任务 | 【视频ID】不得已存在于其他**执行中**任务单（创建中/待审核/待处理/处理中）| 【视频ID】已存在执行中的任务单 | 标记该行错误，**拦截导入** |

**权限校验细则**（第 14 项，频道存在且 taskSource 非空时触发）：

| taskSource | 校验逻辑 |
|-----------|---------|
| DISTRIBUTOR | 通过 `composition_allocate` 校验分销商是否有该作品的分配权；compositionId 无法解析时**不报错** |
| OPERATION_TEAM | 通过 `sign_channel.operation_team` 校验频道是否归属该运营团队；channelId 为 null 或 SignChannel 不存在时**视为有权限**（容错） |

> ℹ️ 第 13 项：仅校验频道 ID 是否在系统中存在；第 14 项权限校验在频道存在后紧接执行，无权限时提示同频道不存在（`【海外频道ID】不存在`）。  
> ℹ️ **导入阶段无异步校验项**：所有 16 项均在点击「确认」时同步完成；视频状态等深度校验在提交后异步执行（见 §5.2）。

### 4.2 第 15 项：执行中任务冲突详细说明

| 维度 | 说明 |
|------|------|
| **"执行中"定义** | 任务单状态为：创建中、待审核、待处理、处理中 |
| **不触发拦截** | 任务单状态为：创建失败、审核拒绝、已完成 |
| **拦截时机** | 导入阶段（点击「确认」时），直接标记冲突行错误，不入库 |
| **用户感知** | 导入时即被拦截，该视频ID无法进入列表，提交阶段感知不到此问题 |

```
相同视频ID已在第一个任务单中（执行中）
用户尝试在第二个任务单中导入相同视频ID
              │
              ▼ 导入校验第 15 项
       发现冲突 → 标记该行错误
              │
              ▼
  仅冲突行被拦截，其余行正常导入
              │
              ▼
  用户可选择：
  ① 删除冲突行后导入剩余数据
  ② 等第一个任务单完成/失败后再导入
```

### 4.3 特殊场景：历史任务视频的二次处理

| 场景 | 是否允许重新导入 | 说明 |
|------|:---:|------|
| 视频已在历史任务单中**成功处理（删除）** | ❌ 不允许 | 视频已被删除，无法再处理 |
| 视频在历史任务单中**处理失败** | ✅ **允许** | 失败视频可重新导入，创建新任务单执行 |
| 视频在历史任务单中**处理成功（私享）** | ❌ 不允许 | 已成功处理，无法重复下架（执行校验②③拦截） |

### 4.4 导入结果处理

| 结果 | 系统提示 | 后续操作 |
|------|---------|---------|
| **全部通过** | ✅ 导入成功，已添加 N 条数据 | 弹窗关闭，按【作品名称+CP名称】分组显示 |
| **部分失败** | 导入成功 N 条，失败 M 条，请下载失败文件查看 | 提供【下载失败文件】（含失败原因列） |
| **全部失败** | 导入失败，请下载识别文件后按失败原因调整 | 提供【下载失败文件】 |

> ℹ️ **多次导入**：第二次导入**覆盖**第一次，清空前次数据，仅保留最新导入结果。

## 五、按视频ID创建 — 提交校验

### 5.1 同步校验（阻止提交）

```
                        用户点击「提交」
                                │
                                ▼
         ┌──────────────────────────────────────────────┐
         │  校验 1  归属信息完整性                        │
         │  · 任务来源（必填）                            │
         │  · 分销商名称（任务来源=分销商时必填）            │
         │  · 运营团队（任务来源=运营团队时必填）            │
         │  · 下架原因 / 处理方式 / 截止执行日期（必填）     │
         └──────────────────────┬───────────────────────┘
         不通过 ← ─ ─ ─ ─ ─ ─ ─ ┤ 通过
         红框标注缺失字段          │
         阻止提交                 │
                                 ▼
         ┌──────────────────────────────────────────────┐
         │  校验 2  视频ID格式                           │
         │  · 列表中所有视频 ID 须为 11 位字符格式          │
         └──────────────────────┬───────────────────────┘
         不通过 ← ─ ─ ─ ─ ─ ─ ─ ┤ 通过
         标注格式错误的视频 ID     │
         阻止提交                 │
                                 ▼
         ┌──────────────────────────────────────────────┐
         │  校验 3  视频数量 > 0                         │
         │  · 列表中至少有 1 个视频 ID                    │
         └──────────────────────┬───────────────────────┘
         不通过 ← ─ ─ ─ ─ ─ ─ ─ ┤ 通过
         提示"请至少添加一个视频"   │
         阻止提交                 │
                                 ▼
                      同步校验全部通过，任务单提交成功
                      提示"提交成功，若提交信息中存在校验失败结果时可前往详情页查看"
                      任务单直接进入「待审核」状态
```

> ℹ️ 按视频ID创建时，视频ID的存在性和执行中任务冲突已在**导入阶段**（第14、15项）完成校验，提交同步阶段不再重复检查。

### 5.2 异步校验（后台执行，详情页展示）

> 按视频ID创建时，异步校验作为补充校验层（主要校验已在导入时完成）。

| 异步项 | 校验内容 | 失败原因 | 说明 |
|--------|---------|----------|------|
| 异步 1 | 作品+CP 系统存在性 | 作品不存在 | |
| 异步 2 | 发布通道存在性及权限 | 无权限或未找到发布通道 | `DISTRIBUTOR`：校验 composition_allocate 分配权；`OPERATION_TEAM`：校验 sign_channel.operation_team |
| 异步 3 | 视频已有执行中任务 | 视频已有执行中任务 | 兜底校验，导入阶段已拦截大部分 |
| 异步 4 | 视频已通过删除任务下架 | 视频已删除，无法再处理 | |
| 异步 5 | 视频已通过转私任务处理 | 视频已转私，无法再处理 | 仅处理方式=转私时校验 |

**异步结果汇总：**
- `videoStatus = FAILED`：业务校验失败，**不重试**
- `videoStatus = EXPAND_FAILED`：网络/系统异常，等待补偿任务重试

### 5.3 相同内容任务单冲突场景

> 场景：第一个任务单仍在执行中，用户尝试提交内容完全相同的第二个任务单（按视频ID创建）。

```
第二个任务单 — 导入相同视频ID
       │
       ▼ 导入校验第 15 项（执行中任务校验）
       │
       │ 发现所有视频ID均已在第一个任务单中
       │
  冲突行全部被标记错误，无法导入
       │
       ▼
  视频列表为空 → 无法点击「提交」
  （第二个任务单实际上无法创建成功）
```

| 维度 | 说明 |
|------|------|
| 拦截时机 | **导入阶段**即被拦截，比按作品创建早两个阶段 |
| 用户感知 | 导入时立即看到错误，任务单无法进入提交流程 |
| 与按作品创建对比 | 按作品创建在**异步阶段**才感知冲突；按视频ID创建在**导入阶段**即被阻止 |

## 六、执行中任务定义（统一说明）

> 本定义适用于第二节~第五节所有涉及"执行中任务"的校验项。

| 任务单状态 | 是否属于执行中 | 说明 |
|-----------|:---:|------|
| 创建中 | ✅ 是 | 异步获取视频ID进行中 |
| 待审核 | ✅ 是 | 等待人工审核 |
| 待处理 | ✅ 是 | 审核通过，在执行队列排队 |
| 处理中 | ✅ 是 | 正在执行下架操作 |
| 创建失败 | ❌ 否 | 终态，视频未执行 |
| 审核拒绝 | ❌ 否 | 终态，视频未执行 |
| 已完成 | ❌ 否 | 终态，执行已结束 |

## 七、两种创建方式校验对比

| 对比项 | 按作品创建 | 按视频ID创建 |
|--------|-----------|-------------|
| 批量导入上限 | **500 条** | **1000 条** |
| 导入模板字段 | 作品名称、CP名称、海外频道ID | **视频ID**、作品名称、CP名称、海外频道ID |
| 海外频道ID是否必填（导入） | 允许为空（导入时校验发布通道绑定情况） | **必填** |
| 重复任务拦截时机 | **异步**（提交后创建中阶段） | **导入阶段**（最早拦截） |
| 提交后任务单状态 | 先进入「**创建中**」（异步获取视频ID） | 直接进入「**待审核**」（同步完成） |
| 同步校验重点 | 作品数量 / 频道选择 / 任务单内重复 | 归属信息完整性 / 视频ID格式 / 视频数量 |
| 发布通道/权限校验时机 | **导入同步**（第9项，含频道为空时绑定校验）+ **提交异步**（兜底） | **导入同步**（第14项，DISTRIBUTOR/OPERATION_TEAM分支）+ **提交异步**（兜底） |
| 异步校验重点 | 作品+CP存在性 + 频道绑定 + 发布通道（含合同状态+pipelineId）+ 执行中/已删除/已转私（**7项**） | 作品+CP存在性 + 发布通道存在性+权限（DISTRIBUTOR/OPERATION_TEAM细分）+ 执行中/已删除/已转私（**5项**，兜底） |
| 批量导入项数 | **14 项** | **16 项** |

## 八、校验触发时机汇总

| 触发时机 | 校验类型 | 不通过结果 | 适用范围 |
|---------|---------|-----------|---------|
| 点击「导入」→「确认」 | 批量导入校验（格式/条数/字段/存在性） | 错误行标记，不入库；可下载失败文件 | 两种创建方式均有 |
| 点击「提交」（立即响应） | **同步校验** | 红框/提示，**阻止提交** | 两种创建方式均有 |
| 提交时发现跨任务单重复 | 重复校验弹窗 | 弹窗提示，用户选择**跳过**或**取消** | 按作品创建 |
| 提交后后台异步运行 | **异步校验** | 详情页展示失败原因，任务单继续流转 | 两种创建方式均有 |
| 异步完成后汇总 | 结果聚合 | 全部失败 → 创建失败；部分/全部成功 → 待审核 | 按作品创建 |
