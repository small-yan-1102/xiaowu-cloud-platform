# 【授权到期后通知分销商并下架视频】调研报告

### 一、背景与现状

1.  调研背景：
    
    1.  当前业务SOP中要求当版权方授权到期解约后内外部运营需终止相关片单的发布并对历史发布视频进行下架处理。
        
    2.  为确保已到期/解约片单可以准确通知到内外部运营并确保已发布的视频全面且快速下架，需要与系统内场景结合起来；
        
2.  调研目的：
    
    1.  梳理业务流程，将指向【视频下架】的多业务场景进行梳理和汇总
        
    2.  串连线上场景，把业务流程与系统场景相结合；减少人工作业的成本

### 二、痛点分析

1.  **依赖人工协同，效率低且易出错‌**
    
    1.  多个关键动作依赖钉钉OA、钉钉通知传递信息（如客服通知分销商下架、总编室同步运营下架），缺乏系统级联动机制。
        
    2.  信息传递链路过长，存在漏传、误读、延迟执行风险。例如，OA流程未与视频ID自动绑定，运营需手动匹配片单与通道视频ID，操作繁琐且易错。
        
2.  **系统能力割裂，缺乏统一触发与执行入口‌**
    
    1.  当前解约流程分散在多个系统（OA、CRM、AMS），但各系统间无自动触发接口
        
    2.  “发起解约”与“下架视频”动作孤立，未形成以片单为核心的数据联动链路‌
        
    3.  缺少一个集中管理的【视频处理看板】，无法统一查看“哪些片单需下架、下架进度如何、涉及哪些海外频道、版权方/分销商
        
3.  **下架动作无标准化流程，内外部协同混乱‌**
    
    1.  临时下架需求来自非结构化通知（如钉钉消息、企微通知），总编室需手动拆解并分发任务，响应速度慢。
        
    2.  内部运营与外部分销商处理动作不一致，缺乏统一指令出口，存在“内部已下架、外部仍可播放”的合规漏洞
        

### 三、优化方案

为解决上述问题，将构建一套以「版权解约下架任务中心」为核心的全链路自动化系统，打通多系统数据链路，实现任务自动生成、流程自动触发、操作自动执行、结果自动校验、风险自动预警，最终达成“减少人工操作、实现快速响应、全流程合规可追溯”的目标，降低因下架不及时不全面带来的版权风险

| 场景 | 触发节点 | 自动化动作 |
| --- | --- | --- |
| 场景1：上游版权解约 | 1、发起解约流程时，判断【是否下架视频】=是<br>2、AMS发起内容解约单（待解约流程线上化后，分销商解约调整为自动解约） | 1、向运营系统、分销商发送【下架视频】 的指令<br>2、系统内发起解约单 → 自动创建【视频处理任务】，并关联对应片单与视频ID |
| 场景2：分销商解约 | AMS发起内容解约单（当前流程，待分销商合约入系统后调整） | 系统内发起解约单 → 自动创建【视频处理任务】，并关联对应片单与视频ID |
| 场景3：临时下架需求 | \- | AMS、运营系统可手动发起【视频处理任务】 |

1.  覆盖场景
    
    1.  上游版权解约场景：CRM监测到合约到期/AMS发起解约单，需下架内外部全渠道视频
        
    2.  分销商解约场景：分销商合作终止，需下架该分销商渠道下的授权视频；
        
    3.  临时调整场景：突发版权纠纷、上线时间变更、内容整改等，需紧急下架或调整视频私享状态。
        
2.  核心设计原则
    
    1.  自动化优先：核心业务动作100%系统自动触发，仅保留必要人工审批与异常干预环节；
        
    2.  数据全联动：打通CRM、AMS、运营系统、客服系统、YouTube Data API等数据通道，实现信息无孤岛流转
        
    3.  流程标准化：统一内外部操作规范，明确各角色职责与任务节点要求
        
    4.  风险可管控：全链路操作留痕，建立实时合规监测与异常预警机制
        
    5.  应急高响应：分级任务优先级，为高风险场景开通快速处理通道
        
3.  核心功能设计
    
    1.  建立统一的【视频处理】，实现流程集中管控；  
        功能定位：作为所有解约类视频处理操作的统一入口与调度中枢，聚合来自CRM、AMS等系统的解约请求，按通道维度生成可执行任务。同时支持手动触发临时下架任务
        
        1.  自动触发任务
            
            1.  上游版权解约：CRM合约库实时推送到期/解约片单数据，系统自动关联片单下的视频ID、发布渠道、分销商信息、CID状态，生成标准化下架任务，并根据合约风险等级自动打标优先级（P1级）；
                
            2.  分销商解约：AMS发起分销商作品解约单后，系统自动匹配该分销商对应的所有授权片单与视频，生成渠道定向下架任务，标记优先级为P2级；
                
        2.  手动触发任务
            
            1.  针对临时调整场景，总编室可在任务中心手动发起任务，支持上传下架文件、自定义任务优先级（P0/P3）、设置截止时间、选择处理方式（下架/私享），系统自动关联对应视频与渠道信息；
                
    2.  全流程可视化管控模块
        
        1.  任务状态全景展示：任务列表按优先级排序，清晰展示任务类型、关联片单、涉及渠道、处理进度、截止时间、当前状态（待开始/处理中/已完成/异常）
            
        2.  异常数据下载：任务超时未处理、下架失败的数据支持一键重试、下载失败原因
            
    3.  预设审批规则
        
        1.  解约推送的下架任务：系统自动审批，无需人工干预；
            
        2.  手动上传的下架任务：需总编室单人审批；

### 五、实施路径

1.  第一阶段：（分为两期实现，第一期：提供手动创建【视频处理】入口；打通分销商解约单自动发起视频处理任务单）
    
    1.  创建【视频处理】入口，支持手动发起【视频下架任务单】
        
    2.  识别AMS的【解约单-解约类型】，自动触发【视频下架任务单】：
        
        1.  自动识别解约单解约类型（上游版权解约 / 分销商解约 ）
            
        2.  【发布通道】自动创建视频处理任务单，统计该通道下的所有视频
            
        3.  接入审批，审批状态：审批中、审批拒绝、审批撤回、审批通过
            
        4.  展示任务状态：待处理 →处理中 → 已完成 / 异常
            
            1.  单视频处理，异常【无权限、不存在、接口报错等原因】不做重试，支持下载视频处理失败明细、失败原因
                
        5.  展示处理进度：视频执行进度、频道频道进度、任务单执行进度
            
        6.  支持按优先级规则进行排队（如“执行时间、截止时间”、“下架原因”）
            
            1.  截止时间+下架原因+创建时间，多策略为且的关系进行排序
                
            2.  预警机制：单视频下架调用接口，一次消耗50配额，单日视频处理上限5000；需根据当天预处理的视频数与配额额度进行比对，临近临界值时，需告警人工介入处理【技术提供预警机制】
                
        7.  打通AMS和分销、运营系统对【下架或私享】接口的调用
            
    3.  支持运营系统创建视频处理任务单，执行内部运营的视频下架
        
2.  第二阶段：OA解约流程线上化，打通版权方解约的内外部视频下架处理
    
3.  第三阶段：分销商合约管理线上化（签约、解约、补充、重签、续签），打通分销商解约的外部运营视频下架处理
