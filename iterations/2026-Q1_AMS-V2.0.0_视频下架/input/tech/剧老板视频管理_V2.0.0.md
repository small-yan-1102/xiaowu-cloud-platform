## V2.0.0 迭代计划

### Quest 1: 视频下架列表页

**产品原文**：

> **数据权限：** 分配该作品的运营人员及拥有该运营人员管理范围的人员可见
>
> **展示维度：** 按作品+频道维度展示，仅展示已完成状态的数据
>
> **展示规则：**
> - 一个作品在一个频道下的下架任务为一条数据
> - 同一作品在多个频道下架时，列表展示多条数据（每个频道一条）
> - 示例：作品"恶魔之吻"在3个频道下架，列表展示3条数据
>
> **数据推送规则：**
> - 推送触发时机：任务单状态 = 已完成时
> - 推送匹配规则：按作品创建/解约单创建：依据通道关系记录中的【作品+海外频道】归属的分销商信息；按视频ID创建：依据海外频道分配的分销商
> - 推送维度：作品+海外频道维度
> - 推送目标：对应剧老板账号

**原型图解读**：

原型图位于 `视频下架原型_v2.html` 第 1338-1431 行，提供了以下界面设计：

- **页面提示**：💡 提示：剧老板端仅展示已完成状态的数据，按作品+频道维度展示
- **搜索栏**：5 个筛选条件横向排列（作品名称、频道名称、频道ID、视频ID、下架原因）
- **表格列**：作品名称、频道名称/ID、下架视频数、下架原因、运营人员、完成时间、操作
- **下架视频数**：支持鼠标悬停显示视频ID明细（Tooltip 形式）
- **操作列**：查看详情（链接样式）

**需求解读**：

- 剧老板端列表与资产管理系统列表有本质区别：按**作品+频道维度**展示，而非任务单维度
- 仅展示**已完成状态**的数据，不涉及创建、审核等流程
- 数据来源：资产管理系统任务单完成后自动推送
- 权限控制：仅展示分配给当前用户的作品数据

**前端设计**：

- **布局**：
  - 使用 AppView 作为页面容器
  - 顶部提示信息（nav-tip 样式）
  - 搜索区域使用 SearchForm 组件，5 个筛选条件
  - 表格使用 SuperTable 组件
  - 底部分页使用 v-pagination 组件

- **交互**：
  - 搜索条件支持交叉查询
  - 点击「重置」清空所有条件
  - 下架视频数支持鼠标悬停显示视频ID明细（使用 OverflowTooltip 或自定义 Tooltip）
  - 点击「查看详情」打开进度查看 Drawer

- **校验**：
  - 无表单提交，无需校验规则

- **修改位置**：
  - `src/views/resourceManagement/videoManage/List.vue`：需要重构，改为按作品+频道维度展示
  - `src/mock/AMS_V2.0.0/data/taskList.js`：需要重构 Mock 数据结构
  - `src/mock/AMS_V2.0.0/data/enums.js`：需要添加下架原因枚举

**验收标准**：

- ✓ 列表按作品+频道维度展示，仅显示已完成状态的数据
- ✓ 搜索条件支持作品名称（模糊）、频道名称（模糊）、频道ID（精准）、视频ID（精准）、下架原因筛选
- ✓ 支持交叉查询和重置功能
- ✓ 下架视频数支持鼠标悬停显示视频ID明细
- ✓ 表格列包含：作品名称、频道名称/ID、下架视频数、下架原因、运营人员、完成时间、操作
- ✓ 点击「查看详情」可打开进度查看页

### Quest 2: 进度查看页（Drawer 形式）

**产品原文**：

> **页面说明：**
> - 剧老板进度查看页为**单作品维度**展示（区别于资产管理系统的多作品维度）
> - 展示当前剧老板有权限查看的单个作品的视频下架进度
>
> **展示内容：**
> - 作品基本信息
> - 视频明细列表
>
> **视频明细列表字段：**
> | 字段 | 说明 |
> |-----|------|
> | 序号 | 自增序号 |
> | 视频ID | 视频唯一标识 |
> | 视频标题 | 视频标题 |
> | 海外频道名/ID | 海外频道名称和ID |
> | 下架状态 | 已完成/处理中/待处理/处理失败 |
> | 完成时间 | 下架完成时间 |
> | 失败原因 | 失败时显示具体原因 |
>
> **注：** 不展示【整体完成进度】模块

**原型图解读**：

原型图中 `boss-progress` Drawer 展示了进度查看页的设计：

- **Drawer 宽度**：900px（与资产管理系统详情页一致）
- **标题**：作品名称 + 频道名称/ID
- **内容**：视频明细列表（表格形式）
- **无整体完成进度模块**：区别于资产管理系统详情页

**需求解读**：

- 剧老板进度查看页是**单作品维度**，不涉及多作品切换
- 不展示整体完成进度模块（进度条、统计卡片等）
- 仅展示视频明细列表，字段与资产管理系统类似
- 下架状态包含：已完成、处理中、待处理、处理失败

**前端设计**：

- **布局**：
  - 使用 BasicDrawer 组件（宽度 900px）
  - 标题格式：作品名称 + 频道名称/ID
  - 内容区域：视频明细表格

- **交互**：
  - 从列表页点击「查看详情」打开 Drawer
  - 表格支持排序（按下架状态、完成时间）
  - 无导出功能（区别于资产管理系统）

- **校验**：
  - 无表单提交，无需校验规则

- **修改位置**：
  - 新建 `src/views/resourceManagement/videoManage/ProgressDrawer.vue`
  - `src/mock/AMS_V2.0.0/api.js`：添加获取进度详情接口

**验收标准**：

- ✓ Drawer 宽度为 900px，标题显示作品名称和频道名称/ID
- ✓ 视频明细列表包含：序号、视频ID、视频标题、海外频道名/ID、下架状态、完成时间、失败原因
- ✓ 不展示整体完成进度模块
- ✓ 下架状态使用 StatusTag 组件展示（已完成-绿色、处理中-蓝色、待处理-橙色、处理失败-红色）
- ✓ 失败原因仅在处理失败状态时显示
