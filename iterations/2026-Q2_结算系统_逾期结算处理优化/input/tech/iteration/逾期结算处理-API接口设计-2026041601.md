## 一、全局约定

### 1.1 基础信息

| 项目 | 值 |
|------|---|
| **Base URL** | `/videoCompositionOverdue` |
| **Content-Type** | `application/json` |
| **字符编码** | UTF-8 |

### 1.2 统一响应格式

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

**错误响应**：

```json
{
  "code": 500,
  "message": "错误信息",
  "data": null
}
```

**分页响应**（data 部分）：

```json
{
  "records": [ ... ],
  "total": 100,
  "current": 1,
  "size": 20,
  "pages": 5
}
```

### 1.3 分页约定

| 参数名 | 类型 | 传参位置 | 说明 | 默认值 |
|--------|------|----------|------|--------|
| `pageNum` | `int` | Body | 当前页码（从 1 开始） | `1` |
| `pageSize` | `int` | Body | 每页条数 | `20` |

### 1.4 认证方式

| 项目 | 值 |
|------|---|
| **认证方式** | Sa-Token Session |
| **注解** | `@NeedLogin` |

> 所有接口均需认证。

### 1.5 错误码定义

| 场景 | HTTP 状态码 | message 示例 |
|------|-----------|-------------|
| 参数校验失败 | 200 (code=500) | "OSS文件键不能为空" |
| 业务异常 | 200 (code=500) | "请选择相同状态的记录进行拆分" |
| 系统异常 | 200 (code=500) | "系统异常，请联系管理员" |

#### 2.1.2 批量拆分 `[修改]`

| 项目 | 内容 |
|------|------|
| **接口方法** | `batchSplit` |
| **HTTP 方法** | `POST` |
| **完整路径** | `/videoCompositionOverdue/batchSplit` |
| **认证要求** | 需认证 |

**Request Body**：

| 字段名 | 类型 | 必选 | 校验规则 | 说明 |
|--------|------|------|----------|------|
| (body) | `List<Long>` | Y | 非空 | 勾选记录的主键ID列表 |

**请求示例**：

```http
POST /videoCompositionOverdue/batchSplit
Content-Type: application/json

[123456, 123457, 123458]
```

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

**错误响应**：

| 触发场景 | message |
|----------|---------|
| 未选择记录 | "请选择需要拆分的记录" |
| 记录不存在或已拆分 | "选中记录状态不正确" |
| **[V4.5新增]** 混合状态拆分 | "请选择相同状态的记录进行拆分" |
| 冲销报表未到账 | "冲销报表未到账，无法拆分" |
| pipelineId 已存在 | "通道已拆分，请勿重复操作" |

#### 2.1.3 异步导出 `[修改]`

| 项目 | 内容 |
|------|------|
| **接口方法** | `asyncExport` |
| **HTTP 方法** | `POST` |
| **完整路径** | `/videoCompositionOverdue/export/async` |
| **认证要求** | 需认证 |

**Request Body**：

同分页列表查询参数（`VideoCompositionOverdueAsyncQuery`），新增 id offset 分页字段。

**请求示例**：

```http
POST /videoCompositionOverdue/export/async
Content-Type: application/json

{
  "status": 2,
  "receiptedMonthStart": "2026-01",
  "receiptedMonthEnd": "2026-03"
}
```

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "taskId": 789
  }
}
```

**V4.5 导出字段变更说明**：

| Tab (status) | 新增导出列 | 说明 |
|-------------|-----------|------|
| 未登记 (0) | 技术漏抓 | "是"/"否" |
| 逾期登记未拆分 (1) | 技术漏抓 | "是"/"否" |
| 跨期正常未拆分 (3) **[新增Tab]** | 技术漏抓 | 字段与逾期Tab一致 |
| 已拆分 (2) | 原状态, 技术漏抓 | 原状态："逾期登记"/"跨期正常"/""（历史）|

#### 2.1.4 异步导入 `[修改]`

| 项目 | 内容 |
|------|------|
| **接口方法** | `asyncImport` |
| **HTTP 方法** | `POST` |
| **完整路径** | `/videoCompositionOverdue/import/async` |
| **认证要求** | 需认证 |

**Request Body**：

| 字段名 | 类型 | 必选 | 校验规则 | 说明 |
|--------|------|------|----------|------|
| `ossKey` | `String` | Y | @NotBlank | OSS文件键 |
| `fileName` | `String` | N | | 原始文件名 |
| `receiptedMonth` | `String` | Y | @NotBlank, yyyy-MM | 到账月份 |

**请求示例**：

```http
POST /videoCompositionOverdue/import/async
Content-Type: application/json

{
  "ossKey": "import/2026/04/abc123.xlsx",
  "fileName": "逾期视频导入.xlsx",
  "receiptedMonth": "2026-03"
}
```

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

**错误响应**：

| 触发场景 | message |
|----------|---------|
| 参数为空 | "OSS文件键不能为空" / "到账月份不能为空" |
| 文件格式错误 | "文件格式错误，请使用标准模板" |
| 超过行数限制 | "导入数据超过最大限制(20000行)" |

**V4.5 导入逻辑变更说明**：

新增 R1~R6 校验逻辑（详见 `02-逾期结算处理-详细设计.md` § 3.1）。
按频道维度执行校验，失败频道记入失败文件。新增 videoTag 计算。
