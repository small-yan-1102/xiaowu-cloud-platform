> **冒烟套件**：`testcase/suite_smoke.md`（7 条用例）

## 一、核心流程（纳入冒烟）

> 判定标准：核心价值链 / 资金状态不可逆变更 / 跨模块数据流入口 → 系统不可用则阻塞部署。

### 1.1 导入（SET-03）

| 冒烟用例 | 验证内容 | 判定理由 |
|---------|---------|---------|
| SMOKE-OVERDUE-001 | 导入 Excel → 异步完成 → 列表出现新记录 | 数据进入系统的唯一入口，缺失则后续所有流程无数据可处理 |
| SMOKE-OVERDUE-EX-001 | 同月重复导入被拒绝（任务互斥） | 幂等屏障，防止重复导入导致数据污染 |

**覆盖的测试点**：TP-S03-001, TP-S03-022, TP-S03-032

### 1.2 MQ 登记同步（SET-01）

| 冒烟用例 | 验证内容 | 判定理由 |
|---------|---------|---------|
| SMOKE-OVERDUE-003 | MQ 消费 → status 0→1（逾期登记） | 跨系统数据流入口（剧老板→结算），状态变更不可逆 |
| SMOKE-OVERDUE-004 ⚠️ | MQ 消费 → status 0→3（跨期正常） | **本期核心新增路径**，修复状态机死锁问题 |

**覆盖的测试点**：TP-S01-001, TP-S01-002, TP-S01-004, TP-S01-013

### 1.3 批量拆分（SET-02）

| 冒烟用例 | 验证内容 | 判定理由 |
|---------|---------|---------|
| SMOKE-OVERDUE-002 | 逾期登记未拆分 Tab → 拆分 → 已拆分 | 资金结算核心操作，涉及冲销表子集写入 |
| SMOKE-OVERDUE-EX-002 | 已存在未结算子集时拒绝重复拆分 | 幂等屏障，防止冲销表子集重复写入 |
| SMOKE-OVERDUE-005 ⚠️ | 跨期正常未拆分 Tab → 拆分 → 已拆分（原状态=跨期正常） | **本期核心新增 Tab**，验证 status=3→2 流转 |

**覆盖的测试点**：TP-S02-003, TP-S02-004, TP-S02-007, TP-S02-016, TP-S02-029, TP-S02-031

### 1.4 冒烟覆盖汇总

```
数据入口          状态流转                    财务处理
─────────       ──────────────            ──────────
导入 Excel  ──→  status=0（未登记）
  │                  │
  │            MQ 同步（逾期）──→ status=1 ──→ 批量拆分 ──→ status=2（已拆分）
  │            MQ 同步（跨期）──→ status=3 ──→ 批量拆分 ──→ status=2（已拆分，原状态=跨期正常）
  │
  └─ 幂等拦截（同月重复导入）
                                    └─ 幂等拦截（未结算子集重复拆分）
```

**7 条用例 = 3 段主干（导入/MQ同步/拆分）× 2 条路径（逾期+跨期正常）+ 2 个幂等屏障**

## 二、非核心流程（不纳入冒烟）

> 判定标准：系统缺失时仍可基本使用 / 属于展示辅助 / 属于校验规则细节。归入 E2E 或专项测试。

### 2.1 导入校验规则链路（SET-03）— E2E / P0-P1

| 流程 | 说明 | 不纳入原因 | 建议层级 |
|------|------|-----------|---------|
| R1 占比完整性校验 | 导入视频占比求和 ≠ 冲销报表占比总和 → 阻断 | 校验规则，缺失时导入仍可完成（只是精度不对） | E2E / P0 |
| R2 零值清理 | ROUND(占比×CMS收益, 2)=0.00 → 舍弃 | 精度处理细节 | E2E / P1 |
| R3 误差抹平 | 差额 ≤$1 → 累加到占比最大视频 | 精度处理细节 | E2E / P0 |
| R4 阈值阻断 | 差额 >$1 → 阻断该频道 | 安全兜底规则 | E2E / P0 |
| R5 全零阻断 | 全维度取整后均为0.00 → 阻断（V4.5 新增） | 极端边界场景 | E2E / P1 |
| R6 美国/新加坡收益独立处理 | 调差2/调差3 分别执行 R2-R4（V4.5 新增） | 多币种处理细节 | E2E / P0 |
| 部分频道阻断 + 其余继续 | 单频道失败不影响其他频道 | 异常分支 | E2E / P1 |
| 失败文件下载 | 阻断后生成可下载的失败文件 | 辅助功能 | E2E / P2 |

## 四、决策依据速查

**纳入冒烟的三个条件**（满足任一即纳入）：

```
1. 核心价值链：缺失则系统完全不可用
2. 资金/状态不可逆变更：涉及金额变动或关键状态转换
3. 跨模块数据流入口：模块 A 的操作直接影响模块 B
```

**不纳入冒烟的排除项**：

```
- 校验规则细节（R1~R6）→ 系统仍可运行，只是精度/安全不足
- 只读展示（漏抓标签、原状态列）→ 不影响业务操作
- 导出功能 → 数据查看通道，非操作入口
- UI 样式/格式 → 非功能验证
- 权限矩阵 → 独立测试维度
- 一次性脚本（数据迁移）→ 非运行时流程
```
