### E2E-001 跨期正常完整结算链路

**优先级**：P0
**关联测试点**：TP-E2E-001
**涉及模块**：SET-01 → SET-02

**前置条件**：
- 剧老板中存在一个未登记的视频，该视频发布日期为上月（登记时不逾期）
- 结算系统中该视频有对应的逾期结算记录（status=0，"未登记"状态）
- 财务人员和运营人员账号均可用

**操作步骤**：

*— 阶段一：SET-01 登记状态分发 —*
1. 在剧老板中对该视频执行登记操作（或通过剧老板 API 触发）
2. 等待约 5 秒（MQ 消费）
3. 登录结算系统，进入"逾期结算处理"页面
4. 切换到【跨期正常未拆分】Tab，搜索该视频 ID

*— 阶段二：SET-02 财务结算处理 —*
5. 确认该视频出现在【跨期正常未拆分】Tab（status=3）
6. 勾选该记录，点击【批量拆分】→ 弹窗确认 → 点击【确认】
7. 等待操作完成
8. 切换到【已拆分】Tab，搜索该视频 ID

**预期结果**：
- 阶段一：视频从"未登记"状态流转为"跨期正常未拆分"（status=3），出现在对应 Tab
- 阶段二：拆分成功，记录进入【已拆分】Tab
- 【已拆分】Tab 中该记录的原状态列显示"跨期正常"（绿色文字）
- 数据库验证：status=2（"已拆分"），original_status=3（"跨期正常"）

### E2E-002 技术漏抓视频完整路径

**优先级**：P0
**关联测试点**：TP-E2E-002
**涉及模块**：SET-04 标签展示 + SET-02 导出

**前置条件**：
- 结算系统中存在符合漏抓条件的视频（scraped_at > 发布次月15日 23:59:59）
- 该视频在"发布视频登记管理"和"逾期结算处理"中均有对应记录

**操作步骤**：

*— 阶段一：SET-04 发布视频登记管理标签展示 —*
1. 登录结算系统，进入"发布视频登记管理"→ 已登记 Tab
2. 找到该漏抓视频，确认视频 ID 下方展示橙色【技术漏抓】标签

*— 阶段二：SET-04 逾期结算管理标签展示 —*
3. 进入"逾期结算处理"页面
4. 在该视频所在的 Tab 中找到该记录，确认同样展示【技术漏抓】标签

*— 阶段三：SET-02 导出验证 —*
5. 在该 Tab 中点击【导出】
6. 打开导出的 Excel 文件，找到该视频行

**预期结果**：
- 阶段一：发布视频登记管理中展示【技术漏抓】标签
- 阶段二：逾期结算处理对应 Tab 中同样展示【技术漏抓】标签（两个页面一致）
- 阶段三：导出文件中「视频标签」字段值 = "技术漏抓"，与列表页展示一致
