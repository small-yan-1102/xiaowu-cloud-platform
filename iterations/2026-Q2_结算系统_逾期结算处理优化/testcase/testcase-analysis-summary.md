### 1.1 我们测什么

本次测试覆盖 **结算系统** 逾期结算处理模块的 V4.5 优化，核心验证以下 4 条价值链路：

```
链路 1: 剧老板登记 → MQ 同步 → 结算系统状态分发（0→1 逾期 / 0→3 跨期正常）
链路 2: 逾期结算导入 → R1~R6 误差处理 → 逐行校验 → 数据入库
链路 3: 财务批量拆分 → 冲销表子集生成 → 无归属收益调整 → 结算单关联
链路 4: 视频标签计算（技术漏抓标识）→ 列表展示 → 导出字段
```

### 5.2 核心预期共识点

#### A. 状态分发边界规则

| 条件 | 结果 | 精度 |
|------|------|------|
| registrationTime ≤ 发布次月28日 **23:59:59** | status = 3（跨期正常） | 秒级（Date.after） |
| registrationTime > 发布次月28日 23:59:59 | status = 1（逾期） | 秒级 |

#### B. R1~R6 误差处理规则

| 规则 | 通过条件 | 失败行为 |
|------|---------|---------|
| R1 占比完整性 | 导入侧占比和 = 系统侧无归属占比和（decimal(10,8)） | 频道级阻断 |
| R2 零值清理 | ROUND(占比×CMS收益,2) > 0 | 舍弃该笔（非失败） |
| R3 误差抹平 | 差额绝对值 ≤ $1.00 | 累加到占比最大的视频 |
| R4 安全阈值 | 差额绝对值 ≤ $1.00 | > $1 阻断频道 |
| R5 极端情况 | R2 后至少有 1 个非零视频 | 全零阻断频道 |
| R6 多收益 | revenue/usRevenue/sgRevenue 独立 R2~R5 | 任一阻断 → 整体阻断 |

#### C. 批量拆分校验规则

| 校验项 | 条件 | 结果 |
|--------|------|------|
| 冲销表到账 | received_status ≠ 0 | 未到账 → 拒绝 |
| 子集唯一性 | checkPipelineIdExist | 已有未结算子集 → 拒绝；已有已结算子集 → 允许 |
| pipelineId | 非空 | 为空 → 拒绝 |

#### D. 漏抓标签判定规则

| 条件 | 展示 |
|------|------|
| scrapedAt > 发布次月15日 23:59:59 | 橙色【技术漏抓】标签 |
| scrapedAt ≤ 发布次月15日 23:59:59 | 不展示 |
| scrapedAt IS NULL | 不展示 |
| publishedDate IS NULL | 不展示 |
