# PRD 需求理解总结（测试视角）

**总结日期**：2026-03-31（增量更新）
**涉及 PRD**：作品管理与交接单改造-PRD.md（V1.2+ 增量更新版，§3.6 字段选项新增、§1.12 触发条件补充、§3.7 同步到配音扩充）
**涉及原型**：AMS 作品管理（localhost:8024，本地地址）、CRM 交接单优化（outputs/交接单优化.html）

---

## 系统全景

本次改造涉及 AMS（内容资产管理系统）和 CRM（客户关系管理系统）两个系统，核心目标是**在作品维度留存发布配置信息**（内部可发布状态、首发时间、频道发布限制、语种配置），为下游分发系统视频发布/制作环节的合规性校验提供数据基础。改造包括 AMS 作品列表新增展示列和详情抽屉（全新功能），CRM 交接单创建/编辑流程增加发布配置联动和语种配置，以及交接单详情页 UI 改造。

## 各模块详细理解

### F00 作品列表调整

**业务目的**：让运营人员在列表层面即可识别作品的内部发布状态，同时提供进入作品详情的快捷入口。

**核心流程**：
1. 运营人员进入 AMS 作品列表
2. 新增「内部是否可发布」列以可视化状态标识展示
3. 点击作品名称链接 → 从右侧滑出详情抽屉（F01）

**业务规则**：
- 列位置：「线索规格」列右侧、「跟进人员」列左侧
- 展示规则：可发布=蓝色圆点+「可发布」；不可发布=橙色圆点+「不可发布」；无数据=「-」
- 列头 Tooltip：鼠标悬停显示「分销商转授权作品，内部运营不可发布」
- 作品名称改为文字链接样式（蓝色/紫色可点击），点击触发 F01 详情抽屉
- 操作列「设置」入口保留不变

### F01 作品详情抽屉（整体新增）

**业务目的**：提供作品多维度信息的集中查看入口，包括基本信息、合约信息、发布配置、语种配置、作品信息和平台数据。

**核心流程**：
1. 在 F00 作品列表点击作品名称
2. 右侧滑入 900px 宽度抽屉
3. 默认显示「概览」Tab，展示左右双栏信息卡片
4. 可切换至「修改记录」Tab 查看字段变更历史
5. 关闭方式：标题栏「×」或点击遮罩层

**业务规则**：
- 概览区左右独立滚动
- 左侧卡片：基本信息（2列网格）→ 合约信息（3列+平台卡片）→ 发布配置（联动显隐）→ 语种配置（条件显示）→ 特殊运作要求
- 右侧卡片：作品信息（Mock 数据）→ 平台数据（2×2 网格，Mock 数据）
- **联动规则**：内部可发布=可发布时，首发时间和可发频道显示；内部可发布=不可发布时，发布配置仅显示此状态，语种配置卡片整体不展示
- 修改记录范围：垂类、别名、入库链接，及 CRM 交接单带入的发布配置、语种配置字段
- 修改记录全量加载无分页，时间倒序
- **修改记录触发条件**（§1.12 新增）：「创建」记录=作品首次入库 AMS 时产生，绿色标签+「创建作品」文字；「修改」记录=上述字段任一变更时产生，蓝色标签+「字段名：旧值（删除线红色）→ 新值（绿色加粗）」
- 合约信息平台卡片：到期状态标签已移除（§1.6 标注"不展示（已移除）"），卡片仅含平台名称、授权有效期、合约状态标签、套餐标签
- 标题栏包含「设置」按钮（沿用现有系统设置弹窗，不在本期改造范围）

**数据流转**：

| 数据实体 | 产生场景 | 变更场景 | 关联影响 |
|---------|---------|---------|---------|
| 发布配置（只读展示） | CRM 交接单带入 | CRM 编辑交接单 | 下游分发系统消费 |
| 语种配置（只读展示） | CRM 交接单带入 | CRM 编辑交接单 | 下游分发系统消费 |
| 修改记录 | 字段变更时自动生成 | 不可变更 | 审计追溯 |
| 作品信息（Mock） | 页面固定展示数据 | 待系统提供实际数据后替换 | 暂无 |

### F03 交接单创建：发布配置联动

**业务目的**：在创建交接单时录入作品的发布配置和语种配置信息，为 AMS 侧数据留存提供数据来源。

**核心流程**：
1. 商务人员在 CRM 新建交接单
2. 默认「不可发布」，发布配置区域隐藏
3. 切换为「可发布」→ 展示频道发布限制、首发时间、字幕/配音语种配置
4. 填写语种后可使用「同步到配音」快捷操作
5. 保存草稿 / 提交

**业务规则**：
- 联动显隐：不可发布→隐藏发布配置和语种配置；可发布→展示并可填写
- 切换为「不可发布」前弹窗确认，确认后清空所有关联字段
- 重置规则：首发时间恢复「随时可发布」、频道恢复「无限制」、语种取消所有勾选
- 语种互斥：同一语种不能同时出现在可发和禁发中，选择时自动从对侧移除
- 「同步到配音」（§3.7，原§3.6 重编号）：位置在字幕禁发下方、配音可发上方，附带说明文案「将字幕可发/禁发同步至配音」；将字幕可发/禁发覆盖写入配音可发/禁发，即时更新无弹窗，同步后配音字段仍可手动调整，字幕为空时清空配音对应字段
- 必填校验：保存草稿和提交均需通过（两者规则一致）；校验失败时错误字段内联高亮，弹窗不关闭
- 保存草稿成功：状态设为「待交接」，Toast「保存成功」，关闭弹窗
- 提交成功：状态设为「已交接」，Toast「提交成功」，关闭弹窗，刷新列表，同时将发布配置和语种配置同步到 AMS 作品维度
- 语种选项来源：与线上交接单语种字典保持一致，交互方式不变。CRM 和 AMS 的语种字段独立配置：CRM 语种来源为线上字典，AMS 语种展示数据来源于 CRM 同步
- 字段选项（§3.6 新增章节）：内部可发布（可发布/不可发布，必填）、首发时间（随时可发布/自定义时间/暂不可发布等待上线通知，内部可发布时必填，**自定义日期不限制范围**）、频道发布限制（无限制=原「合集」/ 仅CP指定频道=原「单开」，内部可发布时必填）、语种（可不填；同一语种不能同时出现在可发和禁发）
- ⚠️ **§3.3 与 §3.6 矛盾**：§3.3 末尾声明「字幕和配音语种，可发和禁发，至少选择一项」，但 §3.6 语种备注为「可不填」，两处描述存在歧义（见 A-7）

### F03-E 交接单编辑（整体新增）

**业务目的**：支持对已创建的交接单进行修改，包括已交接状态下的重新交接。

**核心流程**：
1. 在交接单列表点击编辑 → 打开「编辑交接单」弹窗
2. 自动回显所有已有数据
3. 修改字段内容（交接组数量不可变）
4. 保存：已交接状态→「保存并重新交接」；其他状态→「保存」
5. 关闭时如有未保存修改，弹窗确认

**业务规则**：
- 任何状态下均可编辑（与现有系统「已交接不可编辑」行为相反，为本次改造的关键变更）
- 编辑弹窗不支持新增/删除交接组
- 联动规则与创建弹窗一致（F03 §3.3）
- 「同步到配音」快捷操作与创建弹窗一致（§E.4.1，PRD 引用"F03 § 3.6"但因新增§3.6字段选项章节导致编号错位，实际应为 §3.7）
- 保存并重新交接：已交接状态→Toast「编辑成功，已重新交接」；其他状态→按钮文案「保存」，Toast「编辑成功」
- 关闭确认：有未保存修改时提示「有未保存的修改，确认关闭？」
- 编辑权限沿用现有权限标识「编辑交接单」，有权限时按钮显示（任何状态），无权限时按钮不显示

### F04 交接单详情页 UI 改造

**业务目的**：统一交接单详情页的展示布局，以卡片形式呈现频道/作品信息、账号信息、备注信息和运作要求。

**核心流程**：
1. 在列表点击交接单编号 → 打开详情抽屉
2. 查看频道/作品信息表格（F05 定义）、账号信息、备注信息、运作要求
3. 点击「关闭」退出

**业务规则**：
- 纵向卡片布局：频道/作品信息（全宽）→ 账号信息（左半）+ 备注信息（右半同高）→ 运作要求（全宽）
- 详情页只读，仅有「关闭」按钮
- 适用范围：交接单列表详情抽屉 + 作品库交接单详情抽屉，共两个入口

---

### F05 频道/作品信息表格列定义

**业务目的**：调整交接单中频道/作品信息表格的列结构，新增发布配置和语种配置相关列。

**业务规则**：
- 8 列：作品名称（AMS 内超链接→新标签页打开 AMS 作品详情页；CRM 内纯文本不可点击）、内部是否可发布、首发时间、频道发布限制、字幕可发语种、字幕禁发语种、配音可发语种、配音禁发语种
- 超长内容截断，悬停显示完整内容
- 双入口一致性：交接单列表详情抽屉和作品库交接单详情抽屉使用相同列定义
