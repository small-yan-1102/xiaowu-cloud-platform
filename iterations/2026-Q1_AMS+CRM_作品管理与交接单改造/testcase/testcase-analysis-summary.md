## 一、用例全景概览

### 1.1 我们测什么

本次测试覆盖 **AMS 资产管理系统** 和 **CRM 客户关系管理系统** 两个系统的作品管理与交接单改造功能，核心验证以下 4 条价值链路：

```
链路 1: CRM 创建/编辑交接单 → 发布配置联动 → 语种互斥 → 提交/保存
链路 2: AMS 作品列表新增列 → 作品详情抽屉（含发布配置/语种配置展示）
链路 3: CRM 提交/重新交接 → 同步接口调用（DomesticChannelController.syn()） → AMS 数据更新（跨系统同步）
链路 4: 历史数据订正（首发时间/频道限制/可发布状态/语种 4 类字段批量迁移）
```

## 二、颗粒度分析

### 2.1 用例结构规范

每条用例包含以下标准化字段：

| 字段 | 说明 | 团队关注点 |
|------|------|-----------|
| 优先级 | P0/P1/P2 三级 | **产品**：P0 是否覆盖核心场景 |
| 用例类型 | 冒烟/功能/异常/边界/端到端 | **开发**：异常和边界用例的防御性编码 |
| 关联需求 | PRD 章节编号 | **产品**：用例-需求可追溯 |
| 前置条件 | 登录态/页面定位/数据依赖 | **开发**：测试数据准备配合 |
| 测试步骤 | 2-9 步操作 | **测试**：执行粒度适中 |
| 预期结果 | 3-7 项，分 L1/L2 级别 | **三方**：验收标准一致 |
| 人工介入 | 是/否 + 原因 | **测试**：执行计划安排 |

### 2.2 颗粒度评估

| 评估维度 | 分析结论 | 评价 |
|---------|---------|:----:|
| **步骤粒度** | 平均每条用例 4-5 个操作步骤，最少 2 步（UI 观察类），最多 9 步（E2E 链路） | 适中 |
| **预期粒度** | 平均每条用例 4-5 项预期结果，核心断言（L1）占 85%+，视觉/体验断言（L2）占 15% | 适中 |
| **数据粒度** | 关键用例提供测试数据表，指明字段值和说明 | 充分 |
| **依赖粒度** | 用例间依赖关系明确标注，依赖链最长 3 层 | 清晰 |
| **需求追溯** | 每条用例标注 PRD 章节编号（如 F03 SS3.7），可回溯到需求原文 | 完整 |

**整体颗粒度评价**: 用例颗粒度适中偏细，单条用例聚焦一个验证目标，步骤和预期结果可执行、可判定。适合 AI 直接执行模式。

### 2.3 L1/L2 验证分级说明

| 级别 | 含义 | 判定标准 | 占比 |
|------|------|---------|:----:|
| **L1** | 核心功能验证 | 功能是否正确实现、数据是否正确同步 | ~85% |
| **L2** | UI/体验验证 | 颜色样式、布局对齐、动画效果 | ~15% |

> **共识建议**: L1 不通过 = 用例失败（Bug），L2 不通过 = 标记为 UI 优化项（非阻塞）。
